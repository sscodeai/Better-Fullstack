La compétence d'agent Better Fullstack apprend aux agents de codage à utiliser le CLI au lieu d'écrire manuellement des projets de démarrage. Il mappe la pile demandée aux parties graphiques, exécute d'abord un essai à sec, les échafaudages avec l'installation des dépendances désactivée et rapporte les commandes de suivi exactes.

Utilisez-le lorsque votre agent prend en charge les compétences et que vous souhaitez obtenir le chemin le plus rapide entre une invite et un projet Better Fullstack généré. Utilisez plutôt le [serveur MCP](/docs/ai/mcp/) lorsque vous souhaitez plutôt une recherche de schéma structurée et une planification des appels d'outils.

## Installer

La compétence réside dans ce référentiel à `.agents/skills/better-fullstack`. Étant donné que le référentiel contient également d'autres compétences internes, installez uniquement `better-fullstack`.

Installer pour le Codex :

<PMTabs
  npm="npx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  pnpm="pnpm dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  bun="bunx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  yarn="yarn dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
/>

Installez plusieurs agents courants :

```bash
bunx skills@latest add Marve10s/Better-Fullstack \
  --skill better-fullstack \
  --agent codex claude-code cursor github-copilot gemini-cli \
  --global \
  --yes
```

Pour une installation locale du projet, omettez `--global` de la même commande. Les installations de projet sont utiles lorsqu'une équipe souhaite que la version de la compétence soit enregistrée avec le référentiel.

## Utiliser

Invoquez explicitement la compétence :

```text
Use $better-fullstack to create a React + Vite app with Hono, SQLite, Drizzle, tRPC, Tailwind, DaisyUI, Pino, Vitest, and Biome.
```

Ou demandez naturellement. Les agents qui prennent en charge la correspondance implicite des compétences peuvent choisir la compétence lorsque la demande concerne l'échafaudage, la planification ou l'extension d'un projet Better Fullstack.

## Ce que fait la compétence

1. Mappe la pile demandée en parties de graphique Better Fullstack et en indicateurs CLI.
2. Exécute un essai à sec CLI non interactif.
3. Exécute la même commande sans `--dry-run` lorsque le plan réussit.
4. Utilise `--no-install` et `--no-git` sauf si vous demandez explicitement ces effets secondaires.
5. Signale la commande utilisée, tout ajustement de compatibilité et les prochaines commandes d'installation/test/exécution.

## Compétence contre MCP

| Chemin | Idéal pour |
| --- | --- |
| Compétence d'agent | Échafaudage rapide piloté par CLI lorsque la pile peut être exprimée sous forme d'indicateurs ou de parties de graphique. |
| Serveur MCP | Recherche de schéma structurée, vérifications de compatibilité, planification d'essais à sec et mutation de projets existants via des appels d'outils. |

Les deux chemins utilisent le même générateur Better Fullstack. La compétence est la voie la plus légère ; MCP est l'itinéraire le plus structuré.
