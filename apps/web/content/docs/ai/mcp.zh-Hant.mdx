Better Fullstack MCP server 讓 AI 代理透過結構化工具呼叫來產生項目，而不是猜測 CLI 參數。它負責查詢 schema、驗證相容性、預覽產生結果、建立項目，以及為現有項目添加功能。

此頁面介紹 Better Fullstack 的 MCP server。產生專案時選擇的 `mcp` addon 是另一件事：它會把 MCP client/server 配置寫入產生出來的應用，並可能執行外部設定指令。

如果想要使用更輕量的 CLI 驅動流程，請安裝 [Better Fullstack agent skill](/docs/ai/skills/)。 skill 和 MCP server 使用同一個生成器：skill 更適合快速的參數式腳手架流程，MCP 則更適合結構化工具呼叫。

## 設定

使用你偏好的套件管理器啟動 server：

<PMTabs
  npm="npx -y create-better-fullstack@latest mcp"
  pnpm="pnpm dlx create-better-fullstack@latest mcp"
  bun="bunx create-better-fullstack@latest mcp"
  yarn="yarn dlx create-better-fullstack@latest mcp"
/>

### Claude Code

```bash
claude mcp add --transport stdio better-fullstack -- npx -y create-better-fullstack@latest mcp
```

如果你喜歡 pnpm、Bun 或 Yarn，請使用上方相同啟動方式表。

### Cursor

```json
{
  "mcpServers": {
    "better-fullstack": {
      "command": "npx",
      "args": ["-y", "create-better-fullstack@latest", "mcp"]
    }
  }
}
```

### VS Code

```json
{
  "servers": {

      "type": "stdio",
```

## 工具

完整的輸入、輸出、範例和安全寫入行為，請參閱 [MCP tools reference](/docs/ai/mcp-tools/)。

| 工具 | 用途 |
| --- | --- |
| `bfs_get_guidance` | 傳回工作流程規則和欄位語義。先調用它。 |
| `bfs_get_schema` | 傳回配置分類的有效選項。 |
| `bfs_check_compatibility` | 驗證 stack，並傳回包含 warnings 的調整後 selections。 |
| `bfs_plan_project` | 在記憶體中 dry-run 專案產生結果。 |
| `bfs_create_project` | 將新項目寫入磁碟。 |
| `bfs_plan_addition` | 驗證現有項目的 addon 變更。 |
| `bfs_add_feature` | 為現有項目新增支援的 addons。 |

<Callout kind="info" title="安装行为">
MCP 建立專案時會跳過普通依賴安裝，以避免長時間的代理呼叫。如果產生項目包含 `mcp` addon，該 addon 的設定流程仍可能為了 MCP 用戶端設定而執行外部命令。
</Callout>

## 推薦工具順序

新項目：

1. `bfs_get_guidance`
2. `bfs_get_schema`
3. `bfs_check_compatibility`
4. `bfs_plan_project`
5. `bfs_create_project`

現有項目：

1. `bfs_plan_addition`
2. `bfs_add_feature`

## Agents 關鍵規則

- `frontend` 是 array，可以包含多個 frontend targets。
- `none` 會停用一個分類。
- 先設定 `ecosystem`，這樣 agent 會使用正確的字段族。
- Array fields 包括 `frontend`、`addons`、`examples`、`rustLibraries`、`pythonAi`、`javaLibraries` 和 `javaTestingLibraries`。
- Project planning、creation 和 addition responses 包含 `graphSummary`、`effectiveStack` 和 `stackPartSpecs`，用於 graph-backed stack view。
- 目前 MCP project schemas 不把 `aiDocs` 暴露為 create field；project creation 使用 server 內建的 AI-doc behavior。
- 建立專案前始終呼叫 `bfs_check_compatibility`。

## Agent prompt

```text
Use Better Fullstack MCP. First call bfs_get_guidance to understand
the rules, then bfs_get_schema for the relevant categories, then
bfs_check_compatibility to validate the stack, then bfs_plan_project
for a dry-run. Only call bfs_create_project after the dry-run succeeds.
Skip normal dependency installation and tell me the exact commands to
install, test, and run the project.
```
