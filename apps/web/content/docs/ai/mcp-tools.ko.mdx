MCP 서버는 에이전트를 위한 구조화된 도구를 노출합니다. 에이전트에게 산문에서 CLI 플래그를 추측하도록 요청하거나 긴 명령을 직접 복사하는 대신 이를 사용하세요.

## 권장 워크플로

### 새 프로젝트

1. `bfs_get_guidance`
2. `bfs_get_schema`
3. `bfs_check_compatibility`
4. `bfs_plan_project`
5. `bfs_create_project`

### 기존 프로젝트

1. `bfs_get_guidance`
2. `bfs_plan_addition`
3. `bfs_add_feature`

<Callout kind="warn" title="Mutation behavior">
`bfs_plan_project`은 테스트 실행 전용입니다. `bfs_create_project`은 새 프로젝트를 디스크에 씁니다. `bfs_add_feature`는 기존 프로젝트를 변경합니다. MCP 프로젝트 생성 및 기능 추가는 종속성을 설치하지 않습니다.
</Callout>

<Callout kind="info" title="Graph metadata">
프로젝트 계획, 생성 및 추가 도구에는 `graphSummary`, `effectiveStack` 및 `stackPartSpecs`가 포함되어 있으므로 에이전트는 `bts.jsonc`이 지속되는 동일한 그래프 기반 스택 보기를 검사할 수 있습니다.
</Callout>

## 도구 참조

### `bfs_get_guidance`

| 필드 | 세부 |
| --- | --- |
| 목적 | 워크플로 규칙, 필드 의미 체계, 모호성 규칙 및 중요한 제약 조건을 반환합니다. |
| 전화할 때 | 먼저, 다른 Better Fullstack MCP 도구보다 먼저. |
| 파일 쓰기 | 아니요. |
| 필수 입력 | 없음. |
| 선택적 입력 | 없음. |
| 산출 | 권장 도구 순서, 배열 필드, 생태계 규칙 및 예시를 다루는 JSON 지침입니다. |

```json
{}
```

### `bfs_get_schema`

| 필드 | 세부 |
| --- | --- |
| 목적 | 하나의 범주 또는 모든 범주에 대해 유효한 옵션 값을 반환합니다. |
| 전화할 때 | 스택을 계획하기 전, 특히 사용자가 비공식적으로 기술에 이름을 붙인 경우. |
| 파일 쓰기 | 아니요. |
| 필수 입력 | 없음. |
| 선택적 입력 | `category`, `ecosystem`. |
| 산출 | 카테고리 또는 생태계별로 필터링된 JSON 스키마/옵션 데이터입니다. |

```json
{
  "category": "database",
  "ecosystem": "typescript"
}
```

### `bfs_check_compatibility`

| 필드 | 세부 |
| --- | --- |
| 목적 | 스택 조합의 유효성을 검사하고 경고와 함께 조정된 선택 항목을 반환합니다. |
| 전화할 때 | `bfs_plan_project` 또는 `bfs_create_project` 이전. |
| 파일 쓰기 | 아니요. |
| 필수 입력 | `ecosystem`. |
| 선택적 입력 | 프런트엔드, 백엔드, 런타임, 데이터베이스, ORM, API, 인증, 서비스, UI/CSS, 모바일 필드, 애드온, 예제, 패키지 관리자, 배포 필드 및 생태계별 필드입니다. |
| 산출 | 변경 사항, 메모 및 조정된 스택 데이터에 대한 JSON 호환성 분석. |

```json
{
  "ecosystem": "typescript",
  "frontend": ["next"],
  "backend": "self",
  "runtime": "none",
  "database": "postgres",
  "orm": "drizzle",
  "api": "trpc"
}
```

### `bfs_plan_project`

| 필드 | 세부 |
| --- | --- |
| 목적 | 메모리에 프로젝트를 생성하고 파일 트리 요약을 반환합니다. |
| 전화할 때 | 호환성 검증 후 및 파일 쓰기 전. |
| 파일 쓰기 | 아니요. |
| 필수 입력 | 없음. 하지만 `projectName` 및 `ecosystem`이 권장됩니다. |
| 선택적 입력 | 전체 프로젝트 스택 선택. |
| 산출 | JSON 성공 플래그와 생성된 파일/디렉터리 요약, `graphSummary`, `effectiveStack` 및 `stackPartSpecs`. |

```json
{
  "projectName": "my-app",
  "ecosystem": "typescript",
  "frontend": ["tanstack-router"],
  "backend": "hono",
  "runtime": "bun",
  "database": "sqlite",
  "orm": "drizzle"
}
```

### `bfs_create_project`

| 필드 | 세부 |
| --- | --- |
| 목적 | 새로 생성된 프로젝트를 디스크에 씁니다. |
| 전화할 때 | `bfs_plan_project`이 성공하고 사용자가 승인한 후에만 가능합니다. |
| 파일 쓰기 | 예, 프로젝트 디렉터리와 생성된 파일을 생성합니다. |
| 필수 입력 | `projectName`. |
| 선택적 입력 | 전체 프로젝트 스택 선택. |
| 산출 | JSON 성공 플래그, 프로젝트 디렉터리, 파일 수, 그래프 메타데이터, 애드온 경고 및 설치 명령 지침. |

```json
{
  "projectName": "my-app",
  "ecosystem": "java",
  "javaWebFramework": "spring-boot",
  "javaBuildTool": "maven",
  "javaTestingLibraries": ["junit5"]
}
```

### `bfs_plan_addition`

| 필드 | 세부 |
| --- | --- |
| 목적 | `bts.jsonc`을 읽고 기존 프로젝트에 대해 제안된 추가 사항을 검증합니다. |
| 전화할 때 | `bfs_add_feature`을(를) 사용하여 기존 프로젝트를 변경하기 전. |
| 파일 쓰기 | 아니요. |
| 필수 입력 | `projectDir`. |
| 선택적 입력 | `addons`, `webDeploy`, `serverDeploy`. |
| 산출 | 기존 구성 요약, 제안된 추가 사항, 두 가지 모두에 대한 그래프 메타데이터, 이미 존재하는 애드온 및 호환성 경고. |

```json
{
  "projectDir": "/absolute/path/to/my-app",
  "addons": ["mcp", "skills"]
}
```

### `bfs_add_feature`

| 필드 | 세부 |
| --- | --- |
| 목적 | 기존 Better Fullstack 프로젝트에 지원되는 애드온 또는 배포 설정을 추가합니다. |
| 전화할 때 | `bfs_plan_addition`이 성공한 후 사용자가 승인합니다. |
| 파일 쓰기 | 예, 기존 프로젝트를 변경합니다. |
| 필수 입력 | `projectDir`. |
| 선택적 입력 | `addons`, `webDeploy`, `serverDeploy`, `packageManager`. |
| 산출 | 추가 핸들러의 JSON 결과, 업데이트 후 메타데이터 그래프 및 사용 가능한 경우 설치 지침이 제공됩니다. |

```json
{
  "projectDir": "/absolute/path/to/my-app",
  "addons": ["biome"],
  "packageManager": "bun"
}
```

## 에이전트 규칙

- `frontend`, `addons`, `examples`, `rustLibraries`, `pythonAi`, `javaLibraries` 및 `javaTestingLibraries`을 배열로 처리합니다.
- 생태계별 필드를 선택하기 전에 `ecosystem`을 설정하세요.
- 사용자가 카테고리를 명시적으로 비활성화하려는 경우 `none`을 선호합니다.
- MCP 프로젝트 생성은 현재 모든 CLI 워크플로 플래그를 노출하지 않습니다. 예를 들어 `aiDocs`은 프로젝트 생성 중에 서버에 내장된 기본값을 사용합니다.
- 사용자가 테스트 실행 또는 추가 계획을 승인할 때까지 쓰기 도구를 호출하지 마십시오.
- MCP은 종속성을 설치하지 않기 때문에 MCP 생성 또는 추가 후에 실행할 설치/테스트 명령을 사용자에게 알려줍니다.
