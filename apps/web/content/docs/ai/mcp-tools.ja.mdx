MCP サーバーは、エージェント用の構造化ツールを公開します。エージェントに散文から CLI フラグを推測したり、長いコマンドを手動でコピーしたりするよう依頼する代わりに、これらを使用します。

## 推奨されるワークフロー

### 新しいプロジェクト

1. `bfs_get_guidance`
2. `bfs_get_schema`
3. `bfs_check_compatibility`
4. `bfs_plan_project`
5. `bfs_create_project`

### 既存のプロジェクト

1. `bfs_get_guidance`
2. `bfs_plan_addition`
3. `bfs_add_feature`

<Callout kind="warn" title="Mutation behavior">
`bfs_plan_project` はドライランのみです。 `bfs_create_project` は、新しいプロジェクトをディスクに書き込みます。 `bfs_add_feature` は、既存のプロジェクトを変更します。 MCP プロジェクトの作成と機能の追加では、依存関係はインストールされません。
</Callout>

<Callout kind="info" title="Graph metadata">
プロジェクトの計画、作成、追加ツールには `graphSummary`、`effectiveStack`、`stackPartSpecs` が含まれているため、エージェントは `bts.jsonc` が保持する同じグラフ派生スタック ビューを検査できます。
</Callout>

## ツールリファレンス

### `bfs_get_guidance`

| 分野 | 詳細 |
| --- | --- |
| 目的 | ワークフロー ルール、フィールド セマンティクス、あいまいさルール、および重要な制約を返します。 |
| 電話をかけるタイミング | まず、他の Better Fullstack MCP ツールの前にあります。 |
| ファイルを書き込みます | いいえ。 |
| 必須入力 | なし。 |
| オプションの入力 | なし。 |
| 出力 | 推奨されるツールの順序、配列フィールド、エコシステム ルール、例をカバーする JSON ガイダンス。 |

```json
{}
```

### `bfs_get_schema`

| 分野 | 詳細 |
| --- | --- |
| 目的 | 1 つのカテゴリまたはすべてのカテゴリの有効なオプション値を返します。 |
| 電話をかけるタイミング | スタックを計画する前、特にユーザーがテクノロジーに非公式に名前を付けた場合。 |
| ファイルを書き込みます | いいえ。 |
| 必須入力 | なし。 |
| オプションの入力 | `category`、`ecosystem`。 |
| 出力 | カテゴリまたはエコシステムによってフィルタリングされた JSON スキーマ/オプション データ。 |

```json
{
  "category": "database",
  "ecosystem": "typescript"
}
```

### `bfs_check_compatibility`

| 分野 | 詳細 |
| --- | --- |
| 目的 | スタックの組み合わせを検証し、警告付きで調整された選択を返します。 |
| 電話をかけるタイミング | `bfs_plan_project` または `bfs_create_project` より前。 |
| ファイルを書き込みます | いいえ。 |
| 必須入力 | `ecosystem`。 |
| オプションの入力 | フロントエンド、バックエンド、ランタイム、データベース、ORM、API、認証、サービス、UI/CSS、モバイル フィールド、アドオン、サンプル、パッケージ マネージャー、デプロイメント フィールド、およびエコシステム固有のフィールド。 |
| 出力 | 変更、メモ、調整されたスタック データによる JSON 互換性分析。 |

```json
{
  "ecosystem": "typescript",
  "frontend": ["next"],
  "backend": "self",
  "runtime": "none",
  "database": "postgres",
  "orm": "drizzle",
  "api": "trpc"
}
```

### `bfs_plan_project`

| 分野 | 詳細 |
| --- | --- |
| 目的 | メモリ内にプロジェクトを生成し、ファイル ツリーの概要を返します。 |
| 電話をかけるタイミング | 互換性検証後、ファイルの書き込み前。 |
| ファイルを書き込みます | いいえ。 |
| 必須入力 | なし。ただし、`projectName` および `ecosystem` が推奨されます。 |
| オプションの入力 | 完全なプロジェクト スタックの選択。 |
| 出力 | JSON 成功フラグと生成されたファイル/ディレクトリの概要、`graphSummary`、`effectiveStack`、および `stackPartSpecs`。 |

```json
{
  "projectName": "my-app",
  "ecosystem": "typescript",
  "frontend": ["tanstack-router"],
  "backend": "hono",
  "runtime": "bun",
  "database": "sqlite",
  "orm": "drizzle"
}
```

### `bfs_create_project`

| 分野 | 詳細 |
| --- | --- |
| 目的 | 新しく生成されたプロジェクトをディスクに書き込みます。 |
| 電話をかけるタイミング | `bfs_plan_project` が成功し、ユーザーが承認した場合のみ。 |
| ファイルを書き込みます | はい、プロジェクト ディレクトリと生成されたファイルが作成されます。 |
| 必須入力 | `projectName`。 |
| オプションの入力 | 完全なプロジェクト スタックの選択。 |
| 出力 | JSON 成功フラグ、プロジェクト ディレクトリ、ファイル数、グラフ メタデータ、アドオン警告、インストール コマンド ガイダンス。 |

```json
{
  "projectName": "my-app",
  "ecosystem": "java",
  "javaWebFramework": "spring-boot",
  "javaBuildTool": "maven",
  "javaTestingLibraries": ["junit5"]
}
```

### `bfs_plan_addition`

| 分野 | 詳細 |
| --- | --- |
| 目的 | `bts.jsonc` を読み取り、既存のプロジェクトに提案された追加を検証します。 |
| 電話をかけるタイミング | 既存のプロジェクトを `bfs_add_feature` で変更する前。 |
| ファイルを書き込みます | いいえ。 |
| 必須入力 | `projectDir`。 |
| オプションの入力 | `addons`、`webDeploy`、`serverDeploy`。 |
| 出力 | 既存の構成の概要、提案された追加、両方のグラフ メタデータ、既存のアドオン、および互換性の警告。 |

```json
{
  "projectDir": "/absolute/path/to/my-app",
  "addons": ["mcp", "skills"]
}
```

### `bfs_add_feature`

| 分野 | 詳細 |
| --- | --- |
| 目的 | サポートされているアドオンまたは展開設定を既存の Better Fullstack プロジェクトに追加します。 |
| 電話をかけるタイミング | `bfs_plan_addition` が成功し、ユーザーが承認した後。 |
| ファイルを書き込みます | はい、既存のプロジェクトを変更します。 |
| 必須入力 | `projectDir`。 |
| オプションの入力 | `addons`、`webDeploy`、`serverDeploy`、`packageManager`。 |
| 出力 | 追加ハンドラーからの JSON 結果、更新後のグラフ メタデータ、および利用可能な場合はインストール ガイダンス。 |

```json
{
  "projectDir": "/absolute/path/to/my-app",
  "addons": ["biome"],
  "packageManager": "bun"
}
```

## エージェントルール

- `frontend`、`addons`、`examples`、`rustLibraries`、`pythonAi`、`javaLibraries`、および `javaTestingLibraries` を配列として扱います。
- エコシステム固有のフィールドを選択する前に、`ecosystem` を設定します。
- ユーザーがカテゴリを明示的に無効にしたい場合は、`none` を優先します。
- 現在、MCP プロジェクトの作成では、すべての CLI ワークフロー フラグが公開されるわけではありません。たとえば、`aiDocs` は、プロジェクトの作成時にサーバーの組み込みのデフォルトを使用します。
- ユーザーが予行演習または追加計画を承認するまで、書き込みツールを呼び出さないでください。
- MCP は依存関係をインストールしないため、MCP の作成または追加後に実行するインストール/テスト コマンドをユーザーに指示します。
