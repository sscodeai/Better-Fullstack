Better Fullstack agent skill 會教程式碼代理程式使用 CLI，而不是手寫 starter 專案。它會把請求的技術棧映射到 graph parts，先執行 dry-run，然後在禁用依賴安裝的情況下產生項目，並報告精確的後續命令。

當你的代理人支持 skills，並且你希望從一句提示詞快速產生 Better Fullstack 項目時，使用它。當你需要結構化的 schema 查詢和工具呼叫規劃時，請改用 [MCP server](/docs/ai/mcp/)。

## 安裝

該 skill 位於此倉庫的 `.agents/skills/better-fullstack`。因為倉庫也包含其他內部 skills，請只安裝 `better-fullstack`。

為 Codex 安裝：

<PMTabs
  npm="npx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  pnpm="pnpm dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  bun="bunx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  yarn="yarn dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
/>

為幾個常見代理安裝：

```bash
bunx skills@latest add Marve10s/Better-Fullstack \
  --skill better-fullstack \
  --agent codex claude-code cursor github-copilot gemini-cli \
  --global \
  --yes
```

如需專案本機安裝，從相同指令省略 `--global`。當團隊希望把 skill 版本與倉庫一起記錄時，專案安裝就很有用。

## 使用

顯式調用 skill：

```text
Use $better-fullstack to create a React + Vite app with Hono, SQLite, Drizzle, tRPC, Tailwind, DaisyUI, Pino, Vitest, and Biome.
```

也可以自然提問。支援隱式 skill 匹配的代理，可以在請求涉及 Better Fullstack 項目的生成、規劃或擴展時選擇該 skill。

## Skill 做什麼

1. 將請求的技術堆疊對應為 Better Fullstack graph parts 和 CLI flags。
2. 執行非互動式 CLI dry-run。
3. 當計劃成功時，執行不含 `--dry-run` 的相同指令。
4. 除非你明確要求這些副作用，否則使用 `--no-install` 和 `--no-git`。
5. 報告使用的命令、任何相容性調整，以及下一步安裝、測試和執行命令。

## Skill vs MCP

| 路徑 | 最適合 |
| --- | --- |
| Agent skill | 當技術堆疊可以表示為 flags 或 graph parts 時，用於快速透過 CLI 產生項目。 |
| MCP server | 透過工具呼叫進行結構化 schema 查詢、相容性檢查、dry-run 規劃和現有專案修改。 |

兩條路徑使用同一個 Better Fullstack generator。 skill 是更輕的路徑；MCP 是更結構化的路徑。
