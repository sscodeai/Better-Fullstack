Better Fullstack unterstützt drei AI-Workflows:

- Generieren Sie projektlokale Anweisungsdateien mit `--ai-docs`.
- Installieren Sie den Agenten-Skill Better Fullstack, damit Programmieragenten den CLI verwenden, anstatt Starterdateien handschriftlich zu schreiben.
- Lassen Sie einen AI-Coding-Agent den Better Fullstack MCP-Server für Schemasuche, Kompatibilitätsprüfungen, Probeläufe und Projekterstellung verwenden.

## Bevorzugter Arbeitsablauf

1. Wählen Sie das Zielökosystem: `typescript`, `react-native`, `rust`, `python`, `go`, `java` oder `elixir`.
2. Verwenden Sie den Agenten-Skill Better Fullstack, den Server MCP oder die explizite Referenz CLI.
3. Überprüfen Sie die Kompatibilität vor dem Gerüstbau.
4. Führen Sie einen Probelauf durch, um den Dateibaum zu überprüfen.
5. Gerüst mit deaktivierter Abhängigkeitsinstallation, wenn ein Agent den Fluss steuert.
6. Teilen Sie dem Benutzer die genauen Installations-, Test- und Ausführungsbefehle mit.

## Generierte AI-Dokumente

Verwenden Sie `--ai-docs`, um projektlokale Kontextdateien für Codierungsagenten zu generieren:

| Wert | Datei generiert |
| --- | --- |
| `claude-md` | `CLAUDE.md` |
| `agents-md` | `Agents.md` |
| `cursorrules` | `.cursorrules` |
| `none` | Keine AI-Dokumente |

Es werden mehrere Werte akzeptiert:

```npm
npm create better-fullstack@latest my-app -- --ai-docs claude-md agents-md
```

Verwenden Sie `--ai-docs none`, um diese Dateien zu deaktivieren.

## CLI-Workflow

Übergeben Sie für Agenten ohne MCP-Zugriff explizite Flags und vermeiden Sie Nebenwirkungen:

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --runtime none \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --db-setup none \
  --ai-docs agents-md \
  --version-channel stable \
  --no-install \
  --no-git
```

<Callout kind="info" title="CLI rules for agents">
`--yes` akzeptiert Standardwerte und ist nicht mit Core-Stack-Flags kompatibel. Explizite Flags überspringen nur die Kategorien, die sie abdecken. Übergeben Sie `none` für optionale Kategorien, die Sie deaktivieren möchten.
</Callout>

## Arbeitsablauf für Agentenfähigkeiten

Der Agenten-Skill Better Fullstack befindet sich in diesem Repository und lehrt unterstützte Agenten, den CLI mit dem Benchmarking-Schnellworkflow zu verwenden: Stapel zuordnen, einen Probelauf ausführen, Gerüst ohne Installations- oder Git-Nebeneffekte erstellen und dann Folgebefehle melden.

Weitere Informationen zu den Installationsbefehlen Codex, Claude Code, Cursor, GitHub Copilot und Gemini CLI finden Sie im [Agent Skill Setup Guide](/docs/ai/skills/).

## AI Anbieteroptionen

TypeScript-Projekte können eine AI-Option mit `--ai` auswählen: `vercel-ai`, `mastra`, `voltagent`, `langgraph`, `openai-agents`, `google-adk`, `modelfusion`, `langchain`, `llamaindex`, `tanstack-ai`, `ai-cli` oder `none`.

Python-Projekte verwenden die separate Mehrfachauswahloption `--python-ai` mit `langchain`, `llamaindex`, `openai-sdk`, `anthropic-sdk`, `langgraph`, `crewai` oder `none`.

Die Platzierung der Abhängigkeiten hängt vom ausgewählten Framework und Backend ab. Serverseitige Pakete werden dem Serverpaket oder der Self-Backend-App hinzugefügt; Unterstützte Frontend-Integrationen erhalten ihre passenden Client-Hooks oder Pakete.

## MCP-Workflow

Der Server Better Fullstack MCP stellt Schemasuche, Kompatibilitätsprüfungen, Probelaufvorschauen, Projekterstellung und Funktionserweiterungen als strukturierte Tools zur Verfügung. Siehe die [MCP Einrichtungsanleitung](/docs/ai/mcp/).

Dies unterscheidet sich vom generierten Projekt-Add-on `mcp`. Der Server hilft einem Agenten beim Erstellen von Better Fullstack-Projekten. Das Addon schreibt die MCP-Konfiguration in ein generiertes Projekt.
