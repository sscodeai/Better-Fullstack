Mit der Agentenfähigkeit Better Fullstack lernen Programmieragenten, die CLI zu verwenden, anstatt Starterprojekte handschriftlich zu schreiben. Es ordnet den angeforderten Stapel Diagrammteilen zu, führt zunächst einen Probelauf durch, erstellt Gerüste mit deaktivierter Abhängigkeitsinstallation und meldet die genauen Folgebefehle.

Verwenden Sie dies, wenn Ihr Agent Fähigkeiten unterstützt und Sie den schnellsten Weg von einer Eingabeaufforderung zu einem generierten Better Fullstack-Projekt wünschen. Verwenden Sie den [MCP-Server](/docs/ai/mcp/), wenn Sie stattdessen eine strukturierte Schemasuche und Toolaufrufplanung wünschen.

## Installieren

Der Skill befindet sich in diesem Repository unter `.agents/skills/better-fullstack`. Da das Repo auch andere interne Fähigkeiten enthält, installieren Sie nur `better-fullstack`.

Für Codex installieren:

<PMTabs
  npm="npx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  pnpm="pnpm dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  bun="bunx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  yarn="yarn dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
/>

Installieren Sie für mehrere gängige Agenten:

```bash
bunx skills@latest add Marve10s/Better-Fullstack \
  --skill better-fullstack \
  --agent codex claude-code cursor github-copilot gemini-cli \
  --global \
  --yes
```

Für eine projektlokale Installation lassen Sie `--global` im selben Befehl weg. Projektinstallationen sind nützlich, wenn ein Team möchte, dass die Skillversion im Repository aufgezeichnet wird.

## Verwenden

Rufen Sie den Skill explizit auf:

```text
Use $better-fullstack to create a React + Vite app with Hono, SQLite, Drizzle, tRPC, Tailwind, DaisyUI, Pino, Vitest, and Biome.
```

Oder fragen Sie natürlich. Agenten, die den impliziten Skill-Abgleich unterstützen, können den Skill auswählen, wenn es bei der Anfrage um Gerüstbau, Planung oder Erweiterung eines Better Fullstack-Projekts geht.

## Was die Fähigkeit bewirkt

1. Ordnet den angeforderten Stapel den Diagrammteilen Better Fullstack und den Flags CLI zu.
2. Führt einen nicht interaktiven CLI-Probelauf aus.
3. Führt denselben Befehl ohne `--dry-run` aus, wenn der Plan erfolgreich ist.
4. Verwendet `--no-install` und `--no-git`, es sei denn, Sie fragen ausdrücklich nach diesen Nebenwirkungen.
5. Meldet den verwendeten Befehl, etwaige Kompatibilitätsanpassungen und die nächsten Installations-/Test-/Ausführungsbefehle.

## Fähigkeit vs. MCP

| Weg | Am besten für |
| --- | --- |
| Agentenfähigkeit | Schnelles CLI-gesteuertes Gerüst, wenn der Stapel als Flags oder Diagrammteile ausgedrückt werden kann. |
| MCP Server | Strukturierte Schemasuche, Kompatibilitätsprüfungen, Probelaufplanung und Mutation bestehender Projekte durch Toolaufrufe. |

Beide Pfade verwenden denselben Better Fullstack-Generator. Die Fertigkeit ist der leichtere Weg; MCP ist die strukturiertere Route.
