---
title: エージェントスキル
description: Codex、Claude Code、Cursor、およびその他のエージェントでの高速 CLI 駆動のスキャフォールディングのために、Better Fullstack エージェント スキルをインストールします。
---

Better Fullstack エージェント スキルは、手書きのスターター プロジェクトの代わりに CLI を使用するようにコーディング エージェントに教えます。要求されたスタックをグラフ部分にマップし、最初に予行演習を実行し、依存関係のインストールを無効にしてスキャフォールドを作成し、正確なフォローアップ コマンドを報告します。

エージェントがスキルをサポートしており、プロンプトから生成された Better Fullstack プロジェクトへの最速のパスが必要な場合に、これを使用します。構造化スキーマ検索とツール呼び出し計画が必要な場合は、代わりに [MCP サーバー](/docs/ai/mcp/) を使用してください。

## インストール

スキルはこのリポジトリ `.agents/skills/better-fullstack` に存在します。リポジトリには他の内部スキルも含まれているため、`better-fullstack` のみをインストールします。

Codex 用にインストールします。

<PMTabs
  npm="npx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  pnpm="pnpm dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  bun="bunx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  yarn="yarn dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
/>

いくつかの一般的なエージェント用にインストールします。

```bash
bunx skills@latest add Marve10s/Better-Fullstack \
  --skill better-fullstack \
  --agent codex claude-code cursor github-copilot gemini-cli \
  --global \
  --yes
```

プロジェクト ローカル インストールの場合は、同じコマンドから `--global` を省略します。プロジェクトのインストールは、チームがスキルのバージョンをリポジトリに記録したい場合に便利です。

## 使用

スキルを明示的に呼び出します。

```text
Use $better-fullstack to create a React + Vite app with Hono, SQLite, Drizzle, tRPC, Tailwind, DaisyUI, Pino, Vitest, and Biome.
```

または自然に尋ねてください。暗黙的なスキル マッチングをサポートするエージェントは、リクエストが Better Fullstack プロジェクトのスキャフォールディング、計画、または拡張に関するものである場合にスキルを選択できます。

## スキルの内容

1. 要求されたスタックを Better Fullstack グラフ部分と CLI フラグにマップします。
2. 非対話型の CLI ドライランを実行します。
3. 計画が成功した場合は、`--dry-run` を使用せずに同じコマンドを実行します。
4. これらの副作用を明示的に要求しない限り、`--no-install` および `--no-git` を使用します。
5. 使用されたコマンド、互換性調整、​​および次回のインストール/テスト/実行コマンドをレポートします。

## スキル vs MCP

| パス | こんな方に最適 |
| --- | --- |
| エージェントスキル | スタックをフラグまたはグラフ部分として表現できる場合の、CLI 主導の高速スキャフォールディング。 |
| MCP サーバー | 構造化スキーマの検索、互換性チェック、ドライラン計画、およびツール呼び出しによる既存プロジェクトの変更。 |

どちらのパスも同じ Better Fullstack ジェネレーターを使用します。スキルは軽いルートです。 MCP は、より構造化されたルートです。
