Better Fullstack verfügt über eine kleine Befehlsoberfläche. Die meisten Benutzer benötigen nur `create`, `add` und `history`; AI-Agent-Workflows verwenden `mcp`.

## Befehle

| Befehl | Zweck |
| --- | --- |
| `create` | Erstellen Sie ein Gerüst für ein neues Projekt aus einer Stack-Konfiguration. Dies ist der Standardbefehl. |
| `add` | Fügen Sie unterstützte Add-ons oder Bereitstellungseinstellungen zu einem vorhandenen Better Fullstack-Projekt hinzu. |
| `history` | Zeigen, exportieren oder löschen Sie den lokalen Gerüstverlauf und reproduzierbare Befehle. |
| `mcp` | Starten Sie den Server Better Fullstack MCP über stdio für AI Codierungsagenten. |

## Dienstprogrammbefehle

| Befehl | Zweck |
| --- | --- |
| `docs` | Öffnen Sie die öffentliche Dokumentationsseite Better Fullstack in Ihrem Browser. |
| `builder` | Öffnen Sie das interaktive Stack Builder. |
| `sponsors` | Better Fullstack Sponsoren anzeigen. |

## Führen Sie Dienstprogrammbefehle aus

Dokumente öffnen:

<PMTabs
  npm="npx -y create-better-fullstack@latest docs"
  pnpm="pnpm dlx create-better-fullstack@latest docs"
  bun="bunx create-better-fullstack@latest docs"
  yarn="yarn dlx create-better-fullstack@latest docs"
/>

## Betreuerbefehle

`update-deps` ist für Betreuer gedacht, die Abhängigkeitsversionen überprüfen und aktualisieren. Es wird absichtlich in normalen Benutzerabläufen weggelassen.
