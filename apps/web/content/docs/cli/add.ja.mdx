---
title: 機能の追加
description: サポートされているアドオンを既存の Better Fullstack プロジェクトに追加します。
---

`add` コマンドは、既存の Better Fullstack プロジェクトを更新します。 `bts.jsonc` を読み取り、要求されたアドオンを検証し、サポートされているファイルを書き込み、プロジェクト構成を更新します。

```npm
npm create better-fullstack@latest add -- --addons mcp skills
```

## add とは

- 現在のプロジェクトまたは `--project-dir` から `bts.jsonc` を読み取ります。
- `--addons` が省略された場合、互換性のあるアドオンを求めるプロンプトが表示されます。
- すでにインストールされているアドオンをスキップします。
- 新しいアドオンが選択されたときに、アドオン テンプレートと依存関係の変更を書き込みます。
- 依存関係のインストールがスキップされる場合に、インストール コマンドを出力します。

<Callout kind="warn" title="Addon-first command">
`add` は現在、新しいアドオンが選択されていない場合に終了します。可能な場合は、`create` 中にデプロイメント ターゲットを選択してください。デプロイメントのみの `add --web-deploy ...` または `add --server-deploy ...` フローはまだ完了していません。
</Callout>

## フラグ

| フラグ | 価値観とか行動とか |
| --- | --- |
| `--addons` | `pwa` `tauri` `starlight` `biome` `lefthook` `husky` `ruler` `mcp` `skills` `turborepo` `fumadocs` `ultracite` `oxlint` `opentui` `wxt` `msw` `storybook` `swr` `tanstack-query` `tanstack-table` `tanstack-virtual` `tanstack-db` `tanstack-pacer` `backend-utils` `docker-compose` `none` |
| `--project-dir` | 生成されたプロジェクトへのパス。デフォルトは現在のディレクトリです。 |
| `--install` | ファイルを追加した後、依存関係をインストールします。 `add` のデフォルトは false です。 |
| `--package-manager` | `npm` `pnpm` `bun` `yarn` |
| `--web-deploy` | アドオン処理後の Web デプロイメント ターゲットを記録します。 |
| `--server-deploy` | アドオン処理後のサーバー展開ターゲットを記録します。 |

## アドオンリファレンス

| アドオン | 目的 |
| --- | --- |
| `mcp` | コーディング エージェント用に生成されたプロジェクト MCP サーバー構成。 |
| `skills` | 厳選されたコーディング エージェント スキル ファイル。 |
| `turborepo` | モノレポオーケストレーション。 |
| `docker-compose` | サポートされているスタックのローカル サービス オーケストレーション。 |
| `biome` `oxlint` `ultracite` `lefthook` `husky` `ruler` | コードの品質とフック。 |
| `starlight` `fumadocs` | スタックと互換性がある場合のドキュメント サイト。 |
| `tauri` `pwa` `wxt` | デスクトップ、プログレッシブ Web アプリ、ブラウザ拡張機能のサポート。 |
| `msw` `storybook` | テストおよびコンポーネント開発ユーティリティ。 |
| `swr` `tanstack-query` `tanstack-table` `tanstack-virtual` `tanstack-db` `tanstack-pacer` | データ取得および UI ユーティリティ。 |
| `backend-utils` | asyncHandler、ApiResponse、および Hono、Express、Fastify、Elysia、feTS、または NestJS サーバーのグローバル エラー ハンドラー。 |
| `opentui` | ターミナル UI のサポート。 |

## べき等性

`bts.jsonc` は、インストールされているアドオンの信頼できる情報源です。要求されたすべてのアドオンがすでに存在する場合、`add` は新しいアドオンが選択されていないことを報告し、無関係な変更を適用せずに終了します。

## 推奨されるワークフロー

1. 現在の作業をコミットまたは隠します。
2. 明示的なアドオン フラグを指定して `add` を実行します。
3. 生成されたファイルの変更を確認します。
4. コマンドがインストールをスキップした場合は、依存関係をインストールします。
5. プロジェクトの test、lint、または typecheck コマンドを実行します。
