Better Fullstack 的命令面很小。大多數使用者只需要 `create`、`add` 和 `history`；AI-agent workflows 使用 `mcp`。

## 命令

| 命令 | 用途 |
| --- | --- |
| `create` | 從 stack configuration 產生新專案。這是預設命令。 |
| `add` | 在現有 Better Fullstack 項目中新增支援的 addons 或 deployment settings。 |
| `history` | 顯示、匯出或清理本機 scaffold 歷史和可複現指令。 |
| `mcp` | 透過 stdio 為 AI 代碼代理程式啟動 Better Fullstack MCP server。 |

## 工具命令

| 命令 | 用途 |
| --- | --- |
| `docs` | 在瀏覽器中開啟公開的 Better Fullstack 文件網站。 |
| `builder` | 開啟互動式 Stack Builder。 |
| `sponsors` | 顯示 Better Fullstack sponsors。 |

開啟 docs：

## Maintainer 命令

`update-deps` 用於 maintainers 檢查和更新依賴版本。它有意不出現在普通用戶 workflows 中。
