description: プロンプト、デフォルト、または明示的なフラグを使用して、新しい Better Fullstack プロジェクトをスキャフォールディングします。
---

`create` コマンドは、型指定されたスタック構成から新しいプロジェクトを書き込みます。探索のためにインタラクティブに使用するか、毎回同じプロジェクト形状が必要な場合にフラグを渡します。

<PMTabs
  npm="npm create better-fullstack@latest [project-name] -- [flags]"
  pnpm="pnpm create better-fullstack@latest [project-name] [flags]"
  bun="bun create better-fullstack@latest [project-name] [flags]"
  yarn="yarn create better-fullstack@latest [project-name] [flags]"
/>

npm の場合、Better Fullstack フラグは `--` の後に続きます。生成された npm コマンドは `npx create-better-fullstack@latest` を使用する場合があります。同じパッケージ エントリポイントを非対話形式で実行します。

## コマンドモード

| モード | コマンド形状 | こんなときに使います |
| --- | --- | --- |
| ウィザード | `[launcher] my-app` | プロンプトとデフォルトが必要です。 |
| デフォルト | `[launcher] my-app [flags] --yes` | プロンプトのないデフォルトのスタックが必要です。 |
| スクリプト化された | `[launcher] my-app [flags]` | 反復可能な出力が必要です。 |
| ドライラン | `[launcher] my-app [flags] --dry-run` | 書き込む前にファイルを検査したい。 |

## 素早い行動

- 明示的なフラグは、そのフラグがカバーするカテゴリのみをスキップします。
- `--yes` はデフォルトを受け入れ、`--frontend`、`--backend`、`--database`、`--orm`、`--auth`、`--api` などのコア スタック フラグと競合します。
- `none` を使用して、`--ai-docs none` などのオプションのカテゴリを無効にします。
- 副作用が発生する前に生成されたファイルを検査する場合は、`--no-install` および `--no-git` を使用します。
- `--verify` は、開発サーバーを起動せずにスキャフォールディング後に生成されたチェックを実行します。

## 例

明示的なフラグを使用して TypeScript アプリを作成します。

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database postgres \
  --orm drizzle \
  --db-setup none \
  --api orpc \
  --auth better-auth \
  --css-framework tailwind \
  --ui-library shadcn-ui \
  --ai none \
  --payments none \
  --email none \
  --examples none \
  --addons none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

マルチエコシステム プロジェクトを作成します。

```npm
npm create better-fullstack@latest my-mixed-app -- \
  --part frontend:typescript:next \
  --part backend:go:gin \
  --part backend.orm:go:gorm \
  --part backend.auth:go:jwt \
  --part database:universal:postgres \
  --part mobile:react-native:native-bare \
  --addons turborepo docker-compose
```

## 互換性

- CLI は、生成前にスタックの互換性を検証します。
- TypeScript `backend self` はフルスタック フロントエンド用であり、`runtime none` とペアになります。
- tRPC は React 指向です。
- ワーカー ランタイムのサポートは、フロントエンドとバックエンドの組み合わせによって異なります。
- UI ライブラリ、デプロイメント ターゲット、プロバイダー、アドオンはスタックに依存します。
- 視覚的に確認するには、[互換性マトリックス](/docs/reference/compatibility/) または [Stack Builder](https://better-fullstack.dev/new) を使用してください。
