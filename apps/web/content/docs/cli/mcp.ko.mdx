`mcp` 명령은 stdio를 통해 Better Fullstack MCP 서버를 시작합니다. 에이전트는 이를 사용하여 스키마를 검사하고, 호환성을 확인하고, 생성된 파일을 미리 보고, 프로젝트를 생성하고, 지원되는 기능을 기존 프로젝트에 추가할 수 있습니다.

<PMTabs

  yarn="yarn dlx create-better-fullstack@latest mcp"
/>

서버는 stdio 전송을 사용합니다. 명령을 실행하고 표준 입력/출력을 통해 통신하도록 MCP 클라이언트를 구성합니다.

## 클라이언트 예시

npm을 사용하는 Claude Code:

```bash
claude mcp add --transport stdio better-fullstack -- npx -y create-better-fullstack@latest mcp
```

pnpm, Bun 또는 Yarn의 경우 `--` 뒤의 명령을 일치하는 실행 프로그램으로 바꿉니다.

| 패키지 관리자 | 발사통 |
| --- | --- |
| npm | `npx -y create-better-fullstack@latest mcp` |
| pnpm | `pnpm dlx create-better-fullstack@latest mcp` |
| Bun | `bunx create-better-fullstack@latest mcp` |
| 방사 | `yarn dlx create-better-fullstack@latest mcp` |

npm을 사용한 일반 MCP 클라이언트 구성:

```json
{
  "mcpServers": {
    "better-fullstack": {
      "command": "npx",
      "args": ["-y", "create-better-fullstack@latest", "mcp"]
    }
  }
}
```

다른 패키지 관리자에 해당하는 명령/인수:

| 패키지 관리자 | `command` | `args` |
| --- | --- | --- |
| pnpm | `pnpm` | `["dlx", "create-better-fullstack@latest", "mcp"]` |
| Bun | `bunx` | `["create-better-fullstack@latest", "mcp"]` |
| 방사 | `yarn` | `["dlx", "create-better-fullstack@latest", "mcp"]` |

## 도구 표면

| 도구 | 목적 |
| --- | --- |
| `bfs_get_guidance` | 워크플로 규칙 및 필드 의미를 반환합니다. |
| `bfs_get_schema` | 하나의 범주 또는 모든 범주에 대해 유효한 옵션을 반환합니다. |
| `bfs_check_compatibility` | 생성 전에 스택 선택을 검증합니다. |
| `bfs_plan_project` | 메모리에서 생성을 테스트 실행합니다. |
| `bfs_create_project` | 디스크에 새 프로젝트를 씁니다. |
| `bfs_plan_addition` | 기존 프로젝트에 대한 계획 기능 추가. |
| `bfs_add_feature` | 기존 프로젝트에 지원되는 기능을 추가합니다. |

프로젝트 계획, 생성 및 추가 응답에는 생성된 파일 또는 변형 요약과 함께 그래프 메타데이터(`graphSummary`, `effectiveStack` 및 `stackPartSpecs`)가 포함됩니다.

입력, 출력 및 안전한 워크플로는 [MCP 도구 참조](/docs/ai/mcp-tools/)를 확인하세요.

## 안전 참고 사항

- `bfs_plan_project`은(는) 파일을 쓰지 않습니다.
- `bfs_create_project`은 파일을 쓰지만 종속성을 설치하지는 않습니다.
- `bfs_add_feature`은 기존 Better Fullstack 프로젝트를 변경하고 종속성을 설치하지 않습니다.
- 에이전트는 프로젝트 생성 전에 `bfs_get_guidance` 및 `bfs_check_compatibility`을 호출해야 합니다.

## 관련 명령

- `docs`은 공개 문서 사이트를 엽니다.
- `builder`은 대화형 Stack Builder을 엽니다.
- `sponsors`에는 Better Fullstack 스폰서가 표시됩니다.

이러한 유틸리티 명령의 예는 [CLI 참조](/docs/cli/)를 참조하세요.
