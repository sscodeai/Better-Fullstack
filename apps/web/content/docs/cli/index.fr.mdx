Better Fullstack a une petite surface de commande. La plupart des utilisateurs n'ont besoin que de `create`, `add` et `history` ; Les workflows de l'agent AI utilisent `mcp`.

## Commandes

| Commande | But |
| --- | --- |
| `create` | Échafaudez un nouveau projet à partir d’une configuration de pile. Il s'agit de la commande par défaut. |
| `add` | Ajoutez des modules complémentaires ou des paramètres de déploiement pris en charge à un projet Better Fullstack existant. |
| `history` | Affichez, exportez ou effacez l'historique de l'échafaudage local et les commandes reproductibles. |
| `mcp` | Démarrez le serveur Better Fullstack MCP via stdio pour les agents de codage AI. |

## Commandes utilitaires

| Commande | But |
| --- | --- |
| `docs` | Ouvrez le site de documentation public Better Fullstack dans votre navigateur. |
| `builder` | Ouvrez le Stack Builder interactif. |
| `sponsors` | Afficher Better Fullstack sponsors. |

## Commandes du responsable

`update-deps` est destiné aux responsables qui vérifient et mettent à jour les versions de dépendances. Il est intentionnellement omis des flux de travail utilisateur normaux.
