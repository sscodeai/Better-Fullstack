La commande `mcp` démarre le serveur Better Fullstack MCP via stdio. Les agents peuvent l'utiliser pour inspecter les schémas, valider la compatibilité, prévisualiser les fichiers générés, créer des projets et ajouter des fonctionnalités prises en charge aux projets existants.

<PMTabs
  npm="npx -y create-better-fullstack@latest mcp"
  pnpm="pnpm dlx create-better-fullstack@latest mcp"
  bun="bunx create-better-fullstack@latest mcp"
  yarn="yarn dlx create-better-fullstack@latest mcp"
/>

Le serveur utilise le transport stdio. Configurez votre client MCP pour lancer la commande et communiquer via une entrée/sortie standard.

## Exemples de clients

Claude Code avec npm :

```bash
claude mcp add --transport stdio better-fullstack -- npx -y create-better-fullstack@latest mcp
```

Pour pnpm, Bun ou Yarn, remplacez la commande après `--` par le lanceur correspondant :

| Gestionnaire de paquets | Lanceur |
| --- | --- |
| npm | `npx -y create-better-fullstack@latest mcp` |
| pnpm | `pnpm dlx create-better-fullstack@latest mcp` |
| Bun | `bunx create-better-fullstack@latest mcp` |
| Fil | `yarn dlx create-better-fullstack@latest mcp` |

Configuration client générique MCP avec npm :

```json
{
  "mcpServers": {
    "better-fullstack": {
      "command": "npx",
      "args": ["-y", "create-better-fullstack@latest", "mcp"]
    }
  }
}
```

Commande/arguments équivalents pour d'autres gestionnaires de packages :

| Gestionnaire de paquets | `command` | `args` |
| --- | --- | --- |
| pnpm | `pnpm` | `["dlx", "create-better-fullstack@latest", "mcp"]` |
| Bun | `bunx` | `["create-better-fullstack@latest", "mcp"]` |
| Fil | `yarn` | `["dlx", "create-better-fullstack@latest", "mcp"]` |

## Surface de l'outil

| Outil | But |
| --- | --- |
| `bfs_get_guidance` | Renvoie les règles de workflow et la sémantique des champs. |
| `bfs_get_schema` | Renvoie les options valides pour une catégorie ou toutes les catégories. |
| `bfs_check_compatibility` | Valide les sélections de pile avant la génération. |
| `bfs_plan_project` | Génération d’essais à sec en mémoire. |
| `bfs_create_project` | Écrit un nouveau projet sur le disque. |
| `bfs_plan_addition` | Les plans comportent des ajouts pour un projet existant. |
| `bfs_add_feature` | Ajoute des fonctionnalités prises en charge à un projet existant. |

Les réponses de planification, de création et d'ajout de projet incluent les métadonnées du graphique (`graphSummary`, `effectiveStack` et `stackPartSpecs`) ainsi que le fichier généré ou le résumé de mutation.

Consultez la [référence des outils MCP](/docs/ai/mcp-tools/) pour connaître les entrées, les sorties et les flux de travail sécurisés.

## Notes de sécurité

- `bfs_plan_project` n'écrit pas de fichiers.
- `bfs_create_project` écrit des fichiers mais n'installe pas de dépendances.
- `bfs_add_feature` mute un projet Better Fullstack existant et n'installe pas de dépendances.
- Les agents doivent appeler `bfs_get_guidance` et `bfs_check_compatibility` avant la création du projet.

## Commandes associées

- `docs` ouvre le site de documentation publique.
- `builder` ouvre le Stack Builder interactif.
- `sponsors` affiche Better Fullstack sponsors.

Consultez la [référence CLI](/docs/cli/) pour des exemples de ces commandes utilitaires.
