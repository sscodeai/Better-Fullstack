`mcp` 指令透過 stdio 啟動 Better Fullstack MCP server。代理可以用它檢查 schemas、驗證相容性、預覽生成檔案、建立項目，並向現有項目添加支援的功能。

<PMTabs

  bun="bunx create-better-fullstack@latest mcp"

/>

伺服器使用 stdio transport。設定你的 MCP client 啟動該指令，並透過 standard input/output 通訊。

## Client 範例

使用 npm 的 Claude Code：

```bash
claude mcp add --transport stdio better-fullstack -- npx -y create-better-fullstack@latest mcp
```

如果你偏好 pnpm、Bun 或 Yarn，把 `--` 之後的指令替換成符合的 launcher：

| Package manager | Launcher |
| --- | --- |
| npm | `npx -y create-better-fullstack@latest mcp` |
| pnpm | `pnpm dlx create-better-fullstack@latest mcp` |
| Bun | `bunx create-better-fullstack@latest mcp` |
| Yarn | `yarn dlx create-better-fullstack@latest mcp` |

使用 npm 的通用 MCP client config：

```json
{
  "mcpServers": {
    "better-fullstack": {
      "command": "npx",
      "args": ["-y", "create-better-fullstack@latest", "mcp"]
    }
  }
}
```

其他 package managers 的等價 command/args：

| Package manager | `command` | `args` |
| --- | --- | --- |
| pnpm | `pnpm` | `["dlx", "create-better-fullstack@latest", "mcp"]` |
| Bun | `bunx` | `["create-better-fullstack@latest", "mcp"]` |
| Yarn | `yarn` | `["dlx", "create-better-fullstack@latest", "mcp"]` |

## 工具表面

| 工具 | 用途 |
| --- | --- |
| `bfs_get_guidance` | 傳回 workflow 規則和字段語義。 |
| `bfs_get_schema` | 傳回一個 category 或所有 categories 的有效選項。 |
| `bfs_check_compatibility` | 在生成前驗證 stack selections。 |
| `bfs_plan_project` | 在記憶體中 dry-run 項目產生。 |
| `bfs_create_project` | 將新項目寫入磁碟。 |
| `bfs_plan_addition` | 為現有專案規劃 feature additions。 |
| `bfs_add_feature` | 為現有項目新增支援的功能。 |

專案規劃、建立和新增回應會在 generated file 或 mutation summary 之外，包含 graph metadata（`graphSummary`、`effectiveStack` 和 `stackPartSpecs`）。

查看 [MCP tools reference](/docs/ai/mcp-tools/) 了解 inputs、outputs 和安全 workflows。

## 安全說明

- `bfs_plan_project` 不寫入檔案。
- `bfs_create_project` 寫入文件，但不安裝依賴。
- `bfs_add_feature` 會修改現有 Better Fullstack 項目，且不安裝相依性。
- Agents 應該在建立專案之前呼叫 `bfs_get_guidance` 和 `bfs_check_compatibility`。
