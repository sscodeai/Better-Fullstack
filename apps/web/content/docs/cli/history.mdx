---
title: History Command
description: View, clear, and export reproducible commands from previous Better Fullstack scaffolds.
---

The `history` command shows recent scaffolded projects and the reproducible commands that created them. Use it when you want to recreate a stack, compare generated output, or share a known-good command.

<PMTabs
  npm="npx -y create-better-fullstack@latest history"
  pnpm="pnpm dlx create-better-fullstack@latest history"
  bun="bunx create-better-fullstack@latest history"
  yarn="yarn dlx create-better-fullstack@latest history"
/>

## Flags

| Flag | Default | Description |
| --- | --- | --- |
| `--limit` | `10` | Number of history entries to show. |
| `--clear` | `false` | Clear all saved scaffold history. |
| `--json` | `false` | Print history as JSON for scripts. |

## Examples

Show the five most recent entries:

<PMTabs
  npm="npx -y create-better-fullstack@latest history --limit 5"
  pnpm="pnpm dlx create-better-fullstack@latest history --limit 5"
  bun="bunx create-better-fullstack@latest history --limit 5"
  yarn="yarn dlx create-better-fullstack@latest history --limit 5"
/>

Export history as JSON:

<PMTabs
  npm="npx -y create-better-fullstack@latest history --json"
  pnpm="pnpm dlx create-better-fullstack@latest history --json"
  bun="bunx create-better-fullstack@latest history --json"
  yarn="yarn dlx create-better-fullstack@latest history --json"
/>

Clear local history:

<PMTabs
  npm="npx -y create-better-fullstack@latest history --clear"
  pnpm="pnpm dlx create-better-fullstack@latest history --clear"
  bun="bunx create-better-fullstack@latest history --clear"
  yarn="yarn dlx create-better-fullstack@latest history --clear"
/>

## Reproducible workflow

1. Run `history`.
2. Copy the reproducible command for the stack you want.
3. Re-run it with a new project name or adjusted flags.

History is stored locally under your user directory and does not affect generated project files.
