---
title: Create 命令
description: 使用 prompts、defaults 或显式 flags scaffold 一个新的 Better Fullstack 项目。
---

`create` 命令会从类型化 stack configuration 写入一个新项目。探索时可以交互式使用；当你每次都需要相同项目形状时，传入 flags。

<PMTabs
  npm="npm create better-fullstack@latest [project-name] -- [flags]"
  pnpm="pnpm create better-fullstack@latest [project-name] [flags]"
  bun="bun create better-fullstack@latest [project-name] [flags]"
  yarn="yarn create better-fullstack@latest [project-name] [flags]"
/>

使用 npm 时，Better Fullstack flags 放在 `--` 之后。生成的 npm 命令可能使用 `npx create-better-fullstack@latest`；这会以非交互形式运行同一个 package entrypoint。

## 命令模式

| 模式 | 命令形状 | 适合场景 |
| --- | --- | --- |
| Wizard | `[launcher] my-app` | 你想要 prompts 和 defaults。 |
| Defaults | `[launcher] my-app [flags] --yes` | 你想要默认 stack，不想回答 prompts。 |
| Scripted | `[launcher] my-app [flags]` | 你想要可重复 output。 |
| Dry run | `[launcher] my-app [flags] --dry-run` | 你想在写入前检查文件。 |

## Prompt 行为

- 显式 flags 只跳过它们覆盖的 categories。
- `--yes` 接受 defaults，并与 `--frontend`、`--backend`、`--database`、`--orm`、`--auth`、`--api` 等核心 stack flags 冲突。
- 使用 `none` 关闭可选 categories，包括 `--ai-docs none`。
- 想在副作用前检查生成文件时使用 `--no-install` 和 `--no-git`。
- `--verify` 会在 scaffold 后运行生成的 checks，不启动 dev servers。

## 示例

用显式 flags 创建 TypeScript app：

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database postgres \
  --orm drizzle \
  --db-setup none \
  --api orpc \
  --auth better-auth \
  --css-framework tailwind \
  --ui-library shadcn-ui \
  --ai none \
  --payments none \
  --email none \
  --examples none \
  --addons none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

创建 multi-ecosystem 项目：

```npm
npm create better-fullstack@latest my-mixed-app -- \
  --part frontend:typescript:next \
  --part backend:go:gin \
  --part backend.orm:go:gorm \
  --part backend.auth:go:jwt \
  --part database:universal:postgres \
  --part mobile:react-native:native-bare \
  --addons turborepo docker-compose
```

## 兼容性

- CLI 会在生成前验证 stack 兼容性。
- TypeScript `backend self` 用于 fullstack frontends，并与 `runtime none` 搭配。
- tRPC 面向 React。
- Workers runtime 支持取决于 frontend/backend 组合。
- UI libraries、deployment targets、providers 和 addons 都受 stack 约束。
- 想可视化检查，使用 [Compatibility Matrix](/docs/reference/compatibility/) 或 [Stack Builder](https://better-fullstack.dev/new)。
