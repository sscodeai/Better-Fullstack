Better Fullstack s'exécute via le package `create-better-fullstack`. Vous n'avez pas besoin de l'installer globalement ; exécutez-le avec votre gestionnaire de packages, répondez aux invites et le CLI écrit le projet sélectionné.

<Callout kind="tip" title="Online preview">
Vous préférez choisir visuellement ? Ouvrez le [Stack Builder](https://better-fullstack.dev/new), choisissez une pile compatible et copiez la commande générée.
</Callout>

## Conditions préalables

- Node.js 20 ou plus récent pour le CLI.
- Une borne.
- Git uniquement si vous souhaitez que CLI initialise un référentiel.
- Des outils spécifiques à l'écosystème uniquement pour le projet que vous générez.

| Écosystème | Outillage supplémentaire |
| --- | --- |
| TypeScript | npm, pnpm, Yarn ou Bun |
| React Native | Outils iOS/Android compatibles Expo lors de l'exécution sur un simulateur ou un appareil |
| Rouiller | Chaîne d'outils Rust et Cargo |
| Python | Python et `uv` |
| Aller | Aller à la chaîne d'outils |
| Java | Java21 |
| Élixir | Elixir, Erlang/OTP et PostgreSQL pour les flux Phoenix basés sur une base de données |

## Exécutez le CLI

<PMTabs
  npm="npm create better-fullstack@latest"
  pnpm="pnpm create better-fullstack@latest"
  bun="bun create better-fullstack@latest"
  yarn="yarn create better-fullstack@latest"
/>

Ajoutez un nom de projet pour démarrer l'assistant :

```npm
npm create better-fullstack@latest my-app
```

L'assistant demande l'écosystème, les options de pile, les modules complémentaires facultatifs, Git et l'installation des dépendances.

## Utiliser des indicateurs explicites

Les indicateurs explicites ignorent les invites qu'ils couvrent. Il s'agit du meilleur formulaire pour les scripts, les documents, les commandes générées et les rapports de bogues reproductibles.

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --api orpc \
  --auth better-auth \
  --db-setup none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

Avec `npm create`, les drapeaux Better Fullstack suivent `--`. Les onglets pnpm rendus, Bun et Yarn suppriment ce séparateur npm uniquement.

Utilisez `--ai-docs none` lorsque vous ne souhaitez pas générer de fichiers d'instructions AI. Utilisez `--no-install` lorsque vous souhaitez inspecter les fichiers avant l'installation des dépendances.

## Modèles

Les modèles sont un raccourci pour les piles courantes :

| Modèle | Empiler |
| --- | --- |
| `t3` | Next.js, tRPC, Prisma, PostgreSQL, Better Auth |
| `mern` | MongoDB, Express, React Router, Node.js |
| `pern` | PostgreSQL, Express, TanStack Router, Node.js |
| `uniwind` | Expo + Uniwind, mobile uniquement |
| `none` | Pas de modèle abrégé |

```npm
npm create better-fullstack@latest my-app -- --template t3
```

## Valider avant d'écrire des fichiers

Utilisez `--dry-run` pour prévisualiser les fichiers générés sans écrire de projet :

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

Utilisez `--verify` lorsque vous souhaitez que CLI exécute les vérifications générées après l'échafaudage. Il ne démarre pas les serveurs de développement.
