Better Fullstack läuft durch das Paket `create-better-fullstack`. Sie müssen es nicht global installieren; Führen Sie es mit Ihrem Paketmanager aus, beantworten Sie die Eingabeaufforderungen, und CLI schreibt das ausgewählte Projekt.

<Callout kind="tip" title="Online preview">
Wählen Sie lieber optisch? Öffnen Sie [Stack Builder](https://better-fullstack.dev/new), wählen Sie einen kompatiblen Stack aus und kopieren Sie den generierten Befehl.
</Callout>

## Voraussetzungen

- Node.js 20 oder neuer für den CLI.
- Ein Terminal.
- Git nur, wenn Sie möchten, dass CLI ein Repository initialisiert.
- Ökosystemspezifische Tools nur für das von Ihnen generierte Projekt.

| Ökosystem | Zusätzliches Werkzeug |
| --- | --- |
| TypeScript | npm, pnpm, Garn oder Bun |
| React Native | Expo-kompatible iOS-/Android-Tools bei Ausführung auf einem Simulator oder Gerät |
| Rost | Rust-Toolchain und Cargo |
| Python | Python und `uv` |
| Gehen | Gehen Sie zur Toolchain |
| Java | Java 21 |
| Elixier | Elixir, Erlang/OTP und PostgreSQL für datenbankgestützte Phoenix-Flows |

## Führen Sie CLI aus.

<PMTabs
  npm="npm create better-fullstack@latest"
  pnpm="pnpm create better-fullstack@latest"
  bun="bun create better-fullstack@latest"
  yarn="yarn create better-fullstack@latest"
/>

Fügen Sie einen Projektnamen hinzu, um den Assistenten zu starten:

```npm
npm create better-fullstack@latest my-app
```

Der Assistent fragt nach dem Ökosystem, Stack-Optionen, optionalen Add-ons, Git und der Abhängigkeitsinstallation.

## Verwenden Sie explizite Flags

Explizite Flags überspringen die Eingabeaufforderungen, die sie abdecken. Dies ist die beste Form für Skripte, Dokumente, generierte Befehle und reproduzierbare Fehlerberichte.

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --api orpc \
  --auth better-auth \
  --db-setup none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

Mit `npm create` folgen Better Fullstack Flags nach `--`. Die gerenderten Registerkarten „pnpm“, „Bun“ und „Garn“ entfernen dieses reine NPM-Trennzeichen.

Verwenden Sie `--ai-docs none`, wenn Sie keine generierten AI-Anweisungsdateien wünschen. Verwenden Sie `--no-install`, wenn Sie Dateien vor der Abhängigkeitsinstallation überprüfen möchten.

## Vorlagen

Vorlagen sind eine Abkürzung für gängige Stapel:

| Vorlage | Stapel |
| --- | --- |
| `t3` | Next.js, tRPC, Prisma, PostgreSQL, Better Auth |
| `mern` | MongoDB, Express, React Router, Node.js |
| `pern` | PostgreSQL, Express, TanStack Router, Node.js |
| `uniwind` | Expo + Uniwind, nur für Mobilgeräte |
| `none` | Keine Kurzschriftvorlage |

```npm
npm create better-fullstack@latest my-app -- --template t3
```

## Vor dem Schreiben von Dateien validieren

Verwenden Sie `--dry-run`, um eine Vorschau der generierten Dateien anzuzeigen, ohne ein Projekt zu schreiben:

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

Verwenden Sie `--verify`, wenn CLI nach dem Gerüstbau generierte Prüfungen ausführen soll. Es startet keine Entwicklungsserver.
