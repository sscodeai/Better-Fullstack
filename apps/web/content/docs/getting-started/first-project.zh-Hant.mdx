title: 第一個項目
description: Scaffold、檢查、安裝並執行一個 Better Fullstack 專案。
---

這個 walkthrough 會建立一個項目，預覽 CLI 可以產生什麼，並說明 scaffold 完成後應該檢查哪裡。

## 1. 建立項目

運行嚮導：

```npm
npm create better-fullstack@latest my-app
```

接受 TypeScript starter 的 defaults，或調整 prompts 來符合你想要的 stack。

## 2. 不寫入文件先預覽

想先檢查產生的檔案樹時使用 `--dry-run`：

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

在嘗試新生態、新 deployment target 或 multi-ecosystem stack 之前，dry run 很有用。

## 3. 從命令生成

明確 flags 讓 scaffold 可重複。這個範例建立一個 Next.js self-backend 項目，並跳過 install/Git，方便​​你先檢視文件：

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --runtime none \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --db-setup none \
  --ai none \
  --payments none \
  --examples none \
  --addons none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

<Callout kind="info" title="Prompt 行为">
顯式 flags 會跳過它們覆蓋的 categories。 `--yes` 接受 defaults，並與 `--frontend`、`--backend`、`--database` 等核心 stack flags 衝突。對想關閉的可選 categories 使用 `none`。
</Callout>

## 4. 安裝依賴

如果你使用了 `--no-install`，在專案內手動安裝：

```bash
cd my-app
```

```npm
npm install
```

語言原生生態使用自己的工具：

```bash
cargo build
uv sync
go mod tidy
./mvnw test
./gradlew test
```

## 5. 檢查項目

生成項目包含：

- 記錄所選 stack 和 schema reference 的 `bts.jsonc`。
- 面向所選生態的 framework、package 和 config 檔案。
- CLI 列印的可複現指令。
- 可選 AI docs，例如 `CLAUDE.md`、`Agents.md` 或 `.cursorrules`。
- 當所選模板提供時，會包含 README 和 setup notes。

## 6. 後續新增支援的 features

在生成專案內，使用 `add` 新增支援的 addons：

```npm
npm create better-fullstack@latest add -- --addons mcp skills
```

`add` 指令會讀取 `bts.jsonc`，跳過已經存在的 addons，並在依賴未自動安裝時列印安裝指引。
