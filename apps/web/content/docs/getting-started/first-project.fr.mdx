title: Premier projet
description: Échafaudez, inspectez, installez et exécutez un projet Better Fullstack.
---

Cette procédure pas à pas crée un projet, prévisualise ce que CLI peut générer et montre où s'occuper des finitions de l'échafaudage.

## 1. Créez un projet

Exécutez l'assistant :

```npm
npm create better-fullstack@latest my-app
```

Acceptez les valeurs par défaut pour un démarreur TypeScript ou modifiez les invites pour qu'elles correspondent à la pile souhaitée.

## 2. Aperçu sans écrire

Utilisez `--dry-run` lorsque vous souhaitez d'abord inspecter l'arborescence des fichiers générés :

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

L'essai à sec est utile avant d'essayer un nouvel écosystème, une cible de déploiement ou une pile multi-écosystèmes.

## 3. Échafaudage à partir d'une commande

Les indicateurs explicites rendent l'échafaudage répétable. Cet exemple crée un projet auto-backend Next.js et ignore l'installation/Git afin que vous puissiez d'abord examiner les fichiers :

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --runtime none \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --db-setup none \
  --ai none \
  --payments none \
  --examples none \
  --addons none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

<Callout kind="info" title="Prompt behavior">
Les indicateurs explicites ignorent les catégories qu'ils couvrent. `--yes` accepte les valeurs par défaut et les conflits avec les indicateurs de pile principaux tels que `--frontend`, `--backend` et `--database`. Utilisez `none` pour les catégories facultatives que vous souhaitez désactiver.
</Callout>

## 4. Installer les dépendances

Si vous avez utilisé `--no-install`, installez manuellement dans le projet :

```bash
cd my-app
```

```npm
npm install
```

Les écosystèmes linguistiques natifs utilisent leurs propres outils :

```bash
cargo build
uv sync
go mod tidy
./mvnw test
./gradlew test
```

## 5. Inspectez le projet

Les projets générés incluent :

- `bts.jsonc` avec la pile sélectionnée et la référence de schéma.
- Fichiers de framework, de package et de configuration pour l'écosystème sélectionné.
- Une commande reproductible imprimée par le CLI.
- Documents AI facultatifs tels que `CLAUDE.md`, `Agents.md` ou `.cursorrules`.
- README et notes de configuration lorsque les modèles sélectionnés les fournissent.

## 6. Ajoutez des fonctionnalités prises en charge plus tard

Dans un projet généré, utilisez `add` pour les modules complémentaires pris en charge :

```npm
npm create better-fullstack@latest add -- --addons mcp skills
```

La commande `add` lit `bts.jsonc`, ignore les modules complémentaires déjà présents et imprime les instructions d'installation lorsque les dépendances ne sont pas installées automatiquement.
