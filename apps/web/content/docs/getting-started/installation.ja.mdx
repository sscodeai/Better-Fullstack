---
title: インストール
description: 環境をセットアップし、最初の Better Fullstack プロジェクトをスキャフォールディングします。
---

Better Fullstack は `create-better-fullstack` パッケージを通じて実行されます。グローバルにインストールする必要はありません。パッケージ マネージャーで実行し、プロンプトに答えると、CLI が選択したプロジェクトを書き込みます。

<Callout kind="tip" title="Online preview">
視覚的に選択したいですか? [Stack Builder](https://better-fullstack.dev/new) を開き、互換性のあるスタックを選択し、生成されたコマンドをコピーします。
</Callout>

## 前提条件

- Node.js CLI の場合は 20 以降。
- 端末。
- CLI でリポジトリを初期化する場合にのみ Git を使用します。
- 生成したプロジェクト専用のエコシステム固有のツール。

| 生態系 | 追加の工具 |
| --- | --- |
| TypeScript | npm、pnpm、Yarn、または Bun |
| React Native | シミュレータまたはデバイス上で実行する場合の Expo 互換の iOS/Android ツール |
| さび | Rust ツールチェーンとカーゴ |
| パイソン | Python と `uv` |
| 行く | Go ツールチェーン |
| ジャワ | ジャワ21 |
| エリクサー | Elixir、Erlang/OTP、およびデータベースベースの Phoenix フロー用の PostgreSQL |

## CLI を実行します。

<PMTabs
  npm="npm create better-fullstack@latest"
  pnpm="pnpm create better-fullstack@latest"
  bun="bun create better-fullstack@latest"
  yarn="yarn create better-fullstack@latest"
/>

プロジェクト名を追加してウィザードを開始します。

```npm
npm create better-fullstack@latest my-app
```

ウィザードは、エコシステム、スタック オプション、オプションのアドオン、Git、依存関係のインストールを要求します。

## 明示的なフラグを使用する

明示的なフラグは、それが対象とするプロンプトをスキップします。これは、スクリプト、ドキュメント、生成されたコマンド、および再現可能なバグ レポートに最適な形式です。

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --api orpc \
  --auth better-auth \
  --db-setup none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

`npm create` では、Better Fullstack フラグが `--` の後に続きます。レンダリングされた pnpm、Bun、および Yarn タブでは、npm のみの区切り文字が削除されます。

AI 命令ファイルを生成したくない場合は、`--ai-docs none` を使用します。依存関係のインストール前にファイルを検査する場合は、`--no-install` を使用します。

## テンプレート

テンプレートは一般的なスタックの短縮形です。

| テンプレート | スタック |
| --- | --- |
| `t3` | Next.js、tRPC、Prisma、PostgreSQL、Better Auth |
| `mern` | MongoDB、Express、React Router、Node.js |
| `pern` | PostgreSQL、Express、TanStack Router、Node.js |
| `uniwind` | Expo + Uniwind、モバイルのみ |
| `none` | 短縮テンプレートはありません |

```npm
npm create better-fullstack@latest my-app -- --template t3
```

## ファイルを書き込む前に検証する

`--dry-run` を使用して、プロジェクトを作成せずに生成されたファイルをプレビューします。

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

スキャフォールディング後に CLI で生成されたチェックを実行する場合は、`--verify` を使用します。開発サーバーは起動しません。
