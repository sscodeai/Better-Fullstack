UI 및 경험 선택은 생성된 프런트엔드 코드, 디자인 기본 요소 및 클라이언트 측 애플리케이션 패턴을 형성합니다.

## 스타일링 및 UI 라이브러리

CSS 선택에는 Tailwind, SCSS, Less, PostCSS 전용 또는 없음이 포함됩니다. UI 라이브러리에는 shadcn/ui, shadcn-svelte, 데이지 UI, Radix UI, 헤드리스 UI, Park UI, Chakra UI, NextUI, Mantine, MUI, Ant Design, Base UI, Ark UI, React Aria가 포함되며 없음.

shadcn/ui에는 기본 기본 요소, 시각적 스타일, 아이콘, 색상 테마, 중립 기본 색상, 글꼴 및 반경에 대한 추가 사전 설정이 있습니다.

## 양식, 유효성 검사 및 상태

프런트엔드 애플리케이션 선택 사항은 다음과 같습니다.

- 상태: Zustand, Jotai, Nanostores, Redux Toolkit, MobX, XState, Valtio, TanStack Store, Legend State 또는 없음.
- 양식: TanStack Form, React Hook Form, Formik, Final Form, Conform, Modular Forms 또는 없음.
- 검증: Zod, Valibot, ArkType, TypeBox, Typia, Runtypes, Effect Schema 또는 없음.

## 체험 기능

애니메이션, i18n 및 프런트엔드 인접 앱 플랫폼 애드온은 정확한 플래그가 다른 옵션 테이블에 표시되는 경우에도 개념적으로 여기에 있습니다.

React Native에는 Tamagui, Gluestack UI, Uniwind, Unistyles 또는 없음 등 자체 모바일 UI 선택 사항이 있습니다.
