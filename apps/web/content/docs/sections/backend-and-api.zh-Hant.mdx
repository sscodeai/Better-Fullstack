後端選擇決定應用邏輯在哪裡運行，以及客戶端如何與它通訊。

## 後端框架

TypeScript 後端選項包括 Hono、Express、Fastify、Elysia、NestJS、AdonisJS、Nitro、Encore、Convex、feTS、`self` 和 `none`。

語言生態系會暴露自己的後端介面：

| 生態系統 | 後端選擇 |
| --- | --- |
| Rust | Axum、Actix Web、Rocket 或 none。 |
| Python | FastAPI、Django、Flask、Litestar 或 none。 |
| Go | Gin、Echo、Fiber、Chi 或 none。 |
| Java | Spring Boot、Quarkus 或 none。 |
| Elixir | Phoenix、Phoenix LiveView 或 none。 |

## Runtime

TypeScript 伺服器可以面對 Bun、Node.js、Workers，或不使用單獨 runtime。非 TypeScript 生態系使用它們的原生工具鏈：Cargo、uv、Go modules、Maven/Gradle 或 Mix。

## API 層

API 選取範圍包括 TypeScript RPC/GraphQL 選項、Rust Tonic 或 async-graphql、Python Django API/GraphQL 選項、Go __BFS_ ElPH_4，以及FS_PH3__BFS_FS_BFS_xBFS。

## Multi-ecosystem ownership

在 multi 模式中，API、ORM 和 auth 能力會附加到被選取的後端。例如，`backend.api:rust:tonic` 屬於 Rust 後端，而 `backend.orm:go:gorm` 屬於 Go 後端。
