バックエンドの選択により、アプリケーション ロジックが実行される場所と、クライアントがアプリケーション ロジックと通信する方法が決まります。

## バックエンドフレームワーク

TypeScript バックエンド オプションには、Hono、Express、Fastify、Elysia、NestJS、AdonisJS、Nitro、Encore、Convex、feTS、`self`、`none` が含まれます。

言語エコシステムは、独自のバックエンド サーフェスを公開します。

| 生態系 | バックエンドの選択肢 |
| --- | --- |
| さび | Axum、Actix Web、Rocket、またはなし。 |
| パイソン | FastAPI、Django、Flask、Litestar、またはなし。 |
| 行く | ジン、エコー、ファイバー、チー、またはなし。 |
| ジャワ | Spring Boot、Quarkus、またはなし。 |
| エリクサー | Phoenix、Phoenix LiveView、またはなし。 |

## ランタイム

TypeScript サーバーは、Bun、Node.js、ワーカーをターゲットにすることも、個別のランタイムをターゲットにすることもできません。 TypeScript 以外のエコシステムは、ネイティブ ツールチェーン (Cargo、uv、Go モジュール、Maven/Gradle、または Mix) を使用します。

## API レイヤー

API の選択肢には、TypeScript RPC/GraphQL オプション、Rust Tonic または async-graphql、Python Django API/GraphQL オプション、Go gRPC、および Elixir REST または Absinthe が含まれます。

## 複数の生態系の所有権

マルチ モードでは、API、ORM、および認証機能が選択されたバックエンドに接続されます。たとえば、`backend.api:rust:tonic` は Rust バックエンドに属し、`backend.orm:go:gorm` は Go バックエンドに属します。
