Datenauswahl entscheidet darüber, wo generierte Apps Datensätze speichern, Arbeit zwischenspeichern, Inhalte durchsuchen und Dateien speichern.

## Datenbank und ORM

Die Kategorie der freigegebenen Datenbanken umfasst SQLite, PostgreSQL, MySQL, MongoDB, EdgeDB, Redis und keine. ORMs sind in der Praxis ökosystemspezifisch:

| Ökosystem | ORM oder Datenschicht |
| --- | --- |
| TypeScript | Drizzle, Prisma, Mongoose, TypeORM, Kysely, MikroORM, Sequelize oder keine. |
| Rost | SeaORM, SQLx, Diesel oder keiner. |
| Python | SQLAlchemy, SQLModel, Tortoise ORM oder keines. |
| Gehen | GORM, sqlc, Ent oder keine. |
| Java | Spring Data JPA oder keine. |
| Elixier | Ecto, Ecto SQL oder keines. |

## Anbieter-Setup

`--db-setup` kann Hosting-/Provider-Setup für Turso, Neon, Prisma Postgres, PlanetScale, MongoDB Atlas, Supabase, Upstash, Cloudflare D1, Docker oder keines hinzufügen.

## Dienste im Speicherstil

Zu den an Daten angrenzenden Abschnitten gehören:

- Caching: Upstash Redis in TypeScript sowie ökosystemspezifische Cache-Optionen, sofern unterstützt.
- Suche: Meilisearch, Typesense, Elasticsearch, Algolia und ökosystemspezifische Suchpfade.
- Datei-Upload: UploadThing, FilePond, Uppy oder keine.
- Dateispeicher: S3, R2, Cloudinary oder keine.
