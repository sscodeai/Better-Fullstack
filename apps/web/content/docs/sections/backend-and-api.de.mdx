Die Auswahl des Backends entscheidet darüber, wo die Anwendungslogik ausgeführt wird und wie Clients mit ihr kommunizieren.

## Backend-Framework

Zu den Backend-Optionen von TypeScript gehören Hono, Express, Fastify, Elysia, NestJS, AdonisJS, Nitro, Encore, Convex, feTS, `self` und `none`.

Sprachökosysteme stellen ihre eigenen Backend-Oberflächen zur Verfügung:

| Ökosystem | Backend-Auswahl |
| --- | --- |
| Rost | Axum, Actix Web, Rocket oder keines. |
| Python | FastAPI, Django, Flask, Litestar oder keine. |
| Gehen | Gin, Echo, Fiber, Chi oder keiner. |
| Java | Spring Boot, Quarkus oder keiner. |
| Elixier | Phoenix, Phoenix LiveView oder keines. |

## Laufzeit

TypeScript-Server können auf Bun, Node.js, Worker oder keine separate Laufzeit abzielen. Nicht-TypeScript-Ökosysteme verwenden ihre nativen Toolchains: Cargo, UV, Go-Module, Maven/Gradle oder Mix.

## API Ebene

Zu den API-Optionen gehören TypeScript RPC/GraphQL-Optionen, Rust Tonic oder async-graphql, Python Django, API/GraphQL-Optionen, Go gRPC und Elixir REST oder Absinthe.

## Besitz mehrerer Ökosysteme

Im Multimodus werden API-, ORM- und Authentifizierungsfunktionen an das ausgewählte Backend angehängt. Beispielsweise gehört `backend.api:rust:tonic` zu einem Rust-Backend, während `backend.orm:go:gorm` zu einem Go-Backend gehört.
