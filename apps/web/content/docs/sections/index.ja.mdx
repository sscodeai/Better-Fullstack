Better Fullstack には多くの選択肢があるため、ドキュメントは影響を受ける生成された製品の部分ごとに編成されています。これらのページでは、各セクションの意味、オプションをいつ選択するか、正確な CLI 値を見つける場所について説明します。

完全なオプション テーブルについては、[参照オプション](/docs/reference/options/) を使用してください。言語固有のサーフェスの場合は、[エコシステム](/docs/ecosystems/) を使用します。

| [バックエンドと API](/docs/sections/backend-and-api/) | サーバー フレームワーク、ランタイム、API レイヤー、およびマルチエコシステム バックエンドの選択。 |
| [データとストレージ](/docs/sections/data-and-storage/) | データベース、ORM、ホスティング セットアップ、キャッシュ、検索、ファイル アップロード、およびファイル ストレージ。 |

## ソロモードとマルチモード

ソロ モードでは、1 つのエコシステムが表示されるカテゴリを駆動します。 [マルチエコシステムモード](/docs/ecosystems/multi-ecosystem/) では、`frontend:typescript:next`、`backend:go:gin`、`backend.orm:go:gorm`、`database:universal:postgres` など、同じ概念がスタック部分になります。
