Dieser Abschnitt steuert die benutzerorientierten App-Oberflächen, die Better Fullstack generiert.

## Web-Frontend

TypeScript Web-Frontends werden mit `--frontend` ausgewählt. Die Kategorie unterstützt React, Vinext, Vue/Nuxt, SvelteKit, Solid, Astro, Qwik, Angular, Redwood, Fresh und `none`.

Verwenden Sie `none` für reine Backend-Dienste. Verwenden Sie Fullstack-Frontend-Modi mit `--backend self`, wenn das Framework Serverrouten besitzt.

## React Native Apps

React Native ist ein eigenes Ökosystem. Wählen Sie es mit `--ecosystem react-native` oder als Multi-Ökosystem-Teil mit `mobile:react-native:native-bare` aus.

Zu den mobilen Optionen gehören Navigation, Benutzeroberfläche, Speicher, Tests, Push-Benachrichtigungen, OTA-Updates und Deep Linking. Sie implizieren nicht TypeScript Web-Backend- oder Bereitstellungskategorien.

## Rust WASM-Frontends

Rust kann über `--rust-frontend` Leptos- oder Dioxus-Frontend-Oberflächen generieren. Dies sind Optionen für das Rust-Ökosystem, getrennt von TypeScript-Web-Frontends.

## App-Plattformen

App-Plattform-Add-ons wie Tauri, PWA, WXT und OpenTUI fügen Shell-, Offline-, Erweiterungs- oder Terminal-App-Funktionen rund um kompatible Stacks hinzu.
