UI and experience choices shape the generated frontend code, design primitives, and client-side application patterns.

## Styling and UI libraries

CSS choices include Tailwind, SCSS, Less, PostCSS-only, or none. UI libraries include shadcn/ui, shadcn-svelte, daisyUI, Radix UI, Headless UI, Park UI, Chakra UI, NextUI, Mantine, MUI, Ant Design, Base UI, Ark UI, React Aria, and none.

shadcn/ui has additional presets for base primitives, visual style, icons, color theme, neutral base color, font, and radius.

## Forms, validation, and state

Frontend application choices include:

- State: Zustand, Jotai, Nanostores, Redux Toolkit, MobX, XState, Valtio, TanStack Store, Legend State, or none.
- Forms: TanStack Form, React Hook Form, Formik, Final Form, Conform, Modular Forms, or none.
- Validation: Zod, Valibot, ArkType, TypeBox, Typia, Runtypes, Effect Schema, or none.

## Experience features

Animation, i18n, and frontend-adjacent app-platform addons live here conceptually, even when their exact flags appear in different option tables.

React Native has its own mobile UI choices: Tamagui, Gluestack UI, Uniwind, Unistyles, or none.
