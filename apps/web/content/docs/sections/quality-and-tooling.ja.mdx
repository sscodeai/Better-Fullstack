品質とツールの選択によって、生成されたプロジェクトがどのようにチェック、文書化、保守されるかが決まります。

## テスト

TypeScript テストの選択肢には、Vitest、Playwright、Vitest に加えて Playwright、Jest、Cypress、またはなしが含まれます。

他のエコシステムは、Rust テスト ライブラリ、Python 品質ツール、Java テスト ライブラリ、Elixir ExUnit/Mox/Bypass/Wallaby の選択肢、Maestro または React Native テスト ライブラリを使用したモバイル テストなど、ネイティブ テスト パスを公開しています。

## ロギングと可観測性

ロギングと可観測性はスタック固有である場合があります。 TypeScript には、Pino、Winston、Evlog、OpenTelemetry、Sentry、および Grafana の選択肢があります。 Go と Rust には専用のロギング カテゴリがあります。 Java と Elixir は、エコシステム オプションを通じて可観測性関連のライブラリを公開します。

## コードの品質とドキュメント

コード品質の選択肢には、Biome、Oxlint、Ultracite、Lefthook、Husky、および Ruler が含まれます。ドキュメント アドオンには、互換性のある Starlight および Fumadocs が含まれます。

## プロジェクトのワークフロー

パッケージ マネージャー、バージョン チャネル、AI ドキュメント、Git の初期化、インストール動作、`--dry-run`、および `--verify` はすべて、生成されたアプリ ランタイムではなくワークフロー レイヤーに属します。
