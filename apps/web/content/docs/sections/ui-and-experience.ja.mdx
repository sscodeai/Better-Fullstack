UI とエクスペリエンスの選択により、生成されるフロントエンド コード、デザイン プリミティブ、およびクライアント側のアプリケーション パターンが形成されます。

## スタイルおよび UI ライブラリ

CSS の選択肢には、Tailwind、SCSS、Less、PostCSS-only、または none が含まれます。 UI ライブラリには、shadcn/ui、shadcn-svelte、daisyUI、Radix UI、Headless UI、Park UI、Chakra UI、NextUI、Mantine、MUI、Ant Design、Base UI、Ark UI、React Aria、および none が含まれます。

shadcn/ui には、ベース プリミティブ、ビジュアル スタイル、アイコン、カラー テーマ、ニュートラルなベース カラー、フォント、および半径の追加のプリセットがあります。

## フォーム、検証、状態

フロントエンド アプリケーションの選択肢は次のとおりです。

- 状態: Zustand、Jotai、Nanostores、Redux Toolkit、MobX、XState、Valtio、TanStack Store、Legend State、またはなし。
- フォーム: TanStack フォーム、React Hook フォーム、Formik、Final Form、Conform、Modular Forms、またはなし。
- 検証: Zod、Valibot、ArkType、TypeBox、Typia、Runtypes、エフェクト スキーマ、またはなし。

## 体験機能

アニメーション、i18n、およびフロントエンドに隣接するアプリ プラットフォーム アドオンは、それらの正確なフラグが別のオプション テーブルに表示されている場合でも、概念的にはここに存在します。

React Native には独自のモバイル UI の選択肢があります: Tamagui、Gluestack UI、Uniwind、Unistyles、またはなし。
