품질 및 도구 선택은 생성된 프로젝트의 확인, 문서화 및 유지 관리 방법을 결정합니다.

## 테스트

TypeScript 테스트 선택에는 Vitest, Playwright, Vitest 및 Playwright, Jest, Cypress 또는 없음이 포함됩니다.

다른 생태계에서는 Rust 테스트 라이브러리, Python 품질 도구, Java 테스트 라이브러리, Elixir ExUnit/Mox/Bypass/Wallaby 선택, Maestro 또는 React Native 테스트 라이브러리를 사용한 모바일 테스트와 같은 기본 테스트 경로를 공개합니다.

## 로깅 및 관찰 가능성

로깅 및 관찰 가능성은 스택별로 다를 수 있습니다. TypeScript에는 Pino, Winston, Evlog, OpenTelemetry, Sentry 및 Grafana 선택 항목이 있습니다. Go와 Rust에는 전용 로깅 카테고리가 있습니다. Java와 Elixir는 생태계 옵션을 통해 관찰 가능성 관련 라이브러리를 공개합니다.

## 코드 품질 및 문서

코드 품질 선택에는 Biome, Oxlint, Ultracite, Lefthook, Husky 및 Ruler가 포함됩니다. 문서 애드온에는 호환되는 경우 Starlight 및 Fumadocs가 포함됩니다.

## 프로젝트 워크플로우

패키지 관리자, 버전 채널, AI 문서, Git 초기화, 설치 동작, `--dry-run` 및 `--verify`은 모두 생성된 앱 런타임이 아닌 워크플로 계층에 속합니다.
