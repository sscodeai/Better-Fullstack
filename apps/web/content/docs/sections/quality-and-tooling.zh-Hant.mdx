品質和工具鏈選擇決定產生項目如何被檢查、記錄和維護。

## 測試

TypeScript 測試選擇包含 Vitest、Playwright、Vitest plus Playwright、Jest、Cypress 或 none。

其他生態系統會暴露它們的原生測試路徑，例如 Rust testing libraries、Python quality tools、Java testing libraries、Elixir ExUnit/Mox/Bypass/Wallaby 選擇，以及使用 Maestro 或 React Native Testing Library 的行動端測試。

## 日誌和可觀測性

日誌和可觀測性可以是 stack-specific 的。 TypeScript 有 Pino、Winston、Evlog、OpenTelemetry、Sentry 和 Grafana 選擇。 Go 和 Rust 有專門的日誌分類。 Java 和 Elixir 透過它們的生態系統選項暴露可觀測性相關函式庫。

## 程式碼品質和文檔

程式碼品質選擇包括 Biome、Oxlint、Ultracite、Lefthook、Husky 和 ​​Ruler。文件 addons 在相容時包括 Starlight 和 Fumadocs。

## 項目 workflow

Package manager、version channel、AI docs、Git 初始化、install 行為、`--dry-run` 和 `--verify` 都屬於 workflow 層，而不是產生應用的 runtime。
