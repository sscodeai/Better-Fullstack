L'authentification et les paiements affectent les schémas générés, les variables d'environnement, les rappels, le middleware et les notes de configuration du fournisseur.

## Authentification

Les fournisseurs d'authentification TypeScript incluent Better Auth, Clerk, NextAuth, Stack Auth, Supabase Auth, Auth0 et aucun.

D'autres écosystèmes exposent des assistants natifs là où des modèles existent :

| Écosystème | Choix d'authentification |
| --- | --- |
| Rouiller | OAuth2 ou aucun. |
| Python | Authlib, JWT ou aucun. |
| Aller | Casbin, JWT assistants ou aucun. GoBetterAuth utilise la valeur globale `--auth go-better-auth`. |
| Java | Spring Security ou aucun. |
| Élixir | phx.gen.auth, Ueberauth, Guardian ou aucun. Certains choix d'authentification Elixir restent désactivés sur le graphique jusqu'à ce que les modèles rattrapent leur retard. |

## Paiements

Les fournisseurs de paiement sont TypeScript-premiers aujourd'hui : Polar, Stripe, Lemon Squeezy, Paddle, Dodo ou aucun.

Certains fournisseurs exigent des choix d'authentification ou d'interface spécifiques. Par exemple, Polar nécessite Better Auth et une interface Web.
