Les choix de qualité et d'outillage déterminent la manière dont le projet généré est vérifié, documenté et maintenu.

## Essai

Les choix de test TypeScript incluent Vitest, Playwright, Vitest plus Playwright, Jest, Cypress ou aucun.

D'autres écosystèmes exposent leurs chemins de test natifs, tels que les bibliothèques de tests Rust, les outils de qualité Python, les bibliothèques de tests Java, les choix Elixir ExUnit/Mox/Bypass/Wallaby et les tests mobiles avec Maestro ou la bibliothèque de tests React Native.

## Journalisation et observabilité

La journalisation et l'observabilité peuvent être spécifiques à la pile. TypeScript propose les choix Pino, Winston, Evlog, OpenTelemetry, Sentry et Grafana. Go et Rust ont des catégories de journalisation dédiées. Java et Elixir exposent des bibliothèques liées à l'observabilité via leurs options d'écosystème.

## Qualité du code et documentation

Les choix de qualité de code incluent Biome, Oxlint, Ultracite, Lefthook, Husky et Ruler. Les modules complémentaires de documentation incluent Starlight et Fumadocs lorsqu'ils sont compatibles.

## Flux de travail du projet

Le gestionnaire de packages, le canal de version, les documents AI, l'initialisation de Git, le comportement d'installation, `--dry-run` et `--verify` appartiennent tous à la couche de flux de travail plutôt qu'au runtime d'application généré.
