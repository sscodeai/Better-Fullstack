Les choix de données déterminent où les applications générées conservent les enregistrements, mettent en cache le travail, recherchent du contenu et stockent les fichiers.

## Base de données et ORM

La catégorie de base de données partagée comprend SQLite, PostgreSQL, MySQL, MongoDB, EdgeDB, Redis et aucun. En pratique, les ORM sont spécifiques à un écosystème :

| Écosystème | ORM ou couche de données |
| --- | --- |
| TypeScript | Drizzle, Prisma, Mongoose, TypeORM, Kysely, MikroORM, Sequelize ou aucun. |
| Rouiller | SeaORM, SQLx, Diesel ou aucun. |
| Python | SQLAlchemy, SQLModel, Tortoise ORM ou aucun. |
| Aller | GORM, sqlc, Ent ou aucun. |
| Java | Données de printemps JPA ou aucune. |
| Élixir | Ecto, Ecto SQL ou aucun. |

## Configuration du fournisseur

`--db-setup` peut ajouter une configuration d'hébergement/fournisseur pour Turso, Neon, Prisma Postgres, PlanetScale, MongoDB Atlas, Supabase, Upstash, Cloudflare D1, Docker ou aucun.

## Services de style stockage

Les sections adjacentes aux données comprennent :

- Mise en cache : Upstash Redis dans TypeScript ainsi que des choix de cache spécifiques à l'écosystème lorsqu'ils sont pris en charge.
- Recherche : Meilisearch, Typesense, Elasticsearch, Algolia et chemins de recherche spécifiques à l'écosystème.
- Téléchargement de fichiers : UploadThing, FilePond, Uppy ou aucun.
- Stockage de fichiers : S3, R2, Cloudinary ou aucun.
