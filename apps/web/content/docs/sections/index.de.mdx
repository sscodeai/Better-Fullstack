Better Fullstack bietet viele Auswahlmöglichkeiten, sodass Dokumente nach dem Teil des generierten Produkts organisiert sind, auf den sie sich auswirken. Auf diesen Seiten wird erläutert, was die einzelnen Abschnitte bedeuten, wann eine Option ausgewählt werden muss und wo die genauen CLI-Werte zu finden sind.

Für vollständige Optionstabellen verwenden Sie [Referenzoptionen](/docs/reference/options/). Für sprachspezifische Oberflächen verwenden Sie [Ökosysteme](/docs/ecosystems/).

## Solo- und Multi-Modus

Im Solo-Modus steuert ein Ökosystem die sichtbaren Kategorien. Im [Multi-Ökosystem-Modus](/docs/ecosystems/multi-ecosystem/) werden dieselben Konzepte zu Stack-Teilen, z. B. `frontend:typescript:next`, `backend:go:gin`, `backend.orm:go:gorm` und `database:universal:postgres`.
