Auth 和付款會影響產生的 schemas、環境變數、callbacks、middleware 和 provider setup notes。

## Auth

TypeScript auth providers 包含 Better Auth、Clerk、NextAuth、Stack Auth、Supabase Auth、Auth0 和 none。

其他生態系會在模板存在時暴露原生 helpers：

| 生態系統 | Auth 選擇 |
| --- | --- |
| Rust | OAuth2 或 none。 |
| Python | Authlib、JWT 或 none。 |
| Go | Casbin、JWT helpers 或 none。 GoBetterAuth 使用全域 `--auth go-better-auth` 值。 |
| Java | Spring Security 或 none。 |
| Elixir | phx.gen.auth、Ueberauth、Guardian 或 none。一些 Elixir auth 選擇會在模板補齊前保持 graph-disabled。 |

## 支付

今天的付款提供者以 TypeScript 為先：Polar、Stripe、Lemon Squeezy、Paddle、Dodo 或 none。

有些 providers 需要特定的 auth 或 frontend 選擇。例如，Polar 需要 Better Auth 和 Web frontend。
