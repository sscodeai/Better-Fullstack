Better Fullstack has many choices, so docs are organized by the part of the generated product they affect. These pages explain what each section means, when to choose an option, and where to find the exact CLI values.

For complete option tables, use [Reference Options](/docs/reference/options/). For language-specific surfaces, use [Ecosystems](/docs/ecosystems/).

| Section | What it covers |
| --- | --- |
| [Frontend and Apps](/docs/sections/frontend-and-apps/) | Web frontends, React Native apps, app platforms, and frontend-specific constraints. |
| [Backend and API](/docs/sections/backend-and-api/) | Server frameworks, runtimes, API layers, and multi-ecosystem backend choices. |
| [Data and Storage](/docs/sections/data-and-storage/) | Databases, ORMs, hosting setup, caching, search, file upload, and file storage. |
| [Auth and Payments](/docs/sections/auth-and-payments/) | Identity providers, auth helpers, OAuth/JWT paths, and payment providers. |
| [UI and Experience](/docs/sections/ui-and-experience/) | CSS, UI libraries, shadcn/ui presets, forms, validation, state, animation, and i18n. |
| [AI and Services](/docs/sections/ai-and-services/) | AI SDKs, email, realtime, job queues, CMS, analytics, and feature flags. |
| [Quality and Tooling](/docs/sections/quality-and-tooling/) | Testing, logging, observability, code quality, docs addons, package managers, AI docs, Git, and install behavior. |
| [Deployment](/docs/sections/deployment/) | Web/server deployment choices as one stack section, not a top-level docs area. |

## Solo and multi mode

In solo mode, one ecosystem drives the visible categories. In [Multi-Ecosystem mode](/docs/ecosystems/multi-ecosystem/), the same concepts become stack parts, such as `frontend:typescript:next`, `backend:go:gin`, `backend.orm:go:gorm`, and `database:universal:postgres`.
