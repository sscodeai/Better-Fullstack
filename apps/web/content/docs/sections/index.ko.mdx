Better Fullstack에는 선택 사항이 많으므로 문서는 영향을 받는 생성된 제품의 부분별로 구성됩니다. 이 페이지에서는 각 섹션의 의미, 옵션 선택 시기, 정확한 CLI 값을 찾을 수 있는 위치를 설명합니다.

전체 옵션 테이블을 보려면 [참조 옵션](/docs/reference/options/)을 사용하세요. 언어별 표면의 경우 [생태계](/docs/ecosystems/)를 사용하세요.

| 부분 | 다루는 내용 |
| --- | --- |
| [프런트엔드 및 앱](/docs/sections/frontend-and-apps/) | 웹 프런트엔드, React Native 앱, 앱 플랫폼 및 프런트엔드별 제약 조건. |
| [백엔드 및 API](/docs/sections/backend-and-api/) | 서버 프레임워크, 런타임, API 레이어 및 다중 생태계 백엔드 선택. |
| [데이터 및 저장공간](/docs/sections/data-and-storage/) | 데이터베이스, ORM, 호스팅 설정, 캐싱, 검색, 파일 업로드 및 파일 저장. |
| [인증 및 결제](/docs/sections/auth-and-payments/) | ID 공급자, 인증 도우미, OAuth/JWT 경로 및 결제 공급자. |
| [UI 및 경험](/docs/sections/ui-and-experience/) | CSS, UI 라이브러리, shadcn/ui 사전 설정, 양식, 유효성 검사, 상태, 애니메이션 및 i18n. |
| [AI 및 서비스](/docs/sections/ai-and-services/) | AI SDK, 이메일, 실시간, 작업 대기열, CMS, 분석 및 기능 플래그. |
| [품질 및 툴링](/docs/sections/quality-and-tooling/) | 테스트, 로깅, 관찰 가능성, 코드 품질, 문서 애드온, 패키지 관리자, AI 문서, Git 및 설치 동작. |
| [배포](/docs/sections/deployment/) | 최상위 문서 영역이 아닌 하나의 스택 섹션으로 웹/서버 배포를 선택합니다. |

## 솔로 및 멀티 모드

솔로 모드에서는 하나의 생태계가 가시적인 카테고리를 주도합니다. [다중 생태계 모드](/docs/ecosystems/multi-ecosystem/)에서는 동일한 개념이 `frontend:typescript:next`, `backend:go:gin`, `backend.orm:go:gorm`, `database:universal:postgres`과 같은 스택 부분이 됩니다.
