Cette section contrôle les surfaces d'application destinées aux utilisateurs générées par Better Fullstack.

## Interface Web

Les interfaces Web TypeScript sont sélectionnées avec `--frontend`. La catégorie prend en charge React, Vinext, Vue/Nuxt, SvelteKit, Solid, Astro, Qwik, Angular, Redwood, Fresh et `none`.

Utilisez `none` pour les services backend uniquement. Utilisez les modes frontend fullstack avec `--backend self` lorsque le framework possède des routes de serveur.

## React Native applications

React Native est son propre écosystème. Sélectionnez-le avec `--ecosystem react-native` ou en tant que partie multi-écosystème avec `mobile:react-native:native-bare`.

Les choix mobiles incluent la navigation, l'interface utilisateur, le stockage, les tests, les notifications push, les mises à jour OTA et les liens profonds. Ils n’impliquent pas le backend Web TypeScript ou les catégories de déploiement.

## Interfaces Rust WASM

Rust peut générer des surfaces frontales Leptos ou Dioxus via `--rust-frontend`. Ce sont des choix de l'écosystème Rust, distincts des interfaces Web TypeScript.

## Plateformes d'applications

Les modules complémentaires de plate-forme d'application tels que Tauri, PWA, WXT et OpenTUI ajoutent des fonctionnalités d'application shell, hors ligne, d'extension ou de terminal autour des piles compatibles.
