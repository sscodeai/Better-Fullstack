Better Fullstack 有許多選擇，因此文件會依照它們影響的生成產品部分來組織。這些頁面解釋每個分割區的含義、何時選擇某個選項，以及在哪裡找到精確的 CLI 值。

完整選項表請使用[參考選項](/docs/reference/options/)。特定語言導向的介面請使用[生態系](/docs/ecosystems/)。

| 分割區 | 覆蓋內容 |
| --- | --- |
| [前端與應用](/docs/sections/frontend-and-apps/) | Web 前端、React Native 應用、應用平台，以及前端專用限制。 |
| [後端與 API](/docs/sections/backend-and-api/) | 服務端框架、runtime、API 層，以及 multi-ecosystem 後端選擇。 |
| [資料與儲存](/docs/sections/data-and-storage/) | 資料庫、ORM、託管設定、快取、搜尋、文件上傳和文件儲存。 |
| [Auth 和付款](/docs/sections/auth-and-payments/) | 身分提供者、auth 輔助工具、OAuth/JWT 路徑和付款提供者。 |
| [UI 與體驗](/docs/sections/ui-and-experience/) | CSS、UI 庫、shadcn/ui presets、表單、驗證、狀態、動畫和 i18n。 |
| [AI 和服務](/docs/sections/ai-and-services/) | AI SDK、email、realtime、job queues、CMS、analytics 和 feature flags。 |
| [品質與工具鏈](/docs/sections/quality-and-tooling/) | 測試、日誌、可觀測性、程式碼品質、docs addons、package managers、AI docs、Git 和安裝行為。 |
| [部署](/docs/sections/deployment/) | Web/server 部署選擇作為一個 stack 分區，而不是頂層文件區域。 |

## Solo 和 multi 模式

在 solo 模式中，一個生態系統驅動可見分類。在 [Multi-Ecosystem 模式](/docs/ecosystems/multi-ecosystem/)中，相同概念會變成 stack parts，例如 `frontend:typescript:next`、`backend:go:gin`、`backend.orm:go:gorm` 和 `database:universal:postgres`。
