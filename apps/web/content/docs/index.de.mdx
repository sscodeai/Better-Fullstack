Better Fullstack ist ein Projektgenerator für typisierte Produktstapel. Wählen Sie Frontend, Backend, Datenschicht, Dienste, Tools und Zielökosystem aus; Der CLI schreibt die passenden Dateien und zeichnet den Stapel in `bts.jsonc` auf.

Verwenden Sie das interaktive CLI, wenn Sie eine geführte Einrichtung wünschen, das [Stack Builder](https://better-fullstack.dev/new), wenn Sie Optionen visuell vergleichen möchten, und explizite Flags, wenn Sie einen reproduzierbaren Befehl für CI, Dokumente oder einen anderen Entwickler benötigen.

## Anforderungen

Der CLI läuft auf Node.js 20 oder neuer.

```bash
node --version
```

Generierte Projekte benötigen möglicherweise auch ihre Ökosystem-Toolchain: Expo-Tooling für React Native, Cargo für Rust, `uv` für Python-, Go-, Java 21- oder Elixir/Erlang für Phoenix-Projekte.

## Erstellen Sie Ihr erstes Projekt

Führen Sie den Assistenten aus und beantworten Sie die Eingabeaufforderungen:

```npm
npm create better-fullstack@latest my-app
```

Für npm stehen Better Fullstack-Flags nach dem Trennzeichen `--`. Die gerenderten Paketmanager-Registerkarten zeigen die passenden pnpm-, Bun- und Yarn-Befehle, bei denen das Trennzeichen nicht erforderlich ist.

## Was die Standardeinstellungen bewirken

Durch das Akzeptieren der Standardeinstellungen wird ein TypeScript-Projekt erstellt mit:

- TanStack Router Frontend und Hono Backend.
- Bun Laufzeit, SQLite, Drizzle, Better Auth und tRPC.
- Tailwind CSS, shadcn/ui, Vitest und Turborepo.
- Der Paketmanager, der zum Starten von CLI, stabilen Abhängigkeitsversionen, Git, aktivierter Installation und `CLAUDE.md` verwendet wird.

Sie können diesen Stapel behalten, mit einer Vorlage beginnen oder das Ökosystem komplett wechseln.

## Möglichkeiten, einen Stapel auszuwählen

| Weg | Verwenden Sie es wann |
| --- | --- |
| CLI-Assistent | Sie benötigen Eingabeaufforderungen und sinnvolle Standardeinstellungen. |
| Stack Builder | Sie möchten vor dem Gerüstbau kompatible Kombinationen erkunden. |
| Explizite Flags | Sie möchten eine wiederholbare Ausgabe in Skripten, Dokumenten oder Problemberichten. |
| Agentenfähigkeit | Sie möchten, dass ein Codierungsagent den CLI-Workflow verwendet, ohne Startdateien handschriftlich zu schreiben. |
| MCP Server | Sie möchten, dass ein AI-Codierungsagent das Schema überprüft, die Kompatibilität validiert und einen sicheren Probelauf durchführt. |

## Generierter Projektkontext

Jedes generierte Projekt beinhaltet:

- `bts.jsonc` mit dem ausgewählten Stapel.
- Ein reproduzierbarer Befehl, der vom CLI gedruckt wird.
- Framework-, Paket- und Konfigurationsdateien für das ausgewählte Ökosystem.
- Optionale AI-Anweisungsdateien wie `CLAUDE.md`, `Agents.md` oder `.cursorrules`.
