Better Fullstack은 유형화된 제품 스택을 위한 프로젝트 생성기입니다. 프런트엔드, 백엔드, 데이터 계층, 서비스, 도구, 대상 생태계를 선택하세요. CLI는 일치하는 파일을 쓰고 `bts.jsonc`에 스택을 기록합니다.

안내식 설정이 필요한 경우 대화형 CLI을 사용하고, 옵션을 시각적으로 비교하려는 경우 [Stack Builder](https://better-fullstack.dev/new)를 사용하고, CI, 문서 또는 다른 개발자를 위한 재현 가능한 명령이 필요한 경우 명시적 플래그를 사용합니다.

## 요구사항

CLI은 Node.js 20 이상에서 실행됩니다.

```bash
node --version
```

생성된 프로젝트에는 React Native용 Expo 도구, Rust용 Cargo, Python, Go, Java 21용 `uv` 또는 Phoenix 프로젝트용 Elixir/Erlang과 같은 생태계 툴체인이 필요할 수도 있습니다.

## 첫 번째 프로젝트 만들기

마법사를 실행하고 프롬프트에 응답합니다.

```npm
npm create better-fullstack@latest my-app
```

npm의 경우 Better Fullstack 플래그는 `--` 구분 기호 뒤에 옵니다. 렌더링된 패키지 관리자 탭에는 구분 기호가 필요하지 않은 일치하는 pnpm, Bun 및 Yarn 명령이 표시됩니다.

## 기본값이 만드는 것

기본값을 수락하면 다음을 사용하여 TypeScript 프로젝트가 생성됩니다.

- TanStack Router 프런트엔드 및 Hono 백엔드.
- Bun 런타임, SQLite, Drizzle, Better Auth 및 tRPC.
- Tailwind CSS, shadcn/ui, Vitest 및 Turborepo.
- CLI, 안정적인 종속성 버전, Git, 설치 활성화 및 `CLAUDE.md`을 시작하는 데 사용되는 패키지 관리자입니다.

해당 스택을 유지하거나, 템플릿에서 시작하거나, 생태계를 완전히 전환할 수 있습니다.

## 스택을 선택하는 방법

| 길 | 다음과 같은 경우에 사용하세요. |
| --- | --- |
| CLI 마법사 | 프롬프트와 합리적인 기본값을 원합니다. |
| Stack Builder | 비계를 설치하기 전에 호환 가능한 조합을 탐색하고 싶습니다. |
| 명시적 플래그 | 스크립트, 문서 또는 문제 보고서에서 반복 가능한 출력을 원합니다. |
| 에이전트 스킬 | 코딩 에이전트가 시작 파일을 직접 작성하지 않고도 CLI 워크플로를 사용하기를 원합니다. |
| MCP 서버 | AI 코딩 에이전트가 스키마를 검사하고, 호환성을 검증하고, 안전하게 테스트 실행하기를 원합니다. |

## 생성된 프로젝트 컨텍스트

생성된 모든 프로젝트에는 다음이 포함됩니다.

- `bts.jsonc`을(를) 선택한 스택으로 바꿉니다.
- CLI에 의해 인쇄되는 재현 가능한 명령입니다.
- 선택한 생태계에 대한 프레임워크, 패키지 및 구성 파일입니다.
- `CLAUDE.md`, `Agents.md` 또는 `.cursorrules`와 같은 선택적 AI 명령 파일입니다.
