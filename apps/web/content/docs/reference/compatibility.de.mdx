Better Fullstack validiert die Stapelauswahl vor der Generierung. Der Builder, CLI und der MCP-Server verwenden dieselbe Kompatibilitätslogik, sodass ungültige Kombinationen konsistent abgelehnt oder angepasst werden.

<CompatibilityMatrix />

## Was es überprüft

- Ob eine Option mit dem ausgewählten Baseline-Stack kompatibel ist.
- Der gemeinsame Deaktivierungsgrund, wenn eine Kombination blockiert wird.
- Alle automatischen Anpassungsnotizen aus demselben Kompatibilitätsanalysator, der vom Builder verwendet wird.

## Wie man es benutzt

1. Wählen Sie ein Ökosystem.
2. Passen Sie die Baseline-Stack-Felder an, die für Ihre Frage von Bedeutung sind.
3. Wählen Sie eine Kategorie zur Inspektion aus.
4. Überprüfen Sie, welche Optionen kompatibel sind und warum welche blockiert sind.

Dies ist eine praktische Matrix, nicht jede mögliche paarweise Kombination. Für eine vollständige Projektvalidierung führen Sie CLI aus, verwenden Sie Stack Builder oder rufen Sie `bfs_check_compatibility` bis MCP vor dem Gerüstbau auf.

## Kompatibilitätsverhalten

- `none` deaktiviert optionale Kategorien, sofern unterstützt.
- Einige Auswahlmöglichkeiten lösen automatische Anpassungen aus, z. B. „Convex ersetzt Backend-eigene Datenbank“/API-Auswahlmöglichkeiten.
- Einige Frontend-Optionen schränken absichtlich die Unterstützung für API, Benutzeroberfläche, Laufzeit oder Bereitstellung ein.
- Java, Rust, Python und Go verwenden ökosystemspezifische Kategorien anstelle der vollständigen TypeScript-Stack-Oberfläche.
