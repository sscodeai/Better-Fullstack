Better Fullstack은 생성 전에 스택 선택을 검증합니다. 빌더, CLI 및 MCP 서버는 동일한 호환성 로직을 사용하므로 잘못된 조합은 거부되거나 일관되게 조정됩니다.

<CompatibilityMatrix />

## 검사 내용

- 옵션이 선택한 기준 스택과 호환되는지 여부입니다.
- 조합이 차단된 경우 공유된 비활성화 이유입니다.
- 빌더에서 사용하는 것과 동일한 호환성 분석기의 자동 조정 메모입니다.

## 사용방법

1. 생태계를 선택하세요.
2. 질문에 중요한 기준 스택 필드를 조정합니다.
3. 검사할 카테고리를 선택하세요.
4. 어떤 옵션이 호환되는지, 왜 차단되는지 검토하세요.

이는 가능한 모든 쌍별 조합이 아닌 실용적인 행렬입니다. 전체 프로젝트 유효성을 검사하려면 스캐폴딩 전에 CLI을 실행하거나 Stack Builder을 사용하거나 `bfs_check_compatibility`을(를) 통해 MCP를 호출하세요.

## 호환성 동작

- `none`은 지원되는 경우 선택적 범주를 비활성화합니다.
- 백엔드 소유 데이터베이스를 교체하는 Convex/API 선택과 같은 일부 선택은 자동 조정을 트리거합니다.
- 일부 프런트엔드 선택 사항은 의도적으로 API, UI, 런타임 또는 배포 지원을 좁힙니다.
- Java, Rust, Python 및 Go는 전체 TypeScript 스택 표면 대신 생태계별 카테고리를 사용합니다.
