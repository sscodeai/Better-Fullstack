Better Fullstack 會在生成前驗證 stack choices。 builder、CLI 和 MCP server 使用相同的相容性邏輯，因此無效組合會被一致地拒絕或調整。

<CompatibilityMatrix />

## 檢查內容

- 某個選項是否與選取的 baseline stack 相容。
- 當組合被阻止時，共享的 disabled reason。
- 來自 builder 所用相同相容性分析器的任何自動調整說明。

## 如何使用

1. 選擇一個生態系。
2. 調整與你的問題相關的 baseline stack 欄位。
3. 選擇要檢查的分類。
4. 查看哪些選項相容，以及被封鎖的原因。

這是一個實用矩陣，並不是所有可能的 pairwise 組合。完整項目驗證請執行 CLI、使用 Stack Builder，或在 scaffold 前透過 MCP 呼叫 `bfs_check_compatibility`。

## 相容性行為

- 支援時，`none` 會停用可選分類。
- 有些選擇會觸發自動調整，例如 Convex 取代 backend-owned database/API choices。
- 有些前端選擇會有意收窄 API、UI、runtime 或 deploy 支援。
- Java、Rust、Python 和 Go 使用生態系統專用分類，而不是完整的 TypeScript stack surface。
