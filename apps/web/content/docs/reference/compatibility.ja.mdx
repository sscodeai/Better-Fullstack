Better Fullstack は、生成前にスタックの選択を検証します。ビルダー、CLI、および MCP サーバーは同じ互換性ロジックを使用するため、無効な組み合わせは拒否されるか、一貫して調整されます。

<CompatibilityMatrix />

## チェック内容

- オプションが選択したベースライン スタックと互換性があるかどうか。
- 組み合わせがブロックされた場合の共有の無効な理由。
- ビルダーが使用したのと同じ互換性アナライザーからの自動調整メモ。

## 使い方

1. エコシステムを選択してください。
2. 質問にとって重要なベースライン スタック フィールドを調整します。
3. 検査するカテゴリを選択します。
4. どのオプションに互換性があるのか​​、またどのオプションがブロックされるのかを確認してください。

これは実用的な行列であり、可能なすべてのペアごとの組み合わせではありません。プロジェクトを完全に検証するには、スキャフォールディングの前に CLI を実行するか、Stack Builder を使用するか、`bfs_check_compatibility` から MCP を呼び出します。

## 互換性の動作

- `none` は、サポートされている場合はオプションのカテゴリを無効にします。
- 一部の選択では、バックエンド所有のデータベース/API の選択を Convex で置き換えるなど、自動調整がトリガーされます。
- 一部のフロントエンドの選択では、API、UI、ランタイム、またはデプロイのサポートが意図的に狭められています。
- Java、Rust、Python、Go は、完全な TypeScript スタック サーフェスの代わりに、エコシステム固有のカテゴリを使用します。
