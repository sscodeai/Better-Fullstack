React Native est l'écosystème mobile dédié. Il échafaude les projets Expo et sépare les choix mobiles des catégories TypeScript frontend Web, backend, base de données et déploiement.

## Conditions préalables

- Node.js 20+.
- npm, pnpm, Yarn ou Bun.
- Outils iOS/Android compatibles Expo lorsque vous souhaitez une validation de simulateur ou d'appareil.

## Exemple de script

```npm
npm create better-fullstack@latest mobile-app -- \
  --ecosystem react-native \
  --frontend native-bare \
  --mobile-navigation expo-router \
  --mobile-ui tamagui \
  --mobile-storage mmkv \
  --mobile-testing maestro-react-native-testing-library \
  --mobile-push expo-notifications \
  --mobile-ota expo-updates \
  --mobile-deep-linking expo-linking \
  --ai-docs none \
  --no-install
```

## Catégories d'échafaudages mobiles

| Catégorie | Valeurs |
| --- | --- |
| Interface native | `native-bare` `native-uniwind` `native-unistyles` |
| Navigation | `expo-router` `react-navigation` `none` |
| Interface utilisateur | `tamagui` `gluestack-ui` `uniwind` `unistyles` `none` |
| Stockage | `mmkv` `none` |
| Essai | `maestro` `react-native-testing-library` `maestro-react-native-testing-library` `none` |
| Pousser | `expo-notifications` `none` |
| OTA | `expo-updates` `none` |
| Liens profonds | `expo-linking` `none` |

## Intégrations générées

- Configuration Expo avec paramètres de schéma/d'exécution lorsque les liens profonds ou OTA sont sélectionnés.
- Expo Fichiers de route du routeur ou point d'entrée React Navigation, en fonction de la navigation.
- Fournisseur d'interface utilisateur mobile/fichiers de thème pour Tamagui, Glustack UI, Uniwind ou Unistyles.
- Aides client MMKV lorsque le stockage est sélectionné.
- Exemples de bibliothèques de tests Jest Expo et React Native, flux Maestro, ou les deux.
- Aides à l'enregistrement des notifications push pour les notifications Expo.
- Aides à la mise à jour OTA pour les mises à jour Expo.
- `.env.example` avec les conventions `EXPO_PUBLIC_` pour la configuration du client mobile.

## Remarques sur la compatibilité

- React Native n'expose pas les catégories TypeScript d'interface Web, de backend, de base de données, d'interface utilisateur Web, de déploiement Web ou de déploiement de serveur.
- Les modes d'interface utilisateur Uniwind et Unistyles sont liés à leurs variantes frontales Expo correspondantes.
- Tamagui et Glustack UI utilisent la seule variante Expo pour éviter une configuration de style conflictuelle.
- Les liens profonds sont activés automatiquement lorsqu'un flux d'authentification mobile nécessite des exemples d'URI de redirection.
