TypeScript ist das umfassendste Better Fullstack-Ökosystem. Verwenden Sie es für Web-Apps, APIs, Worker, Desktop-Shells, Browser-Erweiterungen, Dokumentationsseiten, Service-Integrationen und die meisten Add-on-Workflows.

React Native/Expo mobile Gerüste leben im dedizierten [React Native](/docs/ecosystems/react-native/) Ökosystem.

## Voraussetzungen

- Node.js 20+.
- npm, pnpm, Garn oder Bun.
- Git, wenn Sie eine Repository-Initialisierung wünschen.

## Skriptbeispiel

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --db-setup none \
  --api orpc \
  --auth better-auth \
  --addons turborepo \
  --examples none \
  --ai-docs none \
  --version-channel stable \
  --no-install
```

## Kompatibilitätshinweise

- tRPC ist für React-basierte Frontends.
- `self` Backend ist für Fullstack-Frameworks und Paare mit `runtime none`.
- Qwik, Angular, Redwood und Fresh bieten eine engere API/Backend-Unterstützung als die vollständige Optionstabelle.
- Die Unterstützung der Worker-Laufzeit ist vom Backend abhängig.
- UI-Bibliotheken hängen oft von der Frontend-Familie und dem CSS-Framework ab. Beispielsweise ist shadcn/ui reaktionsorientiert und rückenwindbasiert.
- Bei einigen Anbietern gelten zusätzliche Einschränkungen, beispielsweise erfordert Polar Better Auth und ein Web-Frontend.
- Einige Optionen werden aus Gründen der Kompatibilität oder Abhängigkeitsunterstützung angezeigt, bevor jeder Vorlagenpfad gleich tief ist. Führen Sie die Kompatibilitätsprüfung oder Stack Builder aus, bevor Sie große Kombinationen erstellen.
