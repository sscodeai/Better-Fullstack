Les projets .NET génèrent des applications ASP.NET Core qui suivent les conventions standard `dotnet`. Choisissez les options .NET pour le framework Web, l'accès aux données, l'authentification, le style API, le temps réel, les tâches en arrière-plan, l'observabilité, la mise en cache, les tests et le déploiement de fichiers.

## Conditions préalables

- .NET SDK 10 ou version ultérieure.
- PostgreSQL lors de l'utilisation de `ef-core` avec la base de données `postgres` ; SQLite n'a besoin d'aucun serveur.
- Docker lors de l'utilisation de la cible de déploiement `docker` ou `testcontainers-dotnet`.
- Git si vous souhaitez initialiser le référentiel.

## Exemple de script

```npm
npm create better-fullstack@latest my-dotnet-app -- \
  --ecosystem dotnet \
  --dotnet-web-framework aspnet-minimal \
  --dotnet-orm ef-core \
  --dotnet-auth aspnet-identity \
  --dotnet-api minimal-api \
  --dotnet-testing xunit \
  --dotnet-job-queue hangfire \
  --dotnet-realtime signalr \
  --dotnet-observability serilog \
  --dotnet-caching redis \
  --dotnet-deploy docker \
  --database postgres \
  --ai-docs none \
  --no-install \
  --no-git
```

## Remarques sur la compatibilité

- Les options .NET s'appliquent uniquement avec `--ecosystem dotnet`.
- ASP.NET Core est un framework Web .NET autonome ; Les options frontend et backend TypeScript sont ignorées.
- `--dotnet-testing` et `--dotnet-observability` acceptent plusieurs valeurs.
- `ef-core` sélectionne le fournisseur parmi `--database` : `postgres` utilise Npgsql, tout le reste revient à SQLite.
- `aspnet-identity` enregistre les magasins d'identités uniquement lorsqu'ils sont associés à `ef-core` ; sans cela, l'échafaudage enregistre les services d'authentification et d'autorisation simples.
- Valeurs par défaut non interactives lorsque `--ecosystem dotnet` est défini : `aspnet-minimal`, `ef-core`, `aspnet-identity`, `minimal-api`, `xunit`, `signalr`, `serilog` et `docker`.

## Comportement généré

- Le projet comprend un `.csproj`, `Program.cs`, `appsettings.json`, un fichier de requête `.http`, un projet de test, `.gitignore` et des notes de configuration dans le README.
- Les sélections EF Core connectent un `DbContext` au fournisseur Npgsql ou SQLite ainsi qu’au package au moment de la conception pour les migrations.
- Les sélections facultatives ajoutent du code ASP.NET Core fonctionnel pour les services Identity, Hot Chocolate GraphQL, gRPC, les hubs SignalR, les tâches Hangfire ou Quartz.NET, les pipelines Serilog/NLog/OpenTelemetry, la mise en cache Redis et un Dockerfile à plusieurs étapes.
