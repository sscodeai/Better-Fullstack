다중 생태계 모드는 하나의 평면 생태계 선택 대신 연결된 스택 부분으로 프로젝트를 모델링합니다. 제품이 여러 언어에 걸쳐 있는 경우(예: TypeScript 프런트엔드, Go API, 공유 PostgreSQL 데이터베이스 및 React Native 모바일 앱)에 사용하세요.

[Stack Builder](https://better-fullstack.dev/new?mode=multi)에서 열거나 반복되는 `--part` 플래그를 CLI에 전달합니다.

## 스택 부품

각 부분은 다음 모양을 사용합니다.

```txt
role:ecosystem:tool
owner.role:ecosystem:tool
```

최상위 역할은 생성된 앱 또는 공유 인프라를 설명합니다. 소유된 역할은 상위 부품에 기능을 연결합니다.

| 예 | 의미 |
| --- | --- |
| `frontend:typescript:next` | TypeScript Next.js 프런트엔드를 생성합니다. |
| `backend:go:gin` | Go Gin 백엔드를 생성합니다. |
| `backend.orm:go:gorm` | 선택한 백엔드에 GORM을 연결합니다. |
| `backend.api:rust:tonic` | Tonic gRPC을 Rust 백엔드에 연결합니다. |
| `database:universal:postgres` | 공유 PostgreSQL 데이터베이스 부분을 추가합니다. |
| `mobile:react-native:native-bare` | React Native 모바일 앱을 생성합니다. |

## 스크립트 예시

```npm
npm create better-fullstack@latest my-mixed-app -- \
  --part frontend:typescript:next \
  --part backend:go:gin \
  --part backend.orm:go:gorm \
  --part backend.auth:go:jwt \
  --part database:universal:postgres \
  --part mobile:react-native:native-bare \
  --addons turborepo docker-compose
```

## 진실의 근원

다중 생태계 프로젝트는 `stackParts`을(를) `bts.jsonc`에 씁니다. 이러한 부분은 생성된 구조, 기록 및 재현 가능한 명령에 대한 정보의 소스입니다. `backend`, `orm` 또는 `frontend`와 같은 레거시 필드는 여전히 그래프에서 파생된 호환성 대체 항목으로 나타날 수 있습니다.

생성된 명령은 그래프를 반복되는 `--part` 플래그로 유지하므로 숨겨진 빌더 상태에 의존하지 않고 프로젝트를 다시 생성할 수 있습니다.

## 빌더 흐름

Stack Builder의 다중 생태계 모드는 다음을 안내합니다.

1. 프런트엔드 - TypeScript 웹 프런트엔드.
2. 백엔드 - TypeScript, Rust, Python, Go, Java, Elixir 또는 .NET 백엔드 및 자체 백엔드 기능.
3. 데이터베이스 - 공유 범용 데이터베이스 선택.
4. 모바일 - React Native 앱 선택.
5. 마무리 - 애드온, AI 문서, 설치, Git 및 버전 설정.

## 호환성

- 현재 그래프 모델에서는 역할당 하나의 선택된 기본 부분만 지원됩니다.
- 소유된 기능 부분은 `backend.orm:go:gorm`과 같은 소유자를 참조해야 합니다.
- 역량 생태계는 일반적으로 소유자 생태계와 일치해야 합니다.
- 범용 데이터베이스 부분은 생태계 전체에서 공유될 수 있습니다.
- 일부 도구는 묵시적인 부분을 제공합니다. 예를 들어 선택한 백엔드는 생성된 `stackParts`에 나타나는 데이터베이스 또는 라우팅 가정을 제공할 수 있습니다.
- 호환성 검사기는 생성 전에 그래프 부분의 유효성을 검사하고 소유자, 중복 및 지원되지 않는 도구 문제를 보고합니다.
