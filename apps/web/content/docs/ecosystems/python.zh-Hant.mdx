Python 專案使用 `uv` 和 Python-native templates。為 API frameworks、ORMs、validation、AI libraries、auth helpers、task queues、GraphQL 和 quality tooling 選擇 Python options。

## 前置條件

- Python。
- `uv`。
- 如果想初始化倉庫，需要 Git。

## 腳本化範例

```npm
npm create better-fullstack@latest my-python-api -- \
  --ecosystem python \
  --python-web-framework fastapi \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-ai none \
  --python-auth jwt \
  --python-task-queue none \
  --python-graphql none \
  --python-quality ruff \
  --python-testing pytest hypothesis \
  --python-caching redis \
  --python-realtime python-socketio \
  --python-observability opentelemetry \
  --python-cli typer rich \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## 生成行為

- 啟用 install 時，`uv sync` 是安裝路徑。
- `pythonAi` 是 multi-select category，且獨立於 TypeScript `--ai`。
- 當所選 options 不需要時，空的 conditional Python template files 會被跳過。
- AI instruction files 仍可透過 `--ai-docs` 使用，包含 `--ai-docs none`。
