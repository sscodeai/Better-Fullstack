Rust 프로젝트는 Cargo 및 언어 기반 템플릿을 사용합니다. 백엔드 서비스, CLIs, GraphQL/gRPC, WebAssembly 프런트엔드, 로깅, 캐싱 및 인증 도우미에 대한 Rust 옵션을 선택하세요.

## 전제조건

- Rust 툴체인과 Cargo.
- 저장소 초기화를 원하는 경우 Git.

## 스크립트 예시

```npm
npm create better-fullstack@latest my-rust-api -- \
  --ecosystem rust \
  --rust-web-framework axum \
  --rust-orm sqlx \
  --rust-libraries serde validator \
  --rust-logging tracing \
  --rust-error-handling anyhow-thiserror \
  --rust-api none \
  --rust-frontend none \
  --rust-cli none \
  --rust-caching none \
  --rust-auth none \
  --rust-realtime tokio-tungstenite \
  --rust-message-queue lapin \
  --rust-observability opentelemetry \
  --rust-templating askama \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## 생성된 동작

- `cargo build`은 설치가 활성화된 경우의 설치/확인 경로입니다.
- Leptos, Dioxus, Tonic, Clap 및 Ratatui 템플릿 파일은 해당 일치 옵션을 선택한 경우에만 포함됩니다.
- `rustLibraries`은 다중 선택입니다. `none`을 전달하거나 원하지 않는 라이브러리를 생략하세요.
- AI 명령 파일은 `--ai-docs none`을 포함하여 `--ai-docs`을 통해 계속 사용할 수 있습니다.
