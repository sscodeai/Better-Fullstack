title: 생태계

Better Fullstack은 여러 생태계에 하나의 CLI을 사용합니다. 각 생태계에는 고유한 비계 표면, 도구 체인 및 호환성 규칙이 있습니다.

다양한 언어로 구성된 제품 스택의 경우 [다중 생태계 모드](/docs/ecosystems/multi-ecosystem/)를 사용하세요. 프로젝트를 `frontend:typescript:next`, `backend:go:gin`, `backend.orm:go:gorm` 및 `database:universal:postgres`과 같은 스택 부분으로 모델링합니다.

| 생태계 | 다음에 가장 적합 | 메모 |
| --- | --- | --- |
| [TypeScript](/docs/ecosystems/typescript/) | 풀스택 웹, APIs, 작업자, 데스크탑 및 최대 웹 통합 매트릭스. | 웹 옵션은 TypeScript 우선입니다. |
| [React Native](/docs/ecosystems/react-native/) | Expo 탐색, UI, 저장, 테스트, 푸시, OTA 및 딥 링크를 갖춘 모바일 앱. | TypeScript 웹 프런트엔드 카테고리 대신 React Native 생태계 표면을 사용합니다. |
| [러스트](/docs/ecosystems/rust/) | 백엔드 서비스, CLI 앱, GraphQL/gRPC, Rust 웹 프런트엔드 및 강력한 형식의 라이브러리. | 화물을 사용합니다. 템플릿은 선택된 Rust 옵션에 따라 조건부로 적용됩니다. |
| [파이썬](/docs/ecosystems/python/) | 프레임워크, ORM, 유효성 검사, 대기열, GraphQL 및 품질 선택을 갖춘 API 및 AI 중심 서비스입니다. | `uv`을 사용합니다. Python AI은 별도의 다중 선택 범주입니다. |
| [이동](/docs/ecosystems/go/) | API 서비스, gRPC, CLIs, 로깅, ORM 및 Go 인증 도우미. | Go 모듈을 사용합니다. GoBetterAuth는 글로벌 `--auth go-better-auth`로 선택되었습니다. |
| [자바](/docs/ecosystems/java/) | Spring Boot, Quarkus 또는 Maven/Gradle, 지속성, 인증, 라이브러리 및 테스트가 포함된 일반 Java 프로젝트. | Java는 빌드 도구를 선택할 때 Maven 또는 Gradle 래퍼를 사용합니다. |
| [엘릭서](/docs/ecosystems/elixir/) | Ecto, Absinthe, 실시간, 작업, 테스트, 품질 및 배포 선택 기능을 갖춘 Phoenix 및 Phoenix LiveView 앱입니다. | 믹스를 사용합니다. 템플릿이 따라잡을 때까지 몇 가지 스키마 수준 옵션은 비활성화된 상태로 유지됩니다. |
| [.NET](/docs/ecosystems/dotnet/) | EF Core 또는 Dapper, Identity, Minimal APIs 또는 Hot Chocolate, SignalR, 작업 및 테스트를 포함하는 ASP.NET Core 서비스입니다. | .NET SDK를 사용합니다. 테스트 및 관찰 가능성은 다중 선택 범주입니다. |

## 기본 CLI 기준선

기본값을 수락하면 TypeScript 웹 프로젝트가 생성됩니다. 패키지 관리자는 CLI을 시작하는 데 사용되는 패키지 관리자를 따릅니다.

```json
{
  "ecosystem": "typescript",
  "frontend": ["tanstack-router"],
  "backend": "hono",
  "runtime": "bun",
  "database": "sqlite",
  "orm": "drizzle",
  "auth": "better-auth",
  "api": "trpc",
  "cssFramework": "tailwind",
  "uiLibrary": "shadcn-ui",
  "testing": "vitest",
  "addons": ["turborepo"],
  "versionChannel": "stable",
  "aiDocs": ["claude-md"]
}
```

런타임과 패키지 관리자는 별도의 선택입니다. 예를 들어 생성된 TypeScript 프로젝트는 설치에 npm, pnpm, Yarn 또는 Bun을 사용하고 애플리케이션 런타임으로 Bun를 선택할 수 있습니다.

## 공유 옵션은 보편적이지 않습니다

일부 플래그는 하나의 프로젝트 구성에 저장되기 때문에 전역적이지만 모든 플래그가 모든 생태계에 적용되는 것은 아닙니다. 생성되는 내용은 아래 생태계 페이지를 이용하시고, 정확한 CLI 값은 [참조 옵션](/docs/reference/options/)을 이용하세요.
