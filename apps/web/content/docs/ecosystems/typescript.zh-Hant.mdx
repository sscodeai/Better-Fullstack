TypeScript 是 Better Fullstack 最廣的生態。可用於 web apps、APIs、workers、desktop shells、瀏覽器擴充功能、文件網站、service integrations，以及大多數 addon workflows。

React Native/Expo mobile scaffolds 位於專門的 [React Native](/docs/ecosystems/react-native/) 生態。

## 前置條件

- Node.js 20+。
- npm、pnpm、Yarn 或 Bun。
- 如果想初始化倉庫，需要 Git。

## 腳本化範例

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --db-setup none \
  --api orpc \
  --auth better-auth \
  --addons turborepo \
  --examples none \
  --ai-docs none \
  --version-channel stable \
  --no-install
```

## 相容性說明

- tRPC 面向基於 React 的 frontends。
- `self` backend 用於 fullstack frameworks，並與 `runtime none` 搭配。
- Qwik、Angular、Redwood 和 Fresh 的 API/backend 支援比完整選項表更窄。
- Workers runtime 支援取決於 backend。
- UI libraries 通常依賴 frontend family 和 CSS framework。例如 shadcn/ui 面向 React，並基於 Tailwind。
- 有些 providers 有額外約束，例如 Polar 需要 Better Auth 和 web frontend。
- 有些選項會先作為相容或依賴支援暴露，然後每條 template path 再逐步補齊深度。產生大型組合前先執行 compatibility checker 或 Stack Builder。
