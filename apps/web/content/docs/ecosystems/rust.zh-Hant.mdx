Rust 專案使用 Cargo 和語言原生 templates。為 backend services、CLIs、GraphQL/gRPC、WebAssembly frontends、logging、caching 和 auth helpers 選擇 Rust options。

## 前置條件

- Rust toolchain 和 Cargo。
- 如果想初始化倉庫，需要 Git。

## 腳本化範例

```npm
npm create better-fullstack@latest my-rust-api -- \
  --ecosystem rust \
  --rust-web-framework axum \
  --rust-orm sqlx \
  --rust-libraries serde validator \
  --rust-logging tracing \
  --rust-error-handling anyhow-thiserror \
  --rust-api none \
  --rust-frontend none \
  --rust-cli none \
  --rust-caching none \
  --rust-auth none \
  --rust-realtime tokio-tungstenite \
  --rust-message-queue lapin \
  --rust-observability opentelemetry \
  --rust-templating askama \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## 生成行為

- 啟用 install 時，`cargo build` 是安裝/驗證路徑。
- Leptos、Dioxus、Tonic、Clap 和 Ratatui template files 只在選擇對應選項時包含。
- `rustLibraries` 是 multi-select；傳 `none` 或省略不想要的 libraries。
- AI instruction files 仍可透過 `--ai-docs` 使用，包含 `--ai-docs none`。
