Go 프로젝트는 Go 모듈과 Go-native 템플릿을 사용합니다. HTTP 서비스, gRPC, CLIs, 로깅, ORM 및 Go 인증 도우미에 대한 Go 옵션을 선택합니다.

## 전제조건

- 툴체인으로 이동하세요.
- 저장소 초기화를 원하는 경우 Git.

## 스크립트 예시

```npm
npm create better-fullstack@latest my-go-api -- \
  --ecosystem go \
  --go-web-framework gin \
  --go-orm gorm \
  --go-logging zap \
  --go-auth jwt \
  --go-api none \
  --go-cli none \
  --go-testing testify \
  --go-realtime gorilla-websocket \
  --go-message-queue nats \
  --go-caching redis \
  --go-config viper \
  --go-observability opentelemetry \
  --auth none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## 인증 분할

Go에는 두 가지 인증 관련 필드가 있습니다.

- `--go-auth casbin|jwt|none`은 Go-native 인증 도우미를 추가합니다.
- `--auth go-better-auth`은 전역 인증 필드를 통해 GoBetterAuth를 선택합니다.

Clerk, NextAuth, Stack Auth, Supabase Auth 및 Auth0과 같은 다른 전역 인증 제공자는 TypeScript 지향적이므로 일반적인 Go 스캐폴드 지원으로 제시되어서는 안 됩니다.

## 생성된 동작

- `go mod tidy`은 설치가 활성화된 경우의 설치/확인 경로입니다.
- gRPC, Cobra 및 Bubble Tea 파일은 선택한 경우에만 포함됩니다.
- 테스트, 실시간, 메시지 대기열, 캐싱, 구성 및 관찰 가능성 선택이 생성됩니다.
`internal/` 아래의 독립형 패키지(`testify`/`gomock`에 대한 예제 테스트 포함).
- `gqlgen` 비계 `graph/schema.graphqls` + `gqlgen.yml`; `go generate ./graph`을 실행하여 생성
GraphQL 서버 코드(Ent 코드 생성 흐름 미러링)
- `goth`은 `GOOGLE_CLIENT_ID`/`GITHUB_CLIENT_ID` 스타일 환경 자격 증명에서 OAuth 공급자를 등록합니다.
- AI 명령 파일은 `--ai-docs none`을 포함하여 `--ai-docs`을 통해 계속 사용할 수 있습니다.
