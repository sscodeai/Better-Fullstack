---
title: マルチエコシステム
description: エコシステム全体のフロントエンド、バックエンド、データベース、モバイル部分から 1 つの Better Fullstack プロジェクトを構成します。
---

マルチエコシステム モードでは、1 つのフラットなエコシステム選択ではなく、接続されたスタック部分としてプロジェクトをモデル化します。 TypeScript フロントエンド、Go API、共有 PostgreSQL データベース、React Native モバイル アプリなど、製品が複数の言語にまたがる場合に使用します。

[Stack Builder](https://better-fullstack.dev/new?mode=multi) で開くか、繰り返しの `--part` フラグを CLI に渡します。

## スタックパーツ

各パーツは次の形状を使用します。

```txt
role:ecosystem:tool
owner.role:ecosystem:tool
```

最上位のロールは、生成されたアプリまたは共有インフラストラクチャを記述します。所有されているロールは、親パーツに機能を付加します。

| 例 | 意味 |
| --- | --- |
| `frontend:typescript:next` | TypeScript Next.js フロントエンドを生成します。 |
| `backend:go:gin` | Go Gin バックエンドを生成します。 |
| `backend.orm:go:gorm` | 選択したバックエンドに GORM を接続します。 |
| `backend.api:rust:tonic` | Tonic gRPC を Rust バックエンドに接続します。 |
| `database:universal:postgres` | 共有 PostgreSQL データベース パーツを追加します。 |
| `mobile:react-native:native-bare` | React Native モバイル アプリを生成します。 |

## スクリプト化された例

```npm
npm create better-fullstack@latest my-mixed-app -- \
  --part frontend:typescript:next \
  --part backend:go:gin \
  --part backend.orm:go:gorm \
  --part backend.auth:go:jwt \
  --part database:universal:postgres \
  --part mobile:react-native:native-bare \
  --addons turborepo docker-compose
```

## 真実の情報源

マルチエコシステム プロジェクトは、`stackParts` から `bts.jsonc` を書き込みます。これらの部分は、生成された構造、履歴、および再現可能なコマンドの信頼できる情報源です。 `backend`、`orm`、`frontend` などのレガシー フィールドは、グラフから派生した互換性フォールバックとして引き続き表示される場合があります。

生成されたコマンドは、繰り返しの `--part` フラグとしてグラフを保存するため、非表示のビルダー状態に依存せずにプロジェクトを再作成できます。

## ビルダーフロー

Stack Builder のマルチエコシステム モードの手順は次のとおりです。

1. フロントエンド - TypeScript Web フロントエンド。
2. バックエンド - TypeScript、Rust、Python、Go、Java、Elixir、または .NET バックエンドと所有するバックエンド機能。
3. データベース - 共有ユニバーサル データベースの選択肢。
4. モバイル - React Native アプリの選択肢。
5. 最終処理 - アドオン、AI ドキュメント、インストール、Git、およびバージョン設定。

## 互換性

- 現在のグラフ モデルでは、ロールごとに選択された 1 つのプライマリ パーツのみがサポートされます。
- 所有される機能部分は、`backend.orm:go:gorm` などの所有者を参照する必要があります。
- 通常、機能エコシステムはその所有者のエコシステムと一致する必要があります。
- ユニバーサル データベース パーツはエコシステム間で共有できます。
- 一部のツールは暗黙の部分を提供します。たとえば、選択されたバックエンドは、生成された `stackParts` に表示されるデータベースまたはルーティングの仮定を提供する場合があります。
- 互換性チェッカーは、生成前にグラフ パーツを検証し、所有者、重複、およびサポートされていないツールの問題を報告します。
