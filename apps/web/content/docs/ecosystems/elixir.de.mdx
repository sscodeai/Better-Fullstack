Elixir-Projekte generieren Phoenix-Anwendungen, die den Mix- und Phoenix-Konventionen folgen. Wählen Sie Elixir-Optionen für LiveView, Ecto SQL, Authentifizierung, APIs, Echtzeit, Jobs, Tests, Qualität und Bereitstellungsdateien.

## Voraussetzungen

- Elixir und Erlang/OTP kompatibel mit Phoenix 1.7.
- PostgreSQL bei Verwendung von `ecto-sql`, `phx-gen-auth` oder `oban`.
- Git, wenn Sie eine Repository-Initialisierung wünschen.

## Skriptbeispiel

```npm
npm create better-fullstack@latest my-phoenix-app -- \
  --ecosystem elixir \
  --elixir-web-framework phoenix-live-view \
  --elixir-orm ecto-sql \
  --elixir-auth phx-gen-auth \
  --elixir-api absinthe \
  --elixir-realtime presence \
  --elixir-jobs oban \
  --elixir-http req \
  --elixir-json jason \
  --elixir-observability telemetry \
  --elixir-testing ex_unit \
  --elixir-quality credo \
  --elixir-deploy docker \
  --ai-docs none \
  --no-install \
  --no-git
```

## Kompatibilitätshinweise

- Elixieroptionen gelten nur mit `--ecosystem elixir`.
- Phoenix und Phoenix LiveView sind eigenständige Elixir-Webframeworks; TypeScript Frontend- und Backend-Optionen werden ignoriert.
- `phx-gen-auth` und Oban erfordern Ecto SQL, da der generierte Code ein Repo und Migrationen benötigt.
- Oban zielt auf PostgreSQL-unterstützte Ecto SQL-Projekte ab.
- LiveView-Streams erfordern `phoenix-live-view`.
- Einige Auswahlmöglichkeiten auf Schemaebene werden vom Diagrammvalidator absichtlich abgelehnt, bis passende Vorlagen vorhanden sind: `ecto`, `ueberauth`, `guardian`, `nimble-options`, `nebulex`, `opentelemetry`, `prom_ex`, `mox`, `bypass`, `wallaby`, `fly` und `gigalixir`. Der CLI und der Builder zeigen diese als Meldungen über nicht unterstützte Tools an, anstatt einen Teil-Phoenix-Code zu generieren.

## Generiertes Verhalten

- Das Projekt umfasst `mix.exs`, Phoenix-Konfiguration, Router, Endpunkt, Controller, Tests, `.env.example` und Einrichtungshinweise.
- Zu den Ecto SQL-Auswahlmöglichkeiten gehören ein Repo, Migrationen, Schema-/Kontextcode und eine PostgreSQL-Konfiguration.
- Optionale Auswahlmöglichkeiten fügen funktionierende Phoenix-Dateien für LiveView, Channels/Presence, Oban- oder Quantum-Jobs, Absinthe-Schema/Resolver, Req/Finch-HTTP-Clients, Swoosh-Mailer, Cachex, Dockerfile und Release-Konfiguration hinzu.
