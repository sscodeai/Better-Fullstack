Go 專案使用 Go modules 和 Go 原生範本。為 HTTP 服務、gRPC、CLI、日誌、ORM 和 Go 認證輔助代碼選擇 Go 選項。

## 前要求

- Go 工具鏈。
- 如果要初始化倉庫，需要 Git。

## 腳本化範例

```npm
npm create better-fullstack@latest my-go-api -- \
  --ecosystem go \
  --go-web-framework gin \
  --go-orm gorm \
  --go-logging zap \
  --go-auth jwt \
  --go-api none \
  --go-cli none \
  --go-testing testify \
  --go-realtime gorilla-websocket \
  --go-message-queue nats \
  --go-caching redis \
  --go-config viper \
  --go-observability opentelemetry \
  --auth none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## Go 分類

| 分類 | 值 |
| --- | --- |
| Web 框架 | `gin` `echo` `fiber` `chi` `none` |
| ORM | `gorm` `sqlc` `ent` `none` |
| API | `grpc-go` `gqlgen` `none` |
| CLI | `cobra` `bubbletea` `urfave-cli` `none` |
| 紀錄 | `zap` `zerolog` `slog` `logrus` `none` |
| Go 認證輔助代碼 | `casbin` `jwt` `goth` `none` |
| 測試（多選） | `testify` `gomock` `none` |
| 即時能力 | `gorilla-websocket` `centrifuge` `none` |
| 訊息佇列 | `nats` `watermill` `none` |
| 快取 | `redis` `ristretto` `none` |
| 配置 | `viper` `koanf` `none` |
| 可觀測性 | `opentelemetry` `none` |

## Auth 拆分

Go 有兩個與認證相關的欄位：

- `--go-auth casbin|jwt|none` 新增 Go 原生認證輔助代碼。
- `--auth go-better-auth` 透過全域 auth 欄位選擇 GoBetterAuth。

Clerk、NextAuth、Stack Auth、Supabase Auth 和 Auth0 等其他全域 auth 提供者偏向 TypeScript，不應作為通用 Go 腳手架支援來展示。

## 生成行為

- 啟用安裝時，`go mod tidy` 是安裝/驗證路徑。
- gRPC、Cobra 和 Bubble Tea 檔案僅在被選取時包含。
- 測試、即時能力、訊息佇列、快取、配置和可觀測性選擇會在 `internal/` 下產生獨立套件（並為 `testify`/`gomock` 產生範例測試）。
- `gqlgen` 會建構 `graph/schema.graphqls` + `gqlgen.yml`；執行 `go generate ./graph` 產生 GraphQL 伺服器程式碼（與 Ent 的程式碼產生流程一致）。
- `goth` 會從 `GOOGLE_CLIENT_ID`/`GITHUB_CLIENT_ID` 形式的環境憑證註冊 OAuth 提供者。
- AI 指令檔仍可透過 `--ai-docs` 使用，包括 `--ai-docs none`。
