TypeScript est l'écosystème Better Fullstack le plus vaste. Utilisez-le pour les applications Web, les API, les travailleurs, les shells de bureau, les extensions de navigateur, les sites de documentation, les intégrations de services et la plupart des flux de travail complémentaires.

Les échafaudages mobiles React Native/Expo vivent dans l'écosystème dédié [React Native](/docs/ecosystems/react-native/).

## Conditions préalables

- Node.js 20+.
- npm, pnpm, Yarn ou Bun.
- Git si vous souhaitez initialiser le référentiel.

## Exemple de script

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --db-setup none \
  --api orpc \
  --auth better-auth \
  --addons turborepo \
  --examples none \
  --ai-docs none \
  --version-channel stable \
  --no-install
```

## Remarques sur la compatibilité

- tRPC est destiné aux interfaces basées sur React.
- Le backend `self` est destiné aux frameworks fullstack et s'associe à `runtime none`.
- Qwik, Angular, Redwood et Fresh ont une prise en charge API/backend plus étroite que le tableau d'options complet.
- La prise en charge du runtime des Workers dépend du backend.
- Les bibliothèques d'interface utilisateur dépendent souvent de la famille frontend et du framework CSS. Par exemple, shadcn/ui est orienté React et basé sur Tailwind.
- Certains fournisseurs ont des contraintes supplémentaires, comme Polar nécessitant Better Auth et une interface Web.
- Certaines options sont présentées pour la compatibilité ou la prise en charge des dépendances avant que chaque chemin de modèle ne soit également profond. Exécutez le vérificateur de compatibilité ou Stack Builder avant d’échafauder de grandes combinaisons.
