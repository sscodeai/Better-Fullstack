Java プロジェクトは、Spring Boot、Quarkus、およびプレーン Java テンプレートをサポートします。ビルド ツール、永続性、認証、アプリケーション ライブラリ、およびテスト ライブラリの Java オプションを選択します。

## 前提条件

- ジャワ21。
- リポジトリの初期化が必要な場合は Git を使用します。

## スクリプト化された例

```npm
npm create better-fullstack@latest my-java-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool maven \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-api spring-graphql \
  --java-logging logback \
  --java-libraries spring-actuator spring-validation flyway \
  --java-testing-libraries junit5 mockito testcontainers \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## Javaのカテゴリ

| カテゴリ | 価値観 |
| --- | --- |
| Webフレームワーク | `spring-boot` `quarkus` `none` |
| ビルドツール | `maven` `gradle` `none` |
| ORM | `spring-data-jpa` `jooq` `mybatis` `none` |
| 認証 | `spring-security` `keycloak` `none` |
| API | `spring-graphql` `none` |
| ロギング | `logback` `none` |
| 図書館 | `spring-actuator` `spring-validation` `flyway` `liquibase` `springdoc-openapi` `lombok` `mapstruct` `caffeine` `resilience4j` `spring-webflux` `spring-batch` `spring-kafka` `spring-mail` `spring-devtools` `micrometer-prometheus` `thymeleaf` `spring-amqp` `opentelemetry-java` `none` |
| ライブラリのテスト | `junit5` `mockito` `testcontainers` `assertj` `rest-assured` `wiremock` `awaitility` `archunit` `jqwik` `none` |

## 互換性に関する注意事項

- Spring Boot と Quarkus には Maven または Gradle が必要です。
- Java ORM、認証、および Spring 固有のライブラリには、Spring Boot に加えて Maven または Gradle が必要です。
- Flyway には Spring データ JPA が必要です。
- 一部のライブラリはフレームワーク固有です。たとえば、Spring Batch、Spring Kafka、Spring Mail、Spring WebFlux、Springdoc OpenAPI、Actuator、および Spring Security は、Spring Boot プロジェクトをターゲットとします。

## 生成された動作

- Maven プロジェクトは、インストールが有効になっている場合、検証に `./mvnw test` を使用します。
- Gradle プロジェクトは、インストールが有効になっている場合、検証に `./gradlew test` を使用します。
- プレーン Java の動作は、フレームワーク依存のカテゴリの `--java-web-framework none` および互換性のある `none` 値を使用して選択されます。
- AI 命令ファイルは、`--ai-docs none` を含め、`--ai-docs` を通じて引き続き利用できます。
