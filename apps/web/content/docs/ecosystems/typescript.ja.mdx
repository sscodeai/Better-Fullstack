TypeScript は、最も広範な Better Fullstack エコシステムです。 Web アプリ、API、ワーカー、デスクトップ シェル、ブラウザ拡張機能、ドキュメント サイト、サービス統合、およびほとんどのアドオン ワークフローに使用します。

React Native/Expo モバイル足場は、専用の [React Native](/docs/ecosystems/react-native/) エコシステム内に存在します。

## 前提条件

- Node.js 20 歳以上。
- npm、pnpm、Yarn、または Bun。
- リポジトリの初期化が必要な場合は Git を使用します。

## スクリプト化された例

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --db-setup none \
  --api orpc \
  --auth better-auth \
  --addons turborepo \
  --examples none \
  --ai-docs none \
  --version-channel stable \
  --no-install
```

## 互換性に関する注意事項

- tRPC は React ベースのフロントエンド用です。
- `self` バックエンドはフルスタック フレームワーク用であり、`runtime none` とペアになります。
- Qwik、Angular、Redwood、および Fresh では、完全なオプション テーブルよりも狭い API/バックエンド サポートが提供されます。
- ワーカー ランタイムのサポートはバックエンドに依存します。
- UI ライブラリは多くの場合、フロントエンド ファミリと CSS フレームワークに依存します。たとえば、shadcn/ui は React 指向であり、Tailwind ベースです。
- 一部のプロバイダーには追加の制約があります (Polar では Better Auth と Web フロントエンドが必要です)。
- 一部のオプションは、すべてのテンプレート パスが同じ深さになる前に、互換性または依存関係のサポートのために表示されます。大きな組み合わせをスキャフォールディングする前に、互換性チェッカーまたは Stack Builder を実行します。
