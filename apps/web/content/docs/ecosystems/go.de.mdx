Go-Projekte verwenden Go-Module und Go-native-Vorlagen. Wählen Sie Go-Optionen für HTTP-Dienste, gRPC, CLIs, Protokollierung, ORMs und Go-Authentifizierungshilfen.

## Voraussetzungen

- Gehen Sie zur Toolchain.
- Git, wenn Sie eine Repository-Initialisierung wünschen.

## Skriptbeispiel

```npm
npm create better-fullstack@latest my-go-api -- \
  --ecosystem go \
  --go-web-framework gin \
  --go-orm gorm \
  --go-logging zap \
  --go-auth jwt \
  --go-api none \
  --go-cli none \
  --go-testing testify \
  --go-realtime gorilla-websocket \
  --go-message-queue nats \
  --go-caching redis \
  --go-config viper \
  --go-observability opentelemetry \
  --auth none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## Go-Kategorien

| Kategorie | Werte |
| --- | --- |
| Web-Framework | `gin` `echo` `fiber` `chi` `none` |
| ORM | `gorm` `sqlc` `ent` `none` |
| API | `grpc-go` `gqlgen` `none` |
| CLI | `cobra` `bubbletea` `urfave-cli` `none` |
| Protokollierung | `zap` `zerolog` `slog` `logrus` `none` |
| Go-Authentifizierungshelfer | `casbin` `jwt` `goth` `none` |
| Testen (Mehrfachauswahl) | `testify` `gomock` `none` |
| Echtzeit | `gorilla-websocket` `centrifuge` `none` |
| Nachrichtenwarteschlange | `nats` `watermill` `none` |
| Caching | `redis` `ristretto` `none` |
| Konfig | `viper` `koanf` `none` |
| Beobachtbarkeit | `opentelemetry` `none` |

## Authentifizierungsaufteilung

Go verfügt über zwei authentisierungsbezogene Felder:

- `--go-auth casbin|jwt|none` fügt Go-native-Authentifizierungshelfer hinzu.
- `--auth go-better-auth` wählt GoBetterAuth über das globale Authentifizierungsfeld aus.

Andere globale Authentifizierungsanbieter wie Clerk, NextAuth, Stack Auth, Supabase Auth und Auth0 sind TypeScript-orientiert und sollten nicht als allgemeine Go-Gerüstunterstützung dargestellt werden.

## Generiertes Verhalten

- `go mod tidy` ist der Installations-/Überprüfungspfad, wenn die Installation aktiviert ist.
- gRPC-, Cobra- und Bubble Tea-Dateien werden nur einbezogen, wenn sie ausgewählt sind.
- Test-, Echtzeit-, Nachrichtenwarteschlangen-, Caching-, Konfigurations- und Beobachtbarkeitsauswahlmöglichkeiten werden generiert
eigenständige Pakete unter `internal/` (plus Beispieltests für `testify`/`gomock`).
- `gqlgen` Gerüste `graph/schema.graphqls` + `gqlgen.yml`; Führen Sie `go generate ./graph` aus, um zu produzieren
der GraphQL-Servercode (spiegelt den Ent-Codegenerierungsablauf wider).
- `goth` registriert OAuth-Anbieter aus `GOOGLE_CLIENT_ID`/`GITHUB_CLIENT_ID`-Env-Anmeldeinformationen.
- AI-Anweisungsdateien sind weiterhin über `--ai-docs` verfügbar, einschließlich `--ai-docs none`.
