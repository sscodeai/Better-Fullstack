Elixir プロジェクトは、Mix および Phoenix の規則に従った Phoenix アプリケーションを生成します。 LiveView、Ecto SQL、認証、APIs、リアルタイム、ジョブ、テスト、品質、およびデプロイファイルの Elixir オプションを選択します。

## 前提条件

- Elixir および Erlang/OTP は Phoenix 1.7 と互換性があります。
- PostgreSQL (`ecto-sql`、`phx-gen-auth`、または `oban` を使用する場合)。
- リポジトリの初期化が必要な場合は Git を使用します。

## スクリプト化された例

```npm
npm create better-fullstack@latest my-phoenix-app -- \
  --ecosystem elixir \
  --elixir-web-framework phoenix-live-view \
  --elixir-orm ecto-sql \
  --elixir-auth phx-gen-auth \
  --elixir-api absinthe \
  --elixir-realtime presence \
  --elixir-jobs oban \
  --elixir-http req \
  --elixir-json jason \
  --elixir-observability telemetry \
  --elixir-testing ex_unit \
  --elixir-quality credo \
  --elixir-deploy docker \
  --ai-docs none \
  --no-install \
  --no-git
```

## 互換性に関する注意事項

- Elixir オプションは `--ecosystem elixir` にのみ適用されます。
- Phoenix と Phoenix LiveView はスタンドアロンの Elixir Web フレームワークです。 TypeScript フロントエンドおよびバックエンドのオプションは無視されます。
- `phx-gen-auth` と Oban では、生成されたコードにはリポジトリと移行が必要なため、Ecto SQL が必要です。
- Oban は、PostgreSQL が支援する Ecto SQL プロジェクトをターゲットとしています。
- LiveView ストリームには `phoenix-live-view` が必要です。
- いくつかのスキーマ レベルの選択は、一致するテンプレートが存在するまで、グラフ バリデーターによって意図的に拒否されます: `ecto`、`ueberauth`、`guardian`、`nimble-options`、`nebulex`、`opentelemetry`、`prom_ex`、`mox`、`bypass`、 `wallaby`、`fly`、および `gigalixir`。 CLI とビルダーは、部分的な Phoenix コードを生成するのではなく、サポートされていないツールのメッセージとしてこれらを表示します。

## 生成された動作

- プロジェクトには、`mix.exs`、Phoenix 構成、ルーター、エンドポイント、コントローラー、テスト、`.env.example`、およびセットアップ ノートが含まれています。
- Ecto SQL の選択には、リポジトリ、移行、スキーマ/コンテキスト コード、PostgreSQL 構成が含まれます。
- オプションの選択により、LiveView、Channels/Presence、Oban または Quantum ジョブ、Absinthe スキーマ/リゾルバー、Req/Finch HTTP クライアント、Swoosh メーラー、Cachex、Dockerfile、およびリリース構成用の動作する Phoenix ファイルが追加されます。
