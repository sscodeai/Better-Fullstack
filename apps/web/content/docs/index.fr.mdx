Better Fullstack est un générateur de projets pour les piles de produits typés. Choisissez le frontend, le backend, la couche de données, les services, les outils et l'écosystème cible ; le CLI écrit les fichiers correspondants et enregistre la pile dans `bts.jsonc`.

Utilisez le CLI interactif lorsque vous souhaitez une configuration guidée, le [Stack Builder](https://better-fullstack.dev/new) lorsque vous souhaitez comparer les options visuellement et des indicateurs explicites lorsque vous avez besoin d'une commande reproductible pour CI, Docs ou un autre développeur.

## Exigences

Le CLI s'exécute sur Node.js 20 ou version ultérieure.

```bash
node --version
```

Les projets générés peuvent également avoir besoin de leur chaîne d'outils d'écosystème : les outils Expo pour React Native, Cargo pour Rust, `uv` pour Python, Go, Java 21 ou Elixir/Erlang pour les projets Phoenix.

## Créez votre premier projet

Exécutez l'assistant et répondez aux invites :

```npm
npm create better-fullstack@latest my-app
```

Pour npm, les indicateurs Better Fullstack suivent le séparateur `--`. Les onglets rendus du gestionnaire de packages affichent les commandes pnpm, Bun et Yarn correspondantes pour lesquelles le séparateur n'est pas nécessaire.

## Ce que créent les valeurs par défaut

L'acceptation des valeurs par défaut crée un projet TypeScript avec :

- Frontend TanStack Router et backend Hono.
- Exécution Bun, SQLite, Drizzle, Better Auth et tRPC.
- Tailwind CSS, shadcn/ui, Vitest et Turborepo.
- Le gestionnaire de packages utilisé pour lancer CLI, les versions de dépendances stables, Git, l'installation activée et `CLAUDE.md`.

Vous pouvez conserver cette pile, partir d’un modèle ou changer complètement d’écosystème.

## Façons de choisir une pile

| Chemin | Utilisez-le quand |
| --- | --- |
| Assistant CLI | Vous voulez des invites et des valeurs par défaut raisonnables. |
| Stack Builder | Vous souhaitez explorer des combinaisons compatibles avant l’échafaudage. |
| Indicateurs explicites | Vous souhaitez une sortie reproductible dans des scripts, des documents ou des rapports de problèmes. |
| Compétence d'agent | Vous souhaitez qu’un agent de codage utilise le flux de travail CLI sans écrire manuellement les fichiers de démarrage. |
| Serveur MCP | Vous souhaitez qu'un agent de codage AI inspecte le schéma, valide la compatibilité et effectue un essai à sec en toute sécurité. |

## Contexte de projet généré

Chaque projet généré comprend :

- `bts.jsonc` avec la pile sélectionnée.
- Une commande reproductible imprimée par le CLI.
- Fichiers de framework, de package et de configuration pour l'écosystème sélectionné.
- Fichiers d'instructions AI facultatifs tels que `CLAUDE.md`, `Agents.md` ou `.cursorrules`.
