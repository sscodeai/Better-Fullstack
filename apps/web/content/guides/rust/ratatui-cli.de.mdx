Verwenden Sie diesen Stack, wenn Sie eine Terminal-Benutzeroberfläche anstelle eines Webdienstes erstellen möchten.

```npm
npm create better-fullstack@latest my-rust-cli -- \
  --ecosystem rust \
  --rust-web-framework none \
  --rust-frontend none \
  --rust-orm none \
  --rust-api none \
  --rust-cli ratatui \
  --rust-logging none
```

## Was dadurch entsteht

- Ein Rust CLI-Projekt.
- Ratatui als Terminal-UI-Option.
- Kein Webframework, Frontend, ORM oder API-Layer.

## Erzeugte Form

Dieser Stack erstellt eine Terminalanwendung und keinen Server. Ratatui kümmert sich um das Rendern von Widgets im Terminal, während Ihr App-Code für Status, Ereignisse und Befehlsverhalten zuständig ist.

Das nützliche mentale Modell ist eine Schleife:

- Lesen Sie Tastatur- oder Terminalereignisse.
- Anwendungsstatus aktualisieren.
- Rendern Sie den aktuellen Status in Ratatui-Widgets.
- Sauber beenden, wenn der Benutzer zum Beenden auffordert.

## Repräsentativer Dateibaum

```txt
my-rust-cli/
  Cargo.toml
  src/
    main.rs
    app.rs
    event.rs
    ui.rs
```

Konzentrieren Sie sich bei `app.rs` auf Zustandsübergänge, bei `event.rs` auf die Eingabeverarbeitung und bei `ui.rs` auf das Zeichnen. Diese Aufteilung erleichtert das Testen des Terminalverhaltens.

## CLI und UI-Beispiele

Ein kleiner App-Statustyp sorgt dafür, dass das Rendern der Benutzeroberfläche deterministisch bleibt:

```rust
#[derive(Debug, Default)]
pub struct App {
    pub should_quit: bool,
    pub selected_index: usize,
    pub items: Vec<String>,
}

impl App {
    pub fn quit(&mut self) {
        self.should_quit = true;
    }

    pub fn next(&mut self) {
        if !self.items.is_empty() {
            self.selected_index = (self.selected_index + 1) % self.items.len();
        }
    }
}
```

Die Zeichenfunktion sollte den Status empfangen und rendern, ohne das Anwendungsverhalten zu verändern:

```rust
use ratatui::{
    layout::{Constraint, Direction, Layout},
    widgets::{Block, Borders, List, ListItem, Paragraph},
    Frame,
};

use crate::app::App;

pub fn draw(frame: &mut Frame, app: &App) {
    let chunks = Layout::default()
        .direction(Direction::Vertical)
        .constraints([Constraint::Min(1), Constraint::Length(1)])
        .split(frame.area());

    let items = app.items.iter().map(|item| ListItem::new(item.as_str()));
    let list = List::new(items).block(Block::default().title("Tasks").borders(Borders::ALL));
    let help = Paragraph::new("q: quit  j/k: move");

    frame.render_widget(list, chunks[0]);
    frame.render_widget(help, chunks[1]);
}
```

Die Ereignisbehandlung kann klein und explizit bleiben:

```rust
use crossterm::event::{KeyCode, KeyEvent};

use crate::app::App;

pub fn handle_key(app: &mut App, key: KeyEvent) {
    match key.code {
        KeyCode::Char('q') => app.quit(),
        KeyCode::Char('j') | KeyCode::Down => app.next(),
        _ => {}
    }
}
```

## Kompatibilitätshinweise

CLI-Projekte sollten unabhängige Webframework- und Datenbankoptionen vermeiden, es sei denn, Sie benötigen diese ausdrücklich. In dieser Anleitung bleiben diese Flags auf `none` gesetzt.

Belassen Sie `--rust-cli ratatui` und die web/API/database-Flags für ein Terminal-First-Projekt auf `none`. Wenn Sie später ein Backend oder eine Datenbank hinzufügen, behandeln Sie diese als Hybrid-App und führen Sie klare Modulgrenzen ein, anstatt Netzwerkaufrufe in den Rendering-Code zu mischen.

## Wann man es wählt

Wählen Sie Ratatui für Dashboards, Entwicklertools, Infrastrukturtools und Terminal-First-Workflows.

Dies ist besonders nützlich, wenn Benutzer den größten Teil ihres Tages in einer Shell verbringen und eine fokussierte Schnittstelle für wiederholte Vorgänge benötigen.

## Test- und Bereitstellungshinweise

Die meisten Ratatui-Logiken lassen sich am einfachsten unterhalb der Terminalschicht testen. Unit-Test-Zustandsübergänge und Befehlsanalyse direkt:

```bash
cargo test
```

Verwenden Sie für Release-Builds den normalen optimierten Build von Cargo:

```bash
cargo build --release
```

Dokumentieren Sie beim Verteilen der Binärdatei die Terminalerwartungen wie Farbunterstützung, Tastaturkürzel, Speicherorte der Konfigurationsdateien und alle Umgebungsvariablen, die das Tool liest.

## Fehlerbehebung

- Wenn das Terminal nach einem Absturz in einem ungewöhnlichen Modus bleibt, stellen Sie sicher, dass die App den Rohmodus wiederherstellt und beim Herunterfahren den alternativen Bildschirm verlässt.
- Wenn das Rendern flackert, überprüfen Sie, ob die Ereignisschleife ohne Eingaben oder Ticks nicht aggressiv neu gezeichnet wird.
- Wenn Text unerwartet abgeschnitten wird, testen Sie mit schmalen Terminalbreiten und verwenden Sie Ratatui-Layoutbeschränkungen anstelle von fest codierten Koordinaten.
- Wenn Tastaturkürzel in einer Remote-Shell nicht funktionieren, vergleichen Sie das Verhalten des Terminalemulators, bevor Sie die App-Logik ändern.
