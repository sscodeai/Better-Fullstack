Utilisez cette pile lorsque vous souhaitez une application Expo axée sur le mobile et que vous n'avez pas besoin d'un backend généré dans le même échafaudage.

```npm
npm create better-fullstack@latest my-mobile-app -- \
  --ecosystem react-native \
  --frontend native-uniwind \
  --mobile-navigation expo-router \
  --mobile-ui uniwind \
  --mobile-storage mmkv \
  --mobile-testing maestro-react-native-testing-library \
  --mobile-push expo-notifications \
  --mobile-ota expo-updates \
  --mobile-deep-linking expo-linking \
  --ai-docs none \
  --no-install
```

## Ce que cela crée

- Une application native Expo utilisant Uniwind.
- Expo Routeur pour les routes natives basées sur des fichiers.
- Stockage MMKV, bibliothèque de tests React Native, Maestro, notifications push, mises à jour OTA et liens profonds.
- Aucune couche d'interface Web, de backend, de base de données, d'ORM, d'authentification ou API par défaut.
- Une forme de projet axée sur le mobile qui peut être associée ultérieurement à un API.

## Forme générée

Cette commande crée une base frontale native plutôt qu'une application Web fullstack. Il maintient les décisions backend en dehors de l'échafaudage initial afin que l'application mobile puisse ultérieurement pointer vers un API existant ou un service généré séparément.

Forme représentative :

```txt
my-mobile-app/
  bts.jsonc
  app/
  components/
  package.json
  maestro/
```

Les chemins exacts Expo peuvent dépendre de la version du modèle, mais la propriété est simple : les écrans et les composants natifs résident dans l'application, tandis que les problèmes de serveur sont intentionnellement absents.

## Exemple d'écran

Uniwind vous permet d'utiliser des classes de style utilitaire dans les composants React Native :

```tsx
import { Text, View } from "react-native";

export default function HomeScreen() {
  return (
    <View className="flex-1 items-center justify-center bg-white px-6">
      <Text className="text-2xl font-semibold text-slate-950">
        Mobile app
      </Text>
      <Text className="mt-2 text-center text-slate-600">
        Connect an API when the product contract is ready.
      </Text>
    </View>
  );
}
```

Gardez les clients API derrière un petit module lorsque vous ajoutez un backend. Cela facilite l'échange des origines API locales, de préparation et de production sans changer d'écran.

## Remarques sur la compatibilité

React Native est son propre écosystème dans Better Fullstack. Utilisez `--ecosystem react-native` ; ne combinez pas les indicateurs d'interface Web TypeScript avec les indicateurs d'interface Web natifs.

Ce guide laisse intentionnellement la génération du backend, de la base de données, de l'ORM, de l'authentification et de API en dehors de l'échafaudage mobile. L'authentification mobile et les API de serveur nécessitent des décisions spécifiques à la plate-forme qui sont mieux prises une fois que les exigences du produit sont claires.

## Quand le choisir

Choisissez cette option pour les prototypes mobiles, les applications compagnons et les fondations d'applications natives où le API peut être créé séparément.

## Ajout d'un API plus tard

Les étapes suivantes courantes sont un Hono API pour un backend TypeScript léger, une application Web Next.js ou TanStack Start si vous avez également besoin d'une surface Web ou un backend dédié qui existe déjà en dehors de Better Fullstack.

Lors de l'ajout d'un API, décidez comment l'application native gérera :

- URL de base API spécifiques à l'environnement.
- Stockage de la session d'authentification sur l'appareil.
- Actualiser les jetons ou renouveler la session.
- Comportement hors ligne et mises à jour optimistes.
- Profils de build natifs pour le développement, la prévisualisation et la production.

## Notes de déploiement

Le déploiement Expo est différent du déploiement Web. Planifiez les profils de build natifs, les identifiants d’application, les informations d’identification de signature et les variables d’environnement utilisées au moment de la build. Ne mettez pas de secrets de serveur dans le forfait mobile ; les applications natives doivent appeler un backend qui contient des secrets côté serveur.

## Dépannage

- Si les fichiers Web générés sont absents, c'est normal : cette pile utilise une interface native et aucune interface Web.
- Si les routes backend sont absentes, la commande sélectionne explicitement `--backend none`.
- Si le style ne s'applique pas, vérifiez l'installation Uniwind et la configuration de l'analyse de classe dans l'application générée.
- Si les appels API fonctionnent dans le simulateur mais pas sur l'appareil, vérifiez si l'URL de base API pointe vers un hôte accessible depuis l'appareil.
