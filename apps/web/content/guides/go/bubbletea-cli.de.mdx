Verwenden Sie diesen Stack, wenn Sie eine Go-Terminal-Benutzeroberfläche anstelle eines Web-API wünschen.

```npm
npm create better-fullstack@latest my-go-cli -- \
  --ecosystem go \
  --go-web-framework none \
  --go-orm none \
  --go-api none \
  --go-cli bubbletea \
  --go-logging none \
  --go-auth none
```

## Was dadurch entsteht

- Ein Go CLI-Projekt.
- Bubble Tea als Terminal-UI-Option.
- Kein Webframework, ORM, gRPC oder Authentifizierungs-Add-on.

## Erzeugte Form

Dieser Stack erstellt eine interaktive Terminal-App, keinen Server. Bubble Tea organisiert das Programm um ein Modell, Nachrichten, Aktualisierungen und Ansichten herum.

Die Hauptschleife ist einfach:

- Das Modell enthält den aktuellen Anwendungsstatus.
- Meldungen beschreiben Eingaben, Ticks oder Befehlsergebnisse.
- `Update` ändert das Modell und gibt Folgebefehle zurück.
- `View` rendert das aktuelle Modell als Terminaltext.

## Repräsentativer Dateibaum

```txt
my-go-cli/
  go.mod
  cmd/
    app/
      main.go
  internal/
    tui/
      model.go
      update.go
      view.go
```

Behalten Sie nach Möglichkeit Nebenwirkungen in Bubble Tea-Befehlen bei. Dadurch können das Modell und die Ansicht einfacher getestet werden, ohne dass ein echtes Terminal ausgeführt werden muss.

## Modell- und Aktualisierungsbeispiele

Ein Bubble Tea-Modell kann mit gerade genug Status zum Rendern und Reagieren auf Eingaben beginnen:

```go
package tui

type Model struct {
	Items    []string
	Selected int
	Quitting bool
}

func NewModel() Model {
	return Model{
		Items: []string{"Build", "Test", "Deploy"},
	}
}
```

Die Update-Funktion besitzt Zustandsübergänge:

```go
package tui

import tea "github.com/charmbracelet/bubbletea"

func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
	switch msg := msg.(type) {
	case tea.KeyMsg:
		switch msg.String() {
		case "q", "ctrl+c":
			m.Quitting = true
			return m, tea.Quit
		case "j", "down":
			if len(m.Items) > 0 {
				m.Selected = (m.Selected + 1) % len(m.Items)
			}
		case "k", "up":
			if len(m.Items) > 0 {
				m.Selected = (m.Selected - 1 + len(m.Items)) % len(m.Items)
			}
		}
	}

	return m, nil
}
```

Die Ansicht sollte aus dem Status gerendert werden, ohne dass Arbeit ausgeführt wird:

```go
package tui

import "strings"

func (m Model) View() string {
	if m.Quitting {
		return ""
	}

	var b strings.Builder
	b.WriteString("Tasks\n\n")

	for i, item := range m.Items {
		cursor := " "
		if i == m.Selected {
			cursor = ">"
		}
		b.WriteString(cursor + " " + item + "\n")
	}

	b.WriteString("\nq: quit  j/k: move\n")
	return b.String()
}
```

## Kompatibilitätshinweise

Bei CLI-Projekten sollten die Nur-Server-Kategorien auf `none` eingestellt bleiben, es sei denn, es handelt sich absichtlich um Hybridtools.

Behalten Sie `--go-cli bubbletea` mit `--go-web-framework none`, `--go-orm none` und `--go-api none` für eine Terminal-First-App bei. Wenn der CLI später mit einem API kommuniziert, behalten Sie diesen Client in einem separaten Paket und rufen Sie ihn über Befehle statt über `View` auf.

## Wann man es wählt

Wählen Sie Bubble Tea für Terminal-Dashboards, Entwicklertools und Befehlszeilenschnittstellen, bei denen es auf Interaktivität ankommt.

Es eignet sich gut für Tools, die Auswahl, Fortschritt, Formulare oder Live-Status benötigen, ohne eine Browser-Benutzeroberfläche zu erstellen.

## Test- und Bereitstellungshinweise

Unit-Testen Sie die Modellübergänge direkt:

```bash
go test ./...
```

Erstellen Sie für die Verteilung eine einzelne Binärdatei und dokumentieren Sie unterstützte Umgebungsvariablen oder Konfigurationsdateien:

```bash
go build ./cmd/app
```

Wenn das generierte Projekt einen anderen Befehlspfad verwendet, folgen Sie dem Pfad im Verzeichnis `cmd/`.

## Fehlerbehebung

- Wenn Tastendrücke nichts bewirken, überprüfen Sie, ob das fokussierte Terminal die Tastenfolge sendet, die Bubble Tea empfängt.
- Wenn der Bildschirm eine veraltete Ausgabe hinterlässt, überprüfen Sie die Optionen zum Starten und Herunterfahren des Programms auf alternatives Bildschirmverhalten.
- Wenn Tests schwer zu schreiben sind, verschieben Sie Nebenwirkungen in `tea.Cmd`-Funktionen und halten Sie `Update` deterministisch.
- Wenn die Ausgabe schlecht umbrochen wird, testen Sie sie mit schmalen Breiten und sorgen Sie dafür, dass die Formatierung `View` auf den verfügbaren Platz reagiert, wenn das Modell die Abmessungen verfolgt.
