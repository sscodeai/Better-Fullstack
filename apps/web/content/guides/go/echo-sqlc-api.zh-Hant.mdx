當你需要一個使用 Echo、並採用 SQL-first 資料存取方式的 Go API 時，使用這個 stack。

```npm
npm create better-fullstack@latest my-echo-api -- \
  --ecosystem go \
  --go-web-framework echo \
  --go-orm sqlc \
  --go-api none \
  --go-cli none \
  --go-logging zap \
  --go-auth none
```

## 創建內容

- 使用 Echo 的 Go API 項目。
- SQLC 作為 Go 的 ORM/database access 選項。
- Zap 用於日誌記錄。
- 預設不包含 gRPC、CLI 或 Go auth add-on。

## 生成結構

這是一個 SQL-first Go API。 Echo 提供 HTTP layer，SQLC 從 SQL 產生 typed Go method，Zap 負責結構化日誌。

主要設計目標是讓資料庫行為保持可見：

- SQL 檔案定義 query。
- SQLC 產生 Go 類型和 query method。
- Handler 呼叫 service 或 store，而不是直接嵌入 SQL。
- Echo middleware 處理 logging、recovery、CORS 等 request 關注點。

my-echo-api/

把 SQL 放入 version control，並像 review Go code 一樣認真 review query change。 SQLC 讓 Go 側比較安全，但不能取代 schema design。

## Query 與 handler 範例

SQLC query 檔案是產生方法的 source of truth：

```sql
-- name: GetUser :one
select id, email, name, created_at
from users
where id = $1;

-- name: CreateUser :one
insert into users (email, name)
values ($1, $2)
returning id, email, name, created_at;
```

Echo handler 可以 bind JSON，並透過 service 呼叫產生的 query layer：

```go
package handlers

import (
	"net/http"
	"strconv"

	"github.com/labstack/echo/v4"
)

func (h *UserHandler) Get(c echo.Context) error {
	id, err := strconv.ParseInt(c.Param("id"), 10, 64)
	if err != nil {
		return echo.NewHTTPError(http.StatusBadRequest, "invalid user id")
	}

	user, err := h.Users.Get(c.Request().Context(), id)
	if err != nil {
		return echo.NewHTTPError(http.StatusNotFound, "user not found")
	}

	return c.JSON(http.StatusOK, user)
}
```

把 route 註冊放在靠近啟動程式碼的位置，這樣 API surface 比較容易瀏覽：

```go
func RegisterRoutes(e *echo.Echo, users *UserHandler) {
	e.GET("/health", Health)

	api := e.Group("/api")
	api.GET("/users/:id", users.Get)
	api.POST("/users", users.Create)
}
```

## 相容性說明

本指南故意使用 `--go-api none`、`--go-cli none` 和 `--go-auth none`，讓 scaffold 專注於 HTTP API。當你想從 SQL 產生 query code 時，請保留 `--go-orm sqlc`。如果 service boundary 應為 protobuf 而非 JSON，請使用 gRPC guide。

## 何時選擇

當你想要明確 SQL 和產生的 Go types，而不採用更高層 ORM 時，選擇 Echo with SQLC。

## 取捨

SQLC 讓 database behavior 可見，但你需要維護更多 SQL。如果想要更傳統的 ORM flow，請使用 [Gin with PostgreSQL and GORM](/guides/go/gin-postgres-gorm/)。

Echo 很輕量也很靈活，因此專案約定很重要。請一致地安排 request validation、service call 和 error mapping 的位置。

## 測試與部署

為 handler 和 service 執行 Go test，並為重要 SQL query 加上 database-backed integration test。

```bash
go test ./...
```

SQL 改變時，在信任 compiler feedback 前，先用產生項目記錄的指令重新產生 SQLC output。部署時，在啟動 API 前執行 migration，確保 generated query type 與 live schema 相符。

## 故障排查

- 如果產生的 SQLC 程式碼過期，重新執行專案的 SQLC generation command；如果 scaffold 追蹤生成文件，請同時提交 SQL 和產生的 Go 變更。
- 如果 query 能編譯但回傳了意外數據，先檢查 SQL；SQLC 驗證類型，不驗證產品語意。
- 如果 Echo 回傳純文字錯誤，集中處理 HTTP error，讓回應保持一致。
- 如果 `go test ./...` 只在資料庫測試中失敗，請檢查 test database URL 和 migration 狀態。
