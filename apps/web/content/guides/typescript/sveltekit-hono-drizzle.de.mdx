Verwenden Sie diesen Stack, wenn Sie SvelteKit im Frontend und einen kleinen TypeScript API Server daneben haben möchten.

```npm
npm create better-fullstack@latest my-sveltekit-app -- \
  --ecosystem typescript \
  --frontend svelte \
  --backend hono \
  --runtime bun \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api orpc \
  --css-framework tailwind \
```

## Was dadurch entsteht

- Ein SvelteKit-Frontend.
- Ein Hono Backend auf Bun.
- oRPC für typisierte API-Verträge über Nicht-React-Frontend-Optionen hinweg.
- PostgreSQL und Drizzle.
- Better Auth, wo die ausgewählte Datenbank und ORM dies unterstützen.

## Erzeugte Form

Dies ist ein geteilter TypeScript-Stack: SvelteKit kümmert sich um das Frontend-Erlebnis, Hono kümmert sich um den API-Server und oRPC ist die typisierte Grenze zwischen ihnen.

Repräsentative Form:

```txt
my-sveltekit-app/
  bts.jsonc
  apps/
    web/
    server/
  packages/
    shared/
```

Die spezifischen Arbeitsbereichsnamen können sich ändern, die Eigentümerschaft muss jedoch klar bleiben. Svelte-Komponenten sollten Datenbankmodule nicht direkt importieren; Sie sollten typisierte Serveroperationen aufrufen.

## Beispiel für ein SvelteKit-Aufrufmuster

Eine Seite oder Ladefunktion sollte mit dem generierten API-Client kommunizieren, anstatt Anforderungsnutzlasten in der gesamten App fest zu codieren:

```ts
export async function load() {
  return {
    title: "Projects",
  };
}
```

Behalten Sie bei Mutationen die Validierung und Autorisierung auf der Hono-Seite bei, auch wenn die SvelteKit-Benutzeroberfläche bereits Formulareingaben validiert.

## Beispielmodell Drizzle

PostgreSQL plus Drizzle gibt Ihnen explizite Tabellendefinitionen:

```ts
import { pgTable, text, timestamp } from "drizzle-orm/pg-core";

export const project = pgTable("project", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  ownerId: text("owner_id").notNull(),
  createdAt: timestamp("created_at").defaultNow().notNull(),
});
```

Verwenden Sie kleine Serverfunktionen rund um Abfragen, damit Hono-Handler und oRPC-Prozeduren nicht zu einer Mülldeponie für Geschäftsregeln werden.

## Wann man es wählt

Wählen Sie diese Option, wenn Ihr Team SvelteKit bevorzugt, aber dennoch einen expliziten API-Server und typisierte Verträge wünscht. Dies ist nützlich für Apps, die möglicherweise zu separaten Front-End- und Backend-Bereitstellungen werden.

## Kompatibilitätshinweise

- In diesem Handbuch wird `--api orpc` verwendet, da SvelteKit kein React-Frontend ist.
- `--backend hono` und `--runtime bun` erstellen einen separaten API-Dienst, anstatt sich nur auf SvelteKit-Serverrouten zu verlassen.
- Better Auth erfordert, dass die ausgewählte Datenbank und der ORM-Pfad kompatibel bleiben. Halten Sie Authentifizierungsschemaänderungen und Drizzle-Migrationen synchron.
- Wenn Sie später das separate Backend entfernen, gehen Sie noch einmal gemeinsam auf die Client-Einrichtung, CORS und die Authentifizierungs-Cookie-Verwaltung ein.

## Bereitstellungshinweise

Erwarten Sie separate Web- und API-Umgebungsvariablen. Die SvelteKit-App benötigt den öffentlichen API-Ursprung. Der Hono-Server benötigt PostgreSQL und Authentifizierungsgeheimnisse. Wenn sie auf verschiedenen Domänen bereitgestellt werden, konfigurieren Sie CORS und Cookies absichtlich.

Führen Sie Datenbankmigrationen aus, bevor Sie API-Code bereitstellen, der neue Tabellen oder Spalten erwartet.

## Fehlerbehebung

- Wenn Anfragen vom Browser fehlschlagen, aber mit Curl funktionieren, überprüfen Sie CORS und Cookie-Anmeldeinformationen.
- Wenn SvelteKit ohne Daten gerendert wird, überprüfen Sie den API-Ursprung, der vom serverseitigen und browserseitigen Code verwendet wird.
- Wenn die Authentifizierung in allen Domänen inkonsistent ist, überprüfen Sie die Cookie-Domäne, die gleiche Site und die Sicherheitseinstellungen.
- Wenn Drizzle-Schemaänderungen nicht angezeigt werden, bestätigen Sie, dass Migrationen generiert und auf die Zieldatenbank PostgreSQL angewendet wurden.

## Vergleichshinweise

Im Vergleich zu einem Nuxt- oder SolidStart-Self-Backend-Stack gibt Ihnen dieser Leitfaden eine explizitere API-Grenze. Im Vergleich zu [Hono mit tRPC und Drizzle](/guides/typescript/hono-trpc-drizzle/) wird oRPC verwendet, da das Frontend SvelteKit und nicht React ist.
