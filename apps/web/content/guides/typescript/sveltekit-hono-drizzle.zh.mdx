当你想在 frontend 使用 SvelteKit，并在旁边放一个小型 TypeScript API server 时，使用这个 stack。

```npm
npm create better-fullstack@latest my-sveltekit-app -- \
  --ecosystem typescript \
  --frontend svelte \
  --backend hono \
  --runtime bun \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api orpc \
  --css-framework tailwind \
```

## 创建内容

- SvelteKit frontend。
- Bun 上的 Hono backend。
- oRPC，用于非 React frontend 选择之间的 typed API contracts。
- PostgreSQL 和 Drizzle。
- 在所选 database 与 ORM 支持时接入 Better Auth。

## 生成结构

这是一个 split TypeScript stack：SvelteKit 处理 frontend experience，Hono 处理 API server，oRPC 是两者之间的 typed boundary。

代表性结构：

```txt
my-sveltekit-app/
  bts.jsonc
  apps/
    web/
    server/
  packages/
    shared/
```

具体 workspace names 可能变化，但 ownership 要清晰。Svelte components 不应直接导入 database modules；它们应调用 typed server operations。

## 兼容性

- 本指南使用 `--api orpc`，因为 SvelteKit 不是 React frontend。
- `--backend hono` 和 `--runtime bun` 会创建独立 API service，而不是只依赖 SvelteKit server routes。
- Better Auth 要求所选 database 和 ORM path 保持兼容。同步维护 auth schema changes 和 Drizzle migrations。
- 如果之后移除独立 backend，请一起检查 API client setup、CORS 和 auth cookie handling。

## 部署

预期 web 和 API 使用不同环境变量。SvelteKit app 需要 public API origin。Hono server 需要 PostgreSQL 和 auth secrets。如果它们部署在不同域名，请有意配置 CORS 和 cookies。

部署依赖新 tables 或 columns 的 API code 前，先运行 database migrations。
