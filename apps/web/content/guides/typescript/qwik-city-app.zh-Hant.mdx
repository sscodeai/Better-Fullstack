當你想要一個 Qwik app，並且第一天不添加獨立 backend 或 database 時，使用這個 stack。

```npm
npm create better-fullstack@latest my-qwik-app -- \
  --ecosystem typescript \
  --frontend qwik \
  --backend none \
  --runtime none \
  --database none \
  --orm none \
  --auth none \
  --api none \
  --css-framework tailwind \
```

## 創建內容

- 一個 Qwik app。
- Tailwind CSS。
- 預設不包含 database、ORM、auth 或 API layer。
- 一個之後可以繼續擴充的 generated project configuration。

## 生成結構

這篇指南有意從 frontend-only Qwik City app 開始。當你想先搭好 application shell、routing 和 styling，再選擇 backend 或 persistence layer 時，它很有用。

代表性結構：

```txt
my-qwik-app/
  bts.jsonc
  src/
    routes/
    components/
```

產生的 `bts.jsonc` 會記錄 backend、runtime、database、ORM、auth 和 API 都是 `none`。這樣 stack 的最小化是顯式的，而不是隱含的。

## 範例 route

Qwik City routes 可以從很小的頁面開始：

```tsx
import { component$ } from "@builder.io/qwik";

export default component$(() => {
  return (
    <main class="mx-auto max-w-4xl p-6">
      <h1 class="text-3xl font-semibold">Qwik app</h1>
      <p class="text-slate-600">Add server features when the product needs them.</p>
    </main>
  );
});
```

早期 routes 應專注於 UI 和 data requirements。只有當出現真實 server-side responsibility，例如 auth、persistence、webhooks 或 private API calls 時，再加入 backend。

## 相容性

Qwik 在 Better Fullstack 中的 backend/API support 比 React、Nuxt、Svelte 或 Solid options 更窄。本指南有意保持 stack minimal 且相容。

因為指令使用 `--backend none`、`--database none`、`--orm none`、`--auth none` 和 `--api none`，它不應該產生 server contracts 或 database models。如果第一天就需要這些能力，請選擇 Better Fullstack 支援更完整的 framework/backend pairing。

## 何時選擇

當 resumability 和 minimal initial payload 比第一天產生 backend 更重要時，選擇 Qwik 做 frontend-first apps。

## 之後加入 backend

當 app 需要資料時，決定 Qwik 是否繼續作為 frontend，而由另一個 service 負責 API；或另一個 Better Fullstack preset 是否更符合 fullstack requirement。對許多團隊來說，增加一個獨立 Hono API service 比把 backend concerns 強行塞進 frontend-first scaffold 更清楚。

有用的後續添加項包括：

- 用於 private server logic 的 Hono API。
- PostgreSQL 加 Drizzle 或 Prisma，用於 relational data。
- 當 account management 成為產品一部份時加入 Better Auth。
- 當 frontend 和 API 需要 shared types 時，加入 oRPC 或其他 typed contract layer。

## 部署

最小 Qwik app 可以作為 web app 或 static-capable frontend 直接部署，取決於生成的 adapter 和 hosting target。由於這個指令不加入 database 或 auth，production setup 主要關注 build output、routes 和 public environment variables。

## 故障排查

- 如果你預期有 database files，請檢查指令：本指南有意傳入 `--database none` 和 `--orm none`。
- 如果 auth routes 缺失，這個 stack 使用 `--auth none`。
- 如果 Tailwind classes 不生效，檢查產生的 Tailwind content paths 和 framework integration。
- 如果之後加入 API，保持 public client variables 與 private server secrets 分開。

## 對比說明

當 resumability 和 Qwik component model 是專案主要理由時，選擇它而不是 Astro。當內容頁面從一開始就需要產生 Hono backend 和 database 時，選擇 [Astro with React and Hono](/guides/typescript/astro-react-hono/)。
