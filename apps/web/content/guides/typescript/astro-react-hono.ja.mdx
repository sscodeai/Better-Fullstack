コンテンツの多いページとアイランドに Astro を使用し、さらに TypeScript API サーバーが必要な場合は、このスタックを使用します。

```npm
npm create better-fullstack@latest my-astro-app -- \
  --ecosystem typescript \
  --frontend astro \
  --astro-integration react \
  --backend hono \
  --runtime bun \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library daisyui \
```

## これが生み出すもの

- React 統合を備えた Astro フロントエンド。
- Bun で実行されている Hono バックエンド。
- SQLite および Drizzle。
- Better Auth。
- tRPC。この Astro セットアップでは React 統合が使用されているためです。
- Tailwind CSS と DaisyUI。

## 生成された形状

このスタックは、Astro のコンテンツ優先フロントエンドと別の Hono API サーバーを組み合わせます。 React アイランドは `--astro-integration react` を通じて有効になるため、このガイドでは tRPC が利用可能です。

代表的な形状：

```txt
my-astro-app/
  bts.jsonc
  apps/
    web/
    server/
  packages/
    shared/
```

Astro ページは、ルーティング、コンテンツ、およびアイランドの構成に重点を置く必要があります。 Hono は、API ルート、認証対応サーバーの動作、およびデータベース アクセスを所有する必要があります。

## 例 Astro アイランド

ページ シェルには Astro を使用し、対話性が必要な場合にのみ React を使用します。

```astro
---
import ProjectPicker from "../components/ProjectPicker.tsx";
---

<main class="mx-auto max-w-5xl p-6">
  <h1 class="text-3xl font-semibold">Project docs</h1>
  <ProjectPicker client:load />
</main>
```

これにより、インタラクティブな React コンポーネントが型指定された API を呼び出せるようにしながら、静的ページまたはほとんど静的なページを安価に保つことができます。

## Hono エンドポイントの例

ヘルスチェックと単純なサービスエンドポイントは Hono に属します。

```ts
import { Hono } from "hono";

const app = new Hono();

app.get("/health", (c) => c.json({ ok: true }));

export default app;
```

データベースをサポートするエンドポイントの場合、Drizzle クエリをサーバー専用ヘルパーの背後に保持し、tRPC プロシージャまたは Hono ルートを通じて公開します。

## 互換性に関する注意事項

Astro API の互換性は、選択した UI 統合によって異なります。このページでは `--astro-integration react` を使用しており、これにより tRPC が利用可能になります。 React 以外の Astro 統合の場合は、oRPC を推奨します。

このコマンドでは、UI ライブラリとして DaisyUI が選択されています。後で shadcn/ui に切り替える場合は、Astro React 統合と Tailwind セットアップが選択したコンポーネント ライブラリと一致していることを確認してください。

## いつ選択するか

コンテンツ、ドキュメント、マーケティング ページ、または部分的なハイドレーションがアプリの中心であるものの、生成されたバックエンドとデータベースのセットアップが必要な場合は、Astro を選択します。

## 導入メモ

Astro フロントエンドと Hono API の起点が明確に構成されるように計画します。フロントエンドにはパブリック API URL が必要です。 Hono サービスにはデータベースと認証シークレットが必要です。 SQLite はローカル開発には問題ありませんが、SQLite を維持する場合、運用ホスティングでは永続ストレージを提供する必要があります。

フロントエンドと API が別々にデプロイされている場合は、ローカルホストの動作に依存するのではなく、最終ドメインの CORS と認証 Cookie を構成します。

## トラブルシューティング

- React アイランドがハイドレートしない場合は、Astro クライアント ディレクティブと、コンポーネントがブラウザ専用コードを安全にインポートするかどうかを確認してください。
- 統合を変更した後に tRPC が使用できない場合は、Astro 統合がまだ React であることを確認してください。
- デプロイされたページからの API 呼び出しが失敗する場合は、パブリック API 起点と CORS 設定を確認してください。
- デプロイ後に SQLite データが消えた場合は、ホスティング プラットフォームが永続的な書き込み可能なストレージをサポートしていることを確認してください。

## 比較メモ

完全な React アプリ シェルよりもコンテンツ ページと選択的な対話性が重要な場合は、これを選択します。フロントエンドが専用の TanStack Router React アプリである必要がある場合は、[Hono with tRPC および Drizzle](/guides/typescript/hono-trpc-drizzle/) を選択します。
