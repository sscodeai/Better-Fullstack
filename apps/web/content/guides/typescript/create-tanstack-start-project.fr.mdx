Utilisez cette pile lorsque vous souhaitez une application React fullstack construite autour du modèle de route de serveur de TanStack Router et TanStack Start.

```npm
npm create better-fullstack@latest my-tanstack-app -- \
  --ecosystem typescript \
  --frontend tanstack-start \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## Ce que cela crée

- Une application Web TanStack Start avec des itinéraires basés sur des fichiers.
- Routage Fullstack via la couche serveur intégrée du framework.
- SQLite et Drizzle pour une petite configuration de base de données relationnelle locale.
- Better Auth connecté à l'application générée.
- Tailwind CSS et shadcn/ui pour le frontend.
- Une configuration de pile reproductible dans `bts.jsonc`.

## Forme générée

Le projet généré est destiné à être une seule application Web déployable au lieu d’un espace de travail frontend/API divisé. TanStack Start possède le routage, les fonctions de serveur et la gestion des requêtes, tandis que Drizzle possède le schéma de base de données et les requêtes.

Forme représentative :

```txt
my-tanstack-app/
  bts.jsonc
  package.json
  src/
    routes/
    components/
    lib/
      auth/
      db/
      trpc/
```

Les noms de fichiers exacts peuvent changer à mesure que le générateur évolue, mais la limite importante est stable : le code de route reste avec l'application TanStack Start, le code de la base de données reste derrière les assistants Drizzle et les procédures API sont exposées via la couche `--api trpc` sélectionnée.

## Exemple d'itinéraire et de modèle API

Un itinéraire typique conserve l'état de l'interface utilisateur dans React et appelle un assistant API tapé au lieu de créer manuellement des URL de récupération :

```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/dashboard")({
  component: DashboardPage,
});

function DashboardPage() {
  return (
    <main className="mx-auto max-w-5xl p-6">
      <h1 className="text-2xl font-semibold">Dashboard</h1>
      <p className="text-muted-foreground">
        Load account data through the generated server API layer.
      </p>
    </main>
  );
}
```

Pour les nouveaux travaux API, ajoutez des procédures étroites autour des cas d'utilisation plutôt que d'exposer immédiatement le CRUD en forme de tableau :

```ts
import { z } from "zod";

export const projectInput = z.object({
  name: z.string().min(2),
});

export async function createProject(input: z.infer<typeof projectInput>) {
  const values = projectInput.parse(input);

  return {
    id: crypto.randomUUID(),
    name: values.name,
  };
}
```

## Exemple de modèle de base de données

Drizzle fonctionne mieux lorsque les définitions de schéma sont petites et explicites. Une table SQLite représentative pourrait ressembler à ceci :

```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const project = sqliteTable("project", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  ownerId: text("owner_id").notNull(),
  createdAt: integer("created_at", { mode: "timestamp" }).notNull(),
});
```

Séparez les tables appartenant à des propriétaires autorisés des tables de produits. Better Auth peut générer ou attendre son propre schéma en fonction de l'adaptateur sélectionné ; les modèles de produits doivent faire référence à l'identifiant de l'utilisateur plutôt que de dupliquer l'état d'authentification.

## Quand le choisir

Choisissez TanStack Start lorsque vous souhaitez React, un routage typé, un comportement fullstack colocalisé et une surface de framework plus petite qu'un framework d'application avec de grandes batteries incluses.

Cela convient parfaitement aux tableaux de bord, aux outils internes, aux prototypes SaaS et aux applications où la sécurité des types d'itinéraires est importante.

## Notes de déploiement

Avant le déploiement, assurez-vous que les variables d'environnement de production sont définies pour la base de données, les valeurs secrètes Better Auth et l'URL de l'application publique utilisée par les rappels d'authentification. Pour les déploiements SQLite, confirmez que votre hôte prend en charge le stockage persistant en écriture ; de nombreuses plates-formes sans serveur ne le font pas.

Si vous ciblez une plate-forme sans serveur, PostgreSQL est généralement le choix de base de données de production plus sûr qu'un fichier SQLite local. Gardez `bts.jsonc` engagé afin que les futurs coéquipiers et agents puissent reconstruire la pile prévue.

## Dépannage

- Si les rappels d'authentification redirigent vers le mauvais hôte, vérifiez d'abord l'URL de base de l'application et les variables d'environnement Better Auth.
- Si les requêtes de base de données fonctionnent localement mais échouent après le déploiement, vérifiez l'URL de la base de données de production et si les migrations ont été exécutées sur la base de données déployée.
- Si les types de routes générés semblent obsolètes, redémarrez le processus de génération de type de framework et réexécutez la plus petite vérification de type pertinente.
- Si les appels du client tRPC échouent avec des 404, confirmez que la route API générée est toujours montée là où le client l'attend.

## Compromis

TanStack Start est une meilleure solution lorsque vous souhaitez vous appuyer sur l'écosystème TanStack. Si vous souhaitez le plus grand écosystème d'hébergement et de documentation, comparez-le avec [Next.js avec Drizzle et Better Auth](/guides/typescript/nextjs-drizzle-better-auth/).

Par rapport à un backend Hono distinct, cette pile comporte moins de services déployables et moins de configurations multi-origines. Par rapport à Next.js, il vous donne un premier modèle mental TanStack Router et une empreinte de framework plus petite, mais un écosystème plus jeune.
