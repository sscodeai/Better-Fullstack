当你需要主流 React fullstack app，并且需要 relational data、authentication 和 typed API layer 时，使用这个 stack。

```npm
npm create better-fullstack@latest my-next-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## 创建内容

- fullstack mode 的 Next.js app。
- PostgreSQL 作为 relational database。
- Drizzle 作为 TypeScript ORM。
- Better Auth 用于 authentication。
- tRPC 提供 end-to-end typed APIs。
- Tailwind CSS 和 shadcn/ui 用于 frontend styling。

## 生成结构

生成的 app 使用 Next.js 作为 web 和 server boundary。PostgreSQL 与 Drizzle 放在 server-only modules 后面；tRPC 为需要 application data 的 client components 提供 typed calls。

代表性结构：

```txt
my-next-app/
  bts.jsonc
  app/
  components/
  lib/
    auth/
    db/
    trpc/
```

确切路径会随 generator 版本变化，但预期拆分一致：UI components 渲染产品体验，route handlers 和 server code 拥有 request boundaries，database access 不进入 browser bundles。

## Model 与 typed procedure

保持 product tables 明确且易迁移。Better Auth 应拥有 authentication tables 和 session behavior。Product tables 应引用 user IDs，避免复制 session fields。

在 frontend 需要 inferred input/output types 的操作中使用 tRPC。把 authorization check 放在 mutation 附近；创建 workspace 前应证明用户已登录。

## 何时选择

适合 SaaS apps、account-based products、dashboards、admin portals，以及重视 Next.js hosting familiarity 的项目。

## 兼容性

- `--backend self` 表示 Next.js 拥有 server boundary；除非想要明确 service split，否则不要再添加单独 backend。
- `--database postgres` 适合 production hosting、pooled connections 和 managed database providers。
- `--api trpc` 很适合这个 React-based stack。Better Fullstack 中非 React TypeScript frontends 通常使用 oRPC。
- shadcn/ui 需要 Tailwind 和 React。如果以后 regenerate 或添加 components，请保持这些选择一致。

## 部署

在部署环境中设置 PostgreSQL connection string、Better Auth secrets 和 app base URL。部署依赖新 columns 或 tables 的代码前，先运行 migrations。

在 serverless hosting 上，检查 database provider 的 pooling 建议。大量 serverless instances 同时启动时，直接 PostgreSQL connections 很容易耗尽限制。
