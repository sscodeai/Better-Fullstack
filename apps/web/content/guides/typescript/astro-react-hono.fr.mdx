Utilisez cette pile lorsque vous souhaitez Astro pour les pages et les îles riches en contenu, plus un serveur TypeScript API.

```npm
npm create better-fullstack@latest my-astro-app -- \
  --ecosystem typescript \
  --frontend astro \
  --astro-integration react \
  --backend hono \
  --runtime bun \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library daisyui \
```

## Ce que cela crée

- Une interface Astro avec intégration React.
- Un backend Hono exécuté sur Bun.
- SQLite et Drizzle.
- Better Auth.
- tRPC, car cette configuration Astro utilise l'intégration React.
- Tailwind CSS et DaisyUI.

## Forme générée

Cette pile combine l'interface de contenu d'abord de Astro avec un serveur Hono API distinct. Les îlots React sont activés via `--astro-integration react`, c'est pourquoi tRPC est disponible dans ce guide.

Forme représentative :

```txt
my-astro-app/
  bts.jsonc
  apps/
    web/
    server/
  packages/
    shared/
```

Les pages Astro doivent rester concentrées sur le routage, le contenu et la composition des îlots. Hono doit posséder les routes API, le comportement du serveur compatible avec l'authentification et l'accès à la base de données.

## Exemple d'îlot Astro

Utilisez Astro pour le shell de la page et React uniquement lorsque l'interactivité est nécessaire :

```astro
---
import ProjectPicker from "../components/ProjectPicker.tsx";
---

<main class="mx-auto max-w-5xl p-6">
  <h1 class="text-3xl font-semibold">Project docs</h1>
  <ProjectPicker client:load />
</main>
```

Cela permet de garder les pages statiques ou principalement statiques bon marché tout en permettant aux composants React interactifs d'appeler le API typé.

## Exemple de point de terminaison Hono

Les vérifications de l'état et les points de terminaison de service simples appartiennent à Hono :

```ts
import { Hono } from "hono";

const app = new Hono();

app.get("/health", (c) => c.json({ ok: true }));

export default app;
```

Pour les points de terminaison basés sur une base de données, conservez les requêtes Drizzle derrière les assistants serveur uniquement et exposez-les via des procédures tRPC ou des routes Hono.

## Remarques sur la compatibilité

La compatibilité Astro API dépend de l'intégration de l'interface utilisateur sélectionnée. Cette page utilise `--astro-integration react`, qui maintient tRPC disponible. Pour les intégrations non-React Astro, préférez oRPC.

DaisyUI est sélectionnée comme bibliothèque d'interface utilisateur dans cette commande. Si vous passez à shadcn/ui plus tard, assurez-vous que l'intégration Astro React et la configuration de Tailwind correspondent toujours à la bibliothèque de composants que vous choisissez.

## Quand le choisir

Choisissez Astro lorsque le contenu, les documents, les pages marketing ou l'hydratation partielle sont au cœur de l'application, mais que vous avez toujours besoin d'un backend généré et d'une configuration de base de données.

## Notes de déploiement

Prévoyez que l'interface Astro et Hono API aient des origines clairement configurées. Le frontend a besoin de l'URL publique API. Le service Hono a besoin de bases de données et de secrets d'authentification. SQLite peut convenir au développement local, mais l'hébergement de production doit fournir un stockage persistant si vous conservez SQLite.

Si le frontend et API sont déployés séparément, configurez CORS et les cookies d'authentification pour les domaines finaux plutôt que de vous fier au comportement de localhost.

## Dépannage

- Si les îlots React ne s'hydratent pas, vérifiez la directive client Astro et si le composant importe en toute sécurité le code du navigateur uniquement.
- Si tRPC n'est pas disponible après avoir modifié les intégrations, confirmez que l'intégration Astro est toujours React.
- Si les appels API échouent à partir des pages déployées, vérifiez l’origine publique API et les paramètres CORS.
- Si les données SQLite disparaissent après le déploiement, confirmez que votre plate-forme d'hébergement prend en charge le stockage persistant en écriture.

## Notes de comparaison

Choisissez cette option lorsque les pages de contenu et l'interactivité sélective comptent plus qu'un shell d'application React complet. Choisissez [Hono avec tRPC et Drizzle](/guides/typescript/hono-trpc-drizzle/) lorsque le frontend doit être une application React TanStack Router dédiée.
