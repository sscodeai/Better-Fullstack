첫날에 별도의 백엔드나 데이터베이스를 추가하지 않고 Qwik 앱을 원할 때 이 스택을 사용하십시오.

```npm
npm create better-fullstack@latest my-qwik-app -- \
  --ecosystem typescript \
  --frontend qwik \
  --backend none \
  --runtime none \
  --database none \
  --orm none \
  --auth none \
  --api none \
  --css-framework tailwind \
```

## 이것이 만드는 것

- Qwik 앱.
- Tailwind CSS.
- 기본적으로 데이터베이스, ORM, 인증 또는 API 계층이 없습니다.
- 나중에 확장할 수 있는 생성된 프로젝트 구성입니다.

## 생성된 모양

이 가이드는 의도적으로 프런트엔드 전용 Qwik City 앱으로 시작됩니다. 백엔드 또는 지속성 계층을 선택하기 전에 애플리케이션 셸, 라우팅 및 스타일을 적용하려는 경우에 유용합니다.

대표적인 형태:

```txt
my-qwik-app/
  bts.jsonc
  src/
    routes/
    components/
```

생성된 `bts.jsonc`은 백엔드, 런타임, 데이터베이스, ORM, 인증 및 API이 모두 `none`임을 기록합니다. 이는 스택의 최소 특성을 암시적이 아닌 명시적으로 만듭니다.

## 예시 경로

Qwik City 경로는 작게 시작할 수 있습니다.

```tsx
import { component$ } from "@builder.io/qwik";

export default component$(() => {
  return (
    <main class="mx-auto max-w-4xl p-6">
      <h1 class="text-3xl font-semibold">Qwik app</h1>
      <p class="text-slate-600">Add server features when the product needs them.</p>
    </main>
  );
});
```

UI 및 데이터 요구사항에 초점을 맞춰 초기 경로를 유지하세요. 인증, 지속성, 웹훅 또는 비공개 API 호출과 같은 실제 서버 측 책임이 있는 경우에만 백엔드를 추가하세요.

## 호환성 참고사항

Qwik은 React, Nuxt, Svelte 또는 Solid 옵션보다 Better Fullstack에서 더 좁은 백엔드/API 지원을 제공합니다. 이 가이드는 의도적으로 스택을 최소화하고 호환되도록 유지합니다.

명령은 `--backend none`, `--database none`, `--orm none`, `--auth none` 및 `--api none`을 사용하므로 서버 계약이나 데이터베이스 모델을 생성해서는 안 됩니다. 처음부터 이러한 기능이 필요하다면 더 완전한 Better Fullstack 지원을 갖춘 프레임워크/백엔드 쌍을 선택하세요.

## 그것을 선택하는 경우

재개 가능성과 최소 초기 페이로드가 첫날 생성된 백엔드보다 더 중요한 프런트엔드 우선 앱의 경우 Qwik을 선택하세요.

## 나중에 백엔드 추가

앱에 데이터가 필요한 경우 다른 서비스가 API를 소유하는 동안 Qwik을 프런트엔드로 유지할지, 아니면 다른 Better Fullstack 사전 설정이 전체 스택 요구 사항에 더 잘 맞는지 결정하세요. 많은 팀의 경우 Hono API을 별도의 서비스로 추가하는 것이 백엔드 문제를 프런트엔드 우선 스캐폴드에 강제로 적용하는 것보다 더 깔끔합니다.

향후 유용한 추가 사항은 다음과 같습니다.

- 개인 서버 로직의 경우 Hono API입니다.
- 관계형 데이터의 경우 PostgreSQL에 Drizzle 또는 Prisma를 추가합니다.
- Better Auth 계정 관리가 제품의 일부가 되는 경우.
- 프런트엔드와 API에 공유 유형이 필요한 경우 oRPC 또는 다른 유형의 계약 레이어.

## 배포 참고사항

최소 Qwik 앱은 생성된 어댑터 및 호스팅 대상에 따라 웹 앱 또는 정적 지원 프런트엔드로 배포하기가 쉽습니다. 이 명령은 데이터베이스나 인증을 추가하지 않으므로 프로덕션 설정은 주로 빌드 출력, 경로 및 공개 환경 변수에 중점을 둡니다.

## 문제 해결

- 데이터베이스 파일을 예상했다면 명령을 확인하세요. 이 가이드에서는 의도적으로 `--database none` 및 `--orm none`을 전달합니다.
- 인증 경로가 누락된 경우 스택은 `--auth none`을 사용합니다.
- Tailwind 클래스가 적용되지 않는 경우 생성된 Tailwind 콘텐츠 경로 및 프레임워크 통합을 검사하세요.
- 나중에 API을 추가하는 경우 공개 클라이언트 변수를 개인 서버 비밀과 별도로 유지하세요.
