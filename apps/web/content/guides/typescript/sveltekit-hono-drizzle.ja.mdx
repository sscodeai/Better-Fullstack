フロントエンドに SvelteKit を配置し、その横に小規模な TypeScript API サーバーを配置する場合は、このスタックを使用します。

```npm
npm create better-fullstack@latest my-sveltekit-app -- \
  --ecosystem typescript \
  --frontend svelte \
  --backend hono \
  --runtime bun \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api orpc \
  --css-framework tailwind \
```

## これが生み出すもの

- SvelteKit フロントエンド。
- Bun 上の Hono バックエンド。
- 型付き API の oRPC は、React 以外のフロントエンドの選択肢全体でコントラクトします。
- PostgreSQL および Drizzle。
- Better Auth では、選択したデータベースと ORM がそれをサポートします。

## 生成された形状

これは分割された TypeScript スタックです。SvelteKit はフロントエンド エクスペリエンスを処理し、Hono は API サーバーを処理し、oRPC はそれらの間の型付き境界です。

代表的な形状：

```txt
my-sveltekit-app/
  bts.jsonc
  apps/
    web/
    server/
  packages/
    shared/
```

特定のワークスペース名は変更される可能性がありますが、所有権は明確にしておいてください。 Svelte コンポーネントはデータベース モジュールを直接インポートしないでください。型指定されたサーバー操作を呼び出す必要があります。

## SvelteKit 呼び出しパターンの例

ページまたはロード関数は、アプリ全体でリクエスト ペイロードをハードコーディングするのではなく、生成された API クライアントと通信する必要があります。

```ts
export async function load() {
  return {
    title: "Projects",
  };
}
```

変更の場合は、SvelteKit UI がすでにフォーム入力を検証している場合でも、Hono 側で検証と承認を維持します。

## Drizzle モデルの例

PostgreSQL と Drizzle を使用すると、明示的なテーブル定義が得られます。

```ts
import { pgTable, text, timestamp } from "drizzle-orm/pg-core";

export const project = pgTable("project", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  ownerId: text("owner_id").notNull(),
  createdAt: timestamp("created_at").defaultNow().notNull(),
});
```

Hono ハンドラーと oRPC プロシージャがビジネス ルールのゴミ捨て場にならないように、クエリの周囲に小規模なサーバー関数を使用します。

## いつ選択するか

チームが SvelteKit を好み、それでも明示的な API サーバーと型指定されたコントラクトが必要な場合は、これを選択します。これは、フロントエンドとバックエンドの別々の展開に成長する可能性のあるアプリに役立ちます。

## 互換性に関する注意事項

- SvelteKit は React フロントエンドではないため、このガイドでは `--api orpc` を使用します。
- `--backend hono` と `--runtime bun` は、SvelteKit サーバー ルートのみに依存するのではなく、別の API サービスを作成します。
- Better Auth では、選択したデータベースと ORM パスの互換性を維持する必要があります。認証スキーマの変更と Drizzle の移行の同期を維持します。
- 後で個別のバックエンドを削除する場合は、API クライアントのセットアップ、CORS、および認証 Cookie の処理を​​一緒に再検討してください。

## 導入メモ

別々の Web 環境変数と API 環境変数が必要です。 SvelteKit アプリにはパブリック API オリジンが必要です。 Hono サーバーには PostgreSQL と認証シークレットが必要です。異なるドメインにデプロイする場合は、CORS と Cookie を意図的に構成します。

新しいテーブルまたは列を必要とする API コードをデプロイする前に、データベースの移行を実行します。

## トラブルシューティング

- ブラウザからのリクエストは失敗するが、curl では機能する場合は、CORS と Cookie 資格情報を確認してください。
- SvelteKit がデータなしでレンダリングされる場合は、サーバー側およびブラウザー側のコードで使用される API オリジンを確認してください。
- 認証がドメイン間で一貫していない場合は、Cookie ドメイン、同一サイト、および安全な設定を確認してください。
- Drizzle スキーマの変更が表示されない場合は、移行が生成され、ターゲットの PostgreSQL データベースに適用されたことを確認します。
