TanStack Router 및 TanStack Start의 서버 경로 모델을 기반으로 구축된 React 풀스택 앱을 원할 때 이 스택을 사용하세요.

```npm
npm create better-fullstack@latest my-tanstack-app -- \
  --ecosystem typescript \
  --frontend tanstack-start \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## 이것이 만드는 것

- 파일 기반 경로가 있는 TanStack Start 웹 앱.
- 프레임워크에 내장된 서버 계층을 통한 풀스택 라우팅.
- 소규모 로컬 관계형 데이터베이스 설정을 위한 SQLite 및 Drizzle
- Better Auth이 생성된 앱에 연결됩니다.
- 프런트엔드의 경우 Tailwind CSS 및 shadcn/ui입니다.
- `bts.jsonc`의 재현 가능한 스택 구성입니다.

## 생성된 모양

생성된 프로젝트는 분할 프런트엔드/API 작업 영역 대신 배포 가능한 단일 웹 앱이 되도록 고안되었습니다. TanStack Start은 라우팅, 서버 기능 및 요청 처리를 소유하고 Drizzle은 데이터베이스 스키마 및 쿼리를 소유합니다.

대표적인 형태:

```txt
my-tanstack-app/
  bts.jsonc
  package.json
  src/
    routes/
    components/
    lib/
      auth/
      db/
      trpc/
```

정확한 파일 이름은 생성기가 발전함에 따라 변경될 수 있지만 중요한 경계는 안정적입니다. 경로 코드는 TanStack Start 앱에 유지되고 데이터베이스 코드는 Drizzle 도우미 뒤에 유지되며 API 프로시저는 선택된 `--api trpc` 레이어를 통해 노출됩니다.

## 예제 경로 및 API 패턴

일반적인 경로는 React에서 UI 상태를 유지하고 가져오기 URL을 수동으로 작성하는 대신 형식화된 API 도우미를 호출합니다.

```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/dashboard")({
  component: DashboardPage,
});

function DashboardPage() {
  return (
    <main className="mx-auto max-w-5xl p-6">
      <h1 className="text-2xl font-semibold">Dashboard</h1>
      <p className="text-muted-foreground">
        Load account data through the generated server API layer.
      </p>
    </main>
  );
}
```

새로운 API 작업의 경우 테이블 모양의 CRUD를 즉시 노출하는 대신 사용 사례 주변에 좁은 절차를 추가하세요.

```ts
import { z } from "zod";

export const projectInput = z.object({
  name: z.string().min(2),
});

export async function createProject(input: z.infer<typeof projectInput>) {
  const values = projectInput.parse(input);

  return {
    id: crypto.randomUUID(),
    name: values.name,
  };
}
```

## 데이터베이스 모델 예

Drizzle은 스키마 정의가 작고 명시적일 때 가장 잘 작동합니다. 대표적인 SQLite 테이블은 다음과 같습니다.

```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const project = sqliteTable("project", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  ownerId: text("owner_id").notNull(),
  createdAt: integer("created_at", { mode: "timestamp" }).notNull(),
});
```

인증 소유 테이블을 제품 테이블과 별도로 유지하세요. Better Auth은(는) 선택한 어댑터에 따라 자체 스키마를 생성하거나 예상할 수 있습니다. 제품 모델은 인증 상태를 복제하는 대신 사용자 식별자를 참조해야 합니다.

## 그것을 선택하는 경우

React, 입력된 라우팅, 함께 배치된 풀 스택 동작 및 대형 배터리 포함 애플리케이션 프레임워크보다 작은 프레임워크 표면을 원할 경우 TanStack Start을 선택하십시오.

이는 경로 유형 안전이 중요한 대시보드, 내부 도구, SaaS 프로토타입 및 앱에 적합합니다.

## 배포 참고사항

배포하기 전에 데이터베이스, Better Auth 비밀 값 및 인증 콜백에 사용되는 공용 애플리케이션 URL에 대해 프로덕션 환경 변수가 정의되어 있는지 확인하세요. SQLite 배포의 경우 호스트가 영구 쓰기 가능 스토리지를 지원하는지 확인하세요. 많은 서버리스 플랫폼은 그렇지 않습니다.

서버리스 플랫폼을 대상으로 하는 경우 일반적으로 PostgreSQL이 로컬 SQLite 파일보다 더 안전한 프로덕션 데이터베이스 선택입니다. 향후 팀원과 에이전트가 의도한 스택을 재구성할 수 있도록 `bts.jsonc`을(를) 커밋된 상태로 유지하세요.

## 문제 해결

- 인증 콜백이 잘못된 호스트로 리디렉션되는 경우 먼저 앱 기본 URL과 Better Auth 환경 변수를 확인하세요.
- 데이터베이스 쿼리가 로컬에서 작동하지만 배포 후에 실패하는 경우 프로덕션 데이터베이스 URL과 배포된 데이터베이스에 대해 마이그레이션이 실행되었는지 여부를 확인하세요.
- 생성된 경로 유형이 오래되어 보이는 경우 프레임워크 유형 생성 프로세스를 다시 시작하고 가장 작은 관련 유형 검사를 다시 실행하세요.
- tRPC 클라이언트 호출이 404와 함께 실패하는 경우 생성된 API 경로가 클라이언트가 예상하는 위치에 여전히 마운트되어 있는지 확인하세요.

## 트레이드오프

TanStack Start은 TanStack 생태계에 참여하고 싶을 때 더 적합합니다. 가장 큰 호스팅 및 문서 생태계를 원한다면 [Next.js(Drizzle 및 Better Auth 포함)](/guides/typescript/nextjs-drizzle-better-auth/)과 비교하세요.

별도의 Hono 백엔드와 비교할 때 이 스택은 배포 가능한 서비스가 적고 교차 원본 설정이 적습니다. Next.js과 비교하면 TanStack Router 최초의 멘탈 모델과 더 작은 프레임워크 공간을 제공하지만 더 젊은 생태계를 제공합니다.
