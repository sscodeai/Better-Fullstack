Usa este stack cuando quieras una app Vue/Nuxt fullstack con datos relacionales, autenticacion y contratos API tipados.

```npm
npm create better-fullstack@latest my-nuxt-app -- \
  --ecosystem typescript \
  --frontend nuxt \
  --backend self \
  --database postgres \
  --orm prisma \
  --auth better-auth \
  --api orpc \
  --css-framework tailwind \
```

## Que crea

- Una app Nuxt en modo fullstack.
- PostgreSQL y Prisma.
- Better Auth.
- oRPC para contratos API tipados.
- Tailwind CSS.
- Una configuracion reproducible `bts.jsonc`.

## Forma generada

Este stack mantiene la aplicacion dentro del modelo fullstack de Nuxt. Prisma controla esquema y migraciones, Better Auth controla autenticacion, y oRPC da contratos tipados sin asumir un frontend React.

Forma representativa:

```txt
my-nuxt-app/
  bts.jsonc
  app/
  server/
  prisma/
    schema.prisma
```

El limite importante es que base de datos y auth solo de servidor viven en modulos server o API handlers de Nuxt, mientras los componentes Vue llaman contratos de servidor tipados.

## Modelos y contratos

Un modelo de producto pequeno puede empezar en Prisma y referenciar usuarios por identificador. Mantiene los modelos requeridos por Better Auth alineados con el adapter y separados de tablas especificas de la aplicacion.

Usa oRPC para describir operaciones de servidor validadas para Nuxt. Coloca autorizacion en la implementacion de servidor, no solo en componentes Vue. Los guards cliente mejoran UX, pero no protegen datos.

## Compatibilidad

Nuxt usa oRPC en Better Fullstack para contratos tipados. tRPC queda reservado para frontends TypeScript basados en React, asi que esta guia usa `--api orpc`.

`--backend self` es la eleccion correcta para Nuxt fullstack. Si quieres una API desplegada por separado, usa una opcion backend como Hono en un stack que soporte esa combinacion.

## Despliegue

Define la cadena PostgreSQL, secretos de Better Auth y URL publica de la app en el entorno. Ejecuta migraciones Prisma antes de desplegar codigo que depende de nuevo schema.

En plataformas serverless, revisa como se genera Prisma Client y como se agrupan conexiones de base de datos. Proveedores PostgreSQL gestionados suelen documentar la cadena mas segura para runtimes serverless.
