當你想要圍繞 TanStack Router 和 TanStack Start server route model 建立 React fullstack app 時，使用這個 stack。

```npm
npm create better-fullstack@latest my-tanstack-app -- \
  --ecosystem typescript \
  --frontend tanstack-start \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## 創建內容

- 帶有 file-based routes 的 TanStack Start web app。
- 透過 framework 內建 server layer 實作 fullstack routing。
- SQLite 和 Drizzle，用於小型本地關係型資料庫設定。
- 已接取 Better Auth。
- frontend 使用 Tailwind CSS 和 shadcn/ui。
- `bts.jsonc` 中儲存可重複的 stack 配置。

## 生成結構

產生的項目是單獨可部署的 web app，而不是拆開的 frontend/API workspace。 TanStack Start 負責 routing、server functions 和 request handling；Drizzle 負責 database schema 和 queries。

代表性結構：

```txt
my-tanstack-app/
  bts.jsonc
  package.json
  src/
    routes/
    components/
    lib/
      auth/
      db/
      trpc/
```

準確檔案名稱可能隨 generator 演進，但邊界保持穩定：route code 留在 TanStack Start app 中，database code 放在 Drizzle helpers 後面，API procedures 透過 `--api trpc` 暴露。

## Route 和 API 模式

典型 route 會把 UI state 留在 React 中，並且呼叫 typed API helper，而不是手寫 `fetch` URL：

```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/dashboard")({
  component: DashboardPage,
});

function DashboardPage() {
  return (
    <main className="mx-auto max-w-5xl p-6">
      <h1 className="text-2xl font-semibold">Dashboard</h1>
      <p className="text-muted-foreground">
        Load account data through the generated server API layer.
      </p>
    </main>
  );
}
```

新增 API 時，應圍繞 use case 建立窄 procedures，而非一開始就暴露表形 CRUD：

```ts
import { z } from "zod";

export const projectInput = z.object({
  name: z.string().min(2),
});

export async function createProject(input: z.infer<typeof projectInput>) {
  const values = projectInput.parse(input);

  return {
    id: crypto.randomUUID(),
    name: values.name,
  };
}
```

## Database model 範例

Drizzle 在 schema 小而明確時效果最好。一個代表性的 SQLite table 可能像這樣：

```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const project = sqliteTable("project", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  ownerId: text("owner_id").notNull(),
  createdAt: integer("created_at", { mode: "timestamp" }).notNull(),
});
```

把 auth 擁有的 tables 與 product tables 分開。 Better Auth 可能會根據所選 adapter 產生或期望自己的 schema；product models 應引用 user identifier，而不是複製 auth state。

## 何時選擇

當你需要 React、typed routing、colocated fullstack behavior，並且想要比大型 batteries-included framework 更小的 framework surface 時，選擇 TanStack Start。

它適合 dashboards、internal tools、SaaS prototypes，以及 route type safety 很重要的應用。

## 部署

部署前，請確保為 database、Better Auth secrets，以及 auth callbacks 使用的 public application URL 配好環境變數。對於 SQLite 部署，請確認 host 支援持久化可寫儲存體；許多 serverless platforms 不支援。

如果目標是 serverless platform，PostgreSQL 通常比本機 SQLite 檔案更適合作為生產資料庫。提交 `bts.jsonc`，方便之後的 teammates 和 agents 還原預期 stack。

## 故障排查

- 如果 auth callbacks 重新導向到錯誤 host，先檢查 app base URL 和 Better Auth 環境變數。
- 如果 database queries 在本地正常、部署後失敗，請確認 production database URL，並檢查 migrations 是否已在部署的 database 上執行。
- 如果 generated route types 看起來過期，重新啟動 framework type generation process，並重新執行最小相關 typecheck。
- 如果 tRPC client calls 回傳 404，請確認 generated API route 仍然掛載在 client 期望的位置。

## Tradeoffs

當你想深入使用 TanStack ecosystem 時，TanStack Start 比較適合。如果你想要最大的 hosting 和 documentation ecosystem，可以與 [Next.js with Drizzle and Better Auth](/guides/typescript/nextjs-drizzle-better-auth/) 比較。

與單獨的 Hono backend 相比，這個 stack 的 deployable services 更少，cross-origin 設定也更少。與 Next.js 相比，它提供 TanStack Router-first 的心智模型和更小的 framework footprint，但生態系統也更年輕。
