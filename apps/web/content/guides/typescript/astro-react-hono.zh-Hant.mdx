當你想要用 Astro 建立內容密集頁面和 islands，同時還需要 TypeScript API server 時，使用這個 stack。

```npm
npm create better-fullstack@latest my-astro-app -- \
  --ecosystem typescript \
  --frontend astro \
  --astro-integration react \
  --backend hono \
  --runtime bun \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library daisyui \
```

## 創建內容

- 帶有 React integration 的 Astro frontend。
- 執行在 Bun 上的 Hono backend。
- SQLite 和 Drizzle。
- Better Auth。
- tRPC，因為這個 Astro 配置使用 React integration。
- Tailwind CSS 和 DaisyUI。

## 生成結構

這個 stack 把 Astro 的 content-first frontend 和獨立 Hono API server 組合在一起。 React islands 透過 `--astro-integration react` 啟用，因此本指南可以使用 tRPC。

代表性結構：

```txt
my-astro-app/
  bts.jsonc
  apps/
    web/
    server/
  packages/
    shared/
```

Astro pages 應專注於 routing、content 和 island composition。 Hono 應負責 API routes、auth-aware server behavior 和 database access。

## 範例 Astro island

用 Astro 承載頁面外殼，只在需要互動的地方使用 React：

```astro
---
import ProjectPicker from "../components/ProjectPicker.tsx";
---

<main class="mx-auto max-w-5xl p-6">
  <h1 class="text-3xl font-semibold">Project docs</h1>
  <ProjectPicker client:load />
</main>
```

這樣靜態或近似靜態頁面保持輕量，同時互動式 React components 仍然可以呼叫 typed API。

## 範例 Hono endpoint

Health checks 和簡單 service endpoints 適合放在 Hono：

```ts
import { Hono } from "hono";

const app = new Hono();

app.get("/health", (c) => c.json({ ok: true }));

export default app;
```

對於 database-backed endpoints，把 Drizzle queries 放在 server-only helpers 後面，再透過 tRPC procedures 或 Hono routes 暴露。

## 相容性

Astro API 相容性取決於所選的 UI integration。本頁使用 `--astro-integration react`，因此 tRPC 可用。對於非 React 的 Astro integrations，優先使用 oRPC。

這個指令選擇 DaisyUI 作為 UI library。如果之後切到 shadcn/ui，確認 Astro React integration 和 Tailwind setup 仍然匹配你選擇的 component library。

## 何時選擇

當 content、docs、marketing pages 或 partial hydration 是應用核心，但你仍需要產生 backend 和 database setup 時，選擇 Astro。

## 部署

為 Astro frontend 和 Hono API 明確配置 origins。 frontend 需要 public API URL；Hono service 需要 database 和 auth secrets。 SQLite 可以用於 local development，但如果生產環境繼續使用 SQLite，hosting 必須提供 persistent storage。

如果 frontend 和 API 分開部署，請依最終 domains 設定 CORS 和 auth cookies，不要依賴 localhost 行為。

## 故障排查

- 如果 React islands 沒有 hydrate，檢查 Astro client directive，以及 component 是否安全地導入 browser-only code。
- 如果更換 integrations 後 tRPC 不可用，確認 Astro integration 仍然是 React。
- 如果 deployed pages 的 API calls 失敗，檢查 public API origin 和 CORS settings。
- 如果 deploy 後 SQLite data 消失，確認 hosting platform 支援 persistent writable storage。

## 對比說明

當內容頁面和選擇性 interactivity 比完整 React app shell 更重要時，選擇這個 stack。當 frontend 應該是專用 TanStack Router React app 時，選擇 [Hono with tRPC and Drizzle](/guides/typescript/hono-trpc-drizzle/)。
