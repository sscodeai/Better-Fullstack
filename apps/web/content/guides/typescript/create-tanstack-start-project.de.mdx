Verwenden Sie diesen Stack, wenn Sie eine React-Fullstack-App wünschen, die auf dem Serverroutenmodell von TanStack Router und TanStack Start basiert.

```npm
npm create better-fullstack@latest my-tanstack-app -- \
  --ecosystem typescript \
  --frontend tanstack-start \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## Was dadurch entsteht

- Eine TanStack Start-Webanwendung mit dateibasierten Routen.
- Fullstack-Routing durch die integrierte Serverschicht des Frameworks.
- SQLite und Drizzle für ein kleines lokales relationales Datenbank-Setup.
- Better Auth in die generierte App eingebunden.
- Tailwind CSS und shadcn/ui für das Frontend.
- Eine reproduzierbare Stack-Konfiguration in `bts.jsonc`.

## Erzeugte Form

Das generierte Projekt soll eine einzelne bereitstellbare Web-App sein und nicht ein geteilter Frontend-/API-Arbeitsbereich. TanStack Start ist für Routing, Serverfunktionen und Anforderungsbearbeitung zuständig, während Drizzle für das Datenbankschema und die Abfragen verantwortlich ist.

Repräsentative Form:

```txt
my-tanstack-app/
  bts.jsonc
  package.json
  src/
    routes/
    components/
    lib/
      auth/
      db/
      trpc/
```

Genaue Dateinamen können sich mit der Weiterentwicklung des Generators ändern, aber die wichtige Grenze ist stabil: Der Routencode bleibt bei der TanStack Start-App, der Datenbankcode bleibt hinter den Drizzle-Helfern und API-Prozeduren werden über die ausgewählte `--api trpc`-Ebene verfügbar gemacht.

## Beispielroute und API-Muster

Eine typische Route behält den UI-Status in React bei und ruft einen typisierten API-Helfer auf, anstatt manuell Abruf-URLs zu erstellen:

```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/dashboard")({
  component: DashboardPage,
});

function DashboardPage() {
  return (
    <main className="mx-auto max-w-5xl p-6">
      <h1 className="text-2xl font-semibold">Dashboard</h1>
      <p className="text-muted-foreground">
        Load account data through the generated server API layer.
      </p>
    </main>
  );
}
```

Fügen Sie für neue API-Arbeiten enge Verfahren rund um Anwendungsfälle hinzu, anstatt tabellenförmiges CRUD sofort offenzulegen:

```ts
import { z } from "zod";

export const projectInput = z.object({
  name: z.string().min(2),
});

export async function createProject(input: z.infer<typeof projectInput>) {
  const values = projectInput.parse(input);

  return {
    id: crypto.randomUUID(),
    name: values.name,
  };
}
```

## Beispiel für ein Datenbankmodell

Drizzle funktioniert am besten, wenn Schemadefinitionen klein und explizit sind. Eine repräsentative SQLite-Tabelle könnte so aussehen:

```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const project = sqliteTable("project", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  ownerId: text("owner_id").notNull(),
  createdAt: integer("created_at", { mode: "timestamp" }).notNull(),
});
```

Halten Sie authentifizierte Tabellen getrennt von Produkttabellen. Better Auth kann je nach ausgewähltem Adapter ein eigenes Schema generieren oder erwarten; Produktmodelle sollten auf die Benutzerkennung verweisen, anstatt den Authentifizierungsstatus zu duplizieren.

## Wann man es wählt

Wählen Sie TanStack Start, wenn Sie React, typisiertes Routing, kolokalisiertes Fullstack-Verhalten und eine kleinere Framework-Oberfläche als ein großes, batteriebetriebenes Anwendungs-Framework wünschen.

Dies eignet sich gut für Dashboards, interne Tools, SaaS-Prototypen und Apps, bei denen die Sicherheit des Routentyps wichtig ist.

## Bereitstellungshinweise

Stellen Sie vor der Bereitstellung sicher, dass Produktionsumgebungsvariablen für die Datenbank, geheime Better Auth-Werte und die von Authentifizierungsrückrufen verwendete öffentliche Anwendungs-URL definiert sind. Stellen Sie bei SQLite-Bereitstellungen sicher, dass Ihr Host dauerhaften beschreibbaren Speicher unterstützt. Viele serverlose Plattformen tun dies nicht.

Wenn Sie auf eine serverlose Plattform abzielen, ist PostgreSQL normalerweise die sicherere Wahl für die Produktionsdatenbank als eine lokale SQLite-Datei. Halten Sie `bts.jsonc` fest, damit zukünftige Teamkollegen und Agenten den beabsichtigten Stapel wiederherstellen können.

## Fehlerbehebung

- Wenn Authentifizierungsrückrufe zum falschen Host umgeleitet werden, überprüfen Sie zuerst die Basis-URL der App und die Umgebungsvariablen Better Auth.
- Wenn Datenbankabfragen lokal funktionieren, aber nach der Bereitstellung fehlschlagen, überprüfen Sie die URL der Produktionsdatenbank und ob Migrationen für die bereitgestellte Datenbank ausgeführt wurden.
- Wenn generierte Routentypen veraltet aussehen, starten Sie den Framework-Typ-Generierungsprozess neu und führen Sie die kleinste relevante Typprüfung erneut durch.
- Wenn tRPC-Clientaufrufe mit 404-Fehlern fehlschlagen, stellen Sie sicher, dass die generierte API-Route noch dort gemountet ist, wo der Client sie erwartet.

## Kompromisse

TanStack Start passt besser, wenn Sie in das TanStack-Ökosystem einsteigen möchten. Wenn Sie das größte Hosting- und Dokumentations-Ökosystem wünschen, vergleichen Sie es mit [Next.js mit Drizzle und Better Auth](/guides/typescript/nextjs-drizzle-better-auth/).

Im Vergleich zu einem separaten Hono-Backend verfügt dieser Stack über weniger bereitstellbare Dienste und weniger Cross-Origin-Setup. Im Vergleich zu Next.js erhalten Sie ein TanStack Router-erstes mentales Modell und einen kleineren Framework-Footprint, aber ein jüngeres Ökosystem.
