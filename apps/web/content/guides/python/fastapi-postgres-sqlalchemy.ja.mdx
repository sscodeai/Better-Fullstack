使い慣れた非同期対応 Web フレームワークとリレーショナル データベースを備えた Python API が必要な場合は、このスタックを使用します。

```npm
npm create better-fullstack@latest my-fastapi-api -- \
  --ecosystem python \
  --python-web-framework fastapi \
  --database postgres \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff \
```

## これが生み出すもの

- FastAPI を使用する Python プロジェクト。
- PostgreSQL をデータベース オプションとして使用します。
- SQLAlchemy データベースアクセス用。
- 検証用の Pydantic。
- Python コードの品質については Ruff を使用します。
- 反復可能なスキャフォールディング用の Better Fullstack 構成ファイル。

## 生成された形状

スキャフォールドは、完全なモノリポジトリではなく、コンパクトな API サービスです。 FastAPI は HTTP 境界を所有し、SQLAlchemy は永続性を所有し、Pydantic はリクエストとレスポンスのコントラクトを所有します。生成されたアプリには、ヘルス チェック、サンプル ユーザーおよびポスト CRUD コード、データベース セッション接続、Alembic 構成、およびテストが含まれています。

```txt
my-fastapi-api/
├── pyproject.toml
├── alembic.ini
├── migrations/
│   ├── env.py
│   └── versions/
├── src/app/
│   ├── main.py
│   ├── database.py
│   ├── models.py
│   ├── schemas.py
│   └── crud.py
└── tests/
    ├── test_main.py
    └── test_database.py
```

デフォルトのデータベース URL はローカル SQLite にフォールバックするため、プロジェクトはすぐに起動できます。このガイドでは `--database postgres` を選択しているため、生成される依存関係には `psycopg[binary]` が含まれます。実際のターゲット データベースを実行する場合は、`DATABASE_URL` を PostgreSQL 接続文字列に設定します。

```bash
DATABASE_URL="postgresql+psycopg://postgres:postgres@localhost:5432/my_fastapi_api"
```

## いつ選択するか

API、AI に隣接するサービス、内部プラットフォーム、および Python エコシステムへのアクセスが重要なバックエンド サービスには、FastAPI を選択します。

明示的なルート関数、OpenAPI ファーストのリクエスト/レスポンス モデリング、および小規模なプロジェクト サーフェスが必要な場合は、FastAPI が Django よりも適しています。 Django は、管理規約、アプリ構造、成熟した組み込み Web フレームワークのデフォルトが必要な場合に適しています。

| 決定点 | FastAPI + SQLAlchemy | Django REST API |
| --- | --- | --- |
| プロジェクトの形状 | 明示的なモジュールを使用した小規模なサービス | 慣例が多い Django プロジェクト |
| API モデリング | ルートに近い Pydantic スキーマ | Django 規則に関する DRF シリアライザー/ビュー |
| 管理者/バックオフィス | 別途追加 | Django 管理エコシステムは自然な道です |
| 非同期隣接作業 | API および AI サービス境界に共通に適合 | 可能ですが、最小限ではありません |
| ORM の設定 | SQLAlchemy-最初 | Django ORM は従来型ですが、このスタックは SQLAlchemy の選択を記録します |

## 代表的なスニペット

生成されたモデル レイヤーは、SQLAlchemy 2 スタイルのマップされた列を使用します。データベース ロジックをルート ハンドラーに直接追加するのではなく、サンプル `User` モデルと `Post` モデルを拡張します。

```python
from datetime import datetime

from sqlalchemy import DateTime, Integer, String, func
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    pass


class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, index=True)
    email: Mapped[str] = mapped_column(String(255), unique=True, index=True, nullable=False)
    name: Mapped[str | None] = mapped_column(String(255), nullable=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        nullable=False,
    )
```

ルートはリクエストごとのデータベース セッションに依存し、永続性を `crud.py` に委任します。

```python
from fastapi import Depends, HTTPException
from sqlalchemy.orm import Session

from app import crud
from app.database import get_db
from app.schemas import UserCreate, UserResponse


@app.post("/users", response_model=UserResponse)
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    existing = crud.get_user_by_email(db, user.email)
    if existing is not None:
        raise HTTPException(status_code=400, detail="Email already registered")

    return crud.create_user(db, user)
```

## 移行とテスト

生成されたアプリは起動時にテーブルを作成して迅速なローカル ループを実行できますが、運用チームはデータベース変更履歴のソースとして Alembic 移行を使用する必要があります。

```bash
uv run ruff check .
uv run pytest
```

モデルを追加または変更する場合は、移行を作成し、適用する前に確認してください。

```bash
uv run alembic revision --autogenerate -m "add account status"
uv run alembic upgrade head
```

テストをルートの動作とデータベースの動作に分割してください。ルート テストでは FastAPI のテスト クライアントを使用し、HTTP ステータス コードをアサートする必要がありますが、データベース テストでは分離されたテスト エンジンまたはトランザクション境界を作成する必要があります。

## 導入メモ

Uvicorn などの ASGI サーバーの背後でアプリを実行します。運用環境では、新しいアプリケーション インスタンスを開始する前に、`DATABASE_URL` を設定し、広範な CORS オリジンを無効にし、移行を実行します。

```bash
uv run alembic upgrade head
uv run uvicorn app.main:app --host 0.0.0.0 --port "${PORT:-8000}"
```

デプロイメント プラットフォームがリポジトリ ルートから構築されている場合は、Python プロジェクトの依存関係が `pyproject.toml` からインストールされ、生成されたパッケージ レイアウトで `PYTHONPATH` または作業ディレクトリが指定されていることを確認してください。

## トラブルシューティング

| 症状 | チェック |
| --- | --- |
| `psycopg` 接続エラー | `DATABASE_URL` がアプリ ランタイムから到達可能な PostgreSQL URL を使用していることを確認します。 |
| テーブルはローカルに存在しますが、本番環境には存在しません | 起動前に実稼働データベースに対して Alembic 移行を実行します。 |
| `app.main` のインポート エラー | プロジェクトのルートから開始するか、ソース ディレクトリを一貫して構成します。 |
| CORS はローカルでは動作しますが、ブラウザーのデプロイでは失敗します | ワイルドカード CORS をデプロイされたフロントエンドのオリジンに置き換えます。 |
