친숙한 비동기 친화적 웹 프레임워크와 관계형 데이터베이스가 포함된 Python API을 원할 때 이 스택을 사용하세요.

```npm
npm create better-fullstack@latest my-fastapi-api -- \
  --ecosystem python \
  --python-web-framework fastapi \
  --database postgres \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff \
```

## 이것이 만드는 것

- FastAPI을 사용하는 Python 프로젝트.
- PostgreSQL을 데이터베이스 옵션으로 사용합니다.
- 데이터베이스 액세스용 SQLAlchemy.
- 검증을 위한 Pydantic.
- Python 코드 품질을 위한 Ruff.
- 반복 가능한 스캐폴딩을 위한 Better Fullstack 구성 파일입니다.

## 생성된 모양

스캐폴드는 완전한 모노레포가 아닌 소형 API 서비스입니다. FastAPI은 HTTP 경계를 소유하고, SQLAlchemy은 지속성을 소유하며, Pydantic은 요청 및 응답 계약을 소유합니다. 생성된 앱에는 상태 확인, 샘플 사용자 및 게시 CRUD 코드, 데이터베이스 세션 연결, Alembic 구성 및 테스트가 포함됩니다.

```txt
my-fastapi-api/
├── pyproject.toml
├── alembic.ini
├── migrations/
│   ├── env.py
│   └── versions/
├── src/app/
│   ├── main.py
│   ├── database.py
│   ├── models.py
│   ├── schemas.py
│   └── crud.py
└── tests/
    ├── test_main.py
    └── test_database.py
```

기본 데이터베이스 URL은 로컬 SQLite로 대체되므로 프로젝트가 즉시 부팅될 수 있습니다. 이 가이드에서는 `--database postgres`을 선택하므로 생성된 종속성에는 `psycopg[binary]`이 포함됩니다. 실제 대상 데이터베이스를 실행하려면 `DATABASE_URL`을 PostgreSQL 연결 문자열로 설정하세요.

```bash
DATABASE_URL="postgresql+psycopg://postgres:postgres@localhost:5432/my_fastapi_api"
```

## 그것을 선택하는 경우

Python 생태계 액세스가 중요한 APIs, AI 인접 서비스, 내부 플랫폼 및 백엔드 서비스에 대해 FastAPI을 선택하세요.

명시적인 경로 기능, OpenAPI 우선 요청/응답 모델링 및 소규모 프로젝트 표면을 원할 경우 FastAPI은 Django보다 더 적합합니다. Django는 관리 규칙, 앱 구조 및 성숙한 내장 웹 프레임워크 기본값을 원할 때 더 적합합니다.

| 결정 포인트 | FastAPI + SQLAlchemy | Django REST API |
| --- | --- | --- |
| 프로젝트 형태 | 명시적 모듈을 갖춘 소규모 서비스 | 컨벤션이 많은 Django 프로젝트 |
| API 모델링 | 경로에 가까운 Pydantic 스키마 | Django 규칙에 대한 DRF 직렬 변환기/뷰 |
| 관리/백오피스 | 별도로 추가 | Django 관리자 생태계는 자연스러운 경로입니다 |
| 비동기 인접 작업 | API 및 AI 서비스 경계에 대한 공통 적합성 | 가능하지만 덜 최소화됨 |
| ORM 선호도 | SQLAlchemy-첫번째 | Django ORM은 일반적이지만 이 스택은 SQLAlchemy 선택을 기록합니다. |

## 대표 조각

생성된 모델 레이어는 SQLAlchemy 2 스타일 매핑 열을 사용합니다. 경로 핸들러에 직접 데이터베이스 논리를 추가하는 대신 샘플 `User` 및 `Post` 모델을 확장하세요.

```python
from datetime import datetime

from sqlalchemy import DateTime, Integer, String, func
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    pass


class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, index=True)
    email: Mapped[str] = mapped_column(String(255), unique=True, index=True, nullable=False)
    name: Mapped[str | None] = mapped_column(String(255), nullable=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        nullable=False,
    )
```

경로는 요청별 데이터베이스 세션에 따라 달라지며 지속성을 `crud.py`에 위임합니다.

```python
from fastapi import Depends, HTTPException
from sqlalchemy.orm import Session

from app import crud
from app.database import get_db
from app.schemas import UserCreate, UserResponse


@app.post("/users", response_model=UserResponse)
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    existing = crud.get_user_by_email(db, user.email)
    if existing is not None:
        raise HTTPException(status_code=400, detail="Email already registered")

    return crud.create_user(db, user)
```

## 마이그레이션 및 테스트

생성된 앱은 빠른 로컬 루프를 위해 시작 시 테이블을 생성할 수 있지만, 프로덕션 팀은 Alembic 마이그레이션을 데이터베이스 변경 기록의 소스로 사용해야 합니다.

```bash
uv run ruff check .
uv run pytest
```

모델을 추가하거나 변경할 때 마이그레이션을 생성하고 적용하기 전에 검토하세요.

```bash
uv run alembic revision --autogenerate -m "add account status"
uv run alembic upgrade head
```

경로 동작과 데이터베이스 동작 간에 테스트를 분할하여 유지합니다. 경로 테스트는 FastAPI의 테스트 클라이언트를 사용하고 HTTP 상태 코드를 확인해야 하며, 데이터베이스 테스트는 격리된 테스트 엔진 또는 트랜잭션 경계를 생성해야 합니다.

## 배포 참고사항

Uvicorn과 같은 ASGI 서버 뒤에서 앱을 실행합니다. 프로덕션에서는 `DATABASE_URL`을 설정하고, 광범위한 CORS 원본을 비활성화하고, 새 애플리케이션 인스턴스를 시작하기 전에 마이그레이션을 실행하세요.

```bash
uv run alembic upgrade head
uv run uvicorn app.main:app --host 0.0.0.0 --port "${PORT:-8000}"
```

배포 플랫폼이 리포지토리 루트에서 빌드되는 경우 `pyproject.toml`에서 Python 프로젝트 종속성을 설치하고 `PYTHONPATH` 또는 생성된 패키지 레이아웃의 작업 디렉터리를 가리키는지 확인하세요.

## 문제 해결

| 징후 | 확인하다 |
| --- | --- |
| `psycopg` 연결 오류 | `DATABASE_URL`이 앱 런타임에서 연결할 수 있는 PostgreSQL URL을 사용하는지 확인하세요. |
| 테이블은 로컬에 존재하지만 프로덕션에는 존재하지 않습니다. | 부팅하기 전에 프로덕션 데이터베이스에 대해 Alembic 마이그레이션을 실행하세요. |
| `app.main`에 대한 가져오기 오류 | 프로젝트 루트에서 시작하거나 소스 디렉터리를 일관되게 구성하세요. |
| CORS는 로컬에서 작동하지만 브라우저 배포에서는 실패합니다. | 와일드카드 CORS를 배포된 프런트엔드 원본으로 바꿉니다. |

## 트레이드오프

FastAPI은 강력한 API 기본값입니다. 더 많은 배터리 포함 웹 프레임워크와 관리 스타일 규칙이 필요한 경우 Django과 비교하세요.
