當你需要一個 Python API，並希望它使用熟悉、適合 async-adjacent 工作的 web framework，同時搭配關係型資料庫時，使用這個 stack。

```npm
npm create better-fullstack@latest my-fastapi-api -- \
  --ecosystem python \
  --python-web-framework fastapi \
  --database postgres \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff \
```

## 創建內容

- 一個使用 FastAPI 的 Python 專案。
- PostgreSQL 作為資料庫選項。
- SQLAlchemy 用於資料庫存取。
- Pydantic 用於驗證。
- Ruff 用於 Python 程式碼品質。
- 一個 Better Fullstack configuration file，用於可重複的 scaffolding。

## 生成結構

這個 scaffold 是一個緊湊的 API service，而不是完整 monorepo。 FastAPI 負責 HTTP boundary，SQLAlchemy 負責 persistence，Pydantic 負責 request 和 response contracts。產生的 app 包含 health checks、範例 user 和 post CRUD code、database session wiring、Alembic configuration 和 tests。

```txt
my-fastapi-api/
├── pyproject.toml
├── alembic.ini
├── migrations/
│   ├── env.py
│   └── versions/
├── src/app/
│   ├── main.py
│   ├── database.py
│   ├── models.py
│   ├── schemas.py
│   └── crud.py
└── tests/
    ├── test_main.py
    └── test_database.py
```

預設 database URL 會 fallback 到本地 SQLite，讓專案可以立即啟動。因為本指南選擇了 `--database postgres`，產生的 dependencies 會包含 `psycopg[binary]`；當你要測試真實目標資料庫時，把 `DATABASE_URL` 設定為 PostgreSQL connection string。

```bash
DATABASE_URL="postgresql+psycopg://postgres:postgres@localhost:5432/my_fastapi_api"
```

## 何時選擇它

選擇 FastAPI 來建構 APIs、AI-adjacent services、internal platforms，以及需要存取 Python 生態的 backend services。

當你想要明確的 route functions、OpenAPI-first 的 request/response modeling，以及較小的項目 surface 時，FastAPI 比 Django 更合適。當你需要 Django 的 admin conventions、app structure 和成熟內建 web framework defaults 時，Django 比較合適。

| 決策點 | FastAPI + SQLAlchemy | Django REST API |
| --- | --- | --- |
| 專案形態 | 使用顯式模組的小型服務 | 約定更重的 Django project |
| API modeling | 靠近 routes 的 Pydantic schemas | 基於 Django conventions 的 DRF serializers/views |
| Admin/back-office | 單獨添加 | Django admin ecosystem 是自然路徑 |
| Async-adjacent work | 常適合 API 和 AI service boundaries | 可以做到，但不夠 minimal |
| ORM preference | SQLAlchemy-first | Django ORM 比較常規，雖然這個 stack 記錄的是 SQLAlchemy 選擇 |

## 代表性程式碼片段

產生的 model layer 使用 SQLAlchemy 2 風格的 mapped columns。擴充範例 `User` 和 `Post` models，而不是把 database logic 直接寫進 route handlers。

```python
from datetime import datetime

from sqlalchemy import DateTime, Integer, String, func
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    pass


class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, index=True)
    email: Mapped[str] = mapped_column(String(255), unique=True, index=True, nullable=False)
    name: Mapped[str | None] = mapped_column(String(255), nullable=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        nullable=False,
    )
```

Routes 依賴 per-request database session，並把 persistence 委託給 `crud.py`。

```python
from fastapi import Depends, HTTPException
from sqlalchemy.orm import Session

from app import crud
from app.database import get_db
from app.schemas import UserCreate, UserResponse


@app.post("/users", response_model=UserResponse)
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    existing = crud.get_user_by_email(db, user.email)
    if existing is not None:
        raise HTTPException(status_code=400, detail="Email already registered")

    return crud.create_user(db, user)
```

## 遷移和測試

產生的 app 可以在 startup 時建立 tables，以支援快速本地 loop；但生產團隊應把 Alembic migrations 作為 database change history 的來源。

```bash
uv run ruff check .
uv run pytest
```

當新增或修改 models 時，建立 migration，並在應用前 review：

```bash
uv run alembic revision --autogenerate -m "add account status"
uv run alembic upgrade head
```

測試應拆分 route behavior 和 database behavior。 Route tests 應使用 FastAPI 的 test client 並斷言 HTTP status codes；database tests 應建立隔離的 test engine 或 transaction boundary。

## 部署說明

在 ASGI server（例如 Uvicorn）後執行 app。生產環境中設定 `DATABASE_URL`，關閉過寬的 CORS origins，並在啟動新的 application instances 前執行 migrations。

```bash
uv run alembic upgrade head
uv run uvicorn app.main:app --host 0.0.0.0 --port "${PORT:-8000}"
```

如果你的 deployment platform 從 repository root 構建，確保它從 `pyproject.toml` 安裝 Python project dependencies，並把 `PYTHONPATH` 或 working directory 指向生成的 package layout。

## 故障排查

| 現象 | 檢查項 |
| --- | --- |
| `psycopg` connection errors | 確認 `DATABASE_URL` 使用 app runtime 可以存取的 PostgreSQL URL。 |
| Tables 本機存在但生產不存在 | 在啟動前，對 production database 運行 Alembic migrations。 |
| `app.main` import errors | 從 project root 啟動，或一致地設定 source directory。 |
| CORS 本機可用但瀏覽器 deploy 失敗 | 用已部署 frontend origin 取代 wildcard CORS。 |
