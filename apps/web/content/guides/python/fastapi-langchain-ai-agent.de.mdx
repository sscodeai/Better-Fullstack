Verwenden Sie diesen Stack, wenn Sie möchten, dass ein Python-API für den Anwendungscode AI vorbereitet wird.

```npm
npm create better-fullstack@latest my-python-ai-api -- \
  --ecosystem python \
  --python-web-framework fastapi \
  --python-orm sqlmodel \
  --python-validation pydantic \
  --python-ai langchain \
  --python-quality ruff
```

## Was dadurch entsteht

- Ein FastAPI-Projekt.
- LangChain als Python-Bibliotheksoption AI.
- SQLModel und Pydantic.
- Ruff für Codequalität.

## Erzeugte Form

Dieses Gerüst beginnt mit einem HTTP-Hilfsmodul API, SQLModel-Persistenz und LangChain. Die Routenschicht übernimmt die Validierung und das Streamen von Antworten, während das Clientmodul AI Modellaufrufe besitzt. Diese Trennung ist wichtig, sobald sich Eingabeaufforderungen, Abruf, Tools und Beharrlichkeit unterschiedlich schnell ändern.

```txt
my-python-ai-api/
├── pyproject.toml
├── alembic.ini
├── src/app/
│   ├── main.py
│   ├── database.py
│   ├── models.py
│   ├── crud.py
│   ├── langchain_client.py
│   ├── langchain_schemas.py
│   └── settings.py
└── tests/
    ├── test_main.py
    └── test_database.py
```

Zu den generierten Abhängigkeiten gehören `langchain` und `langchain-openai`. Legen Sie Anbieteranmeldeinformationen über Umgebungsvariablen fest, bevor Sie die AI-Endpunkte aufrufen.

```bash
OPENAI_API_KEY="sk-..."
DATABASE_URL="postgresql+psycopg://postgres:postgres@localhost:5432/my_python_ai_api"
```

## Wann man es wählt

Wählen Sie dies für AI APIs, Abrufworkflows, Agent-Backends und Python-Dienste, die eine Webgrenze für die Modell- oder Tool-Orchestrierung benötigen.

Wählen Sie dies gegenüber einer einfachen FastAPI-Datenbank API, wenn der erstklassige Workflow die Modellorchestrierung ist. Wählen Sie den Leitfaden SQLAlchemy, wenn die Persistenz im Mittelpunkt des Dienstes steht und AI möglicherweise später hinzugefügt wird.

| Brauchen | Dieser LangChain-Stapel | Einfach FastAPI API |
| --- | --- | --- |
| Chat-/Abschlussendpunkt | Generierter Ausgangspunkt | Von Hand hinzufügen |
| Streaming-Antworten | Im Routenmuster AI enthalten | Bei Bedarf hinzufügen |
| Datenmodellstil | SQLModel kombiniert Tabellen und Pydantic-Formen | SQLAlchemy Modelle plus Pydantic-Schemata |
| Schnelles/Werkzeugwachstum | Das Client-Modul AI gibt ein klares Zuhause | Sie wählen die Modulgrenze |
| Produktionsproblem | Anbieterschlüssel, Latenz, Wiederholungsversuche, Beobachtbarkeit | Zuerst Datenbank- und HTTP-Verhalten |

## Repräsentative Auszüge

SQLModel hält Persistenz und Validierung für kleine Dienste eng beieinander.

```python
from datetime import datetime

from pydantic import EmailStr
from sqlmodel import Field, SQLModel


class UserBase(SQLModel):
    email: EmailStr = Field(index=True, unique=True)
    name: str | None = Field(default=None, max_length=255)


class User(UserBase, table=True):
    __tablename__ = "users"

    id: int | None = Field(default=None, primary_key=True)
    created_at: datetime = Field(default_factory=datetime.utcnow)
    updated_at: datetime = Field(default_factory=datetime.utcnow)
```

Halten Sie die Route dünn und überlassen Sie `langchain_client.py` die Modelldetails.

```python
from fastapi.responses import StreamingResponse

from app.langchain_client import chat, chat_stream
from app.langchain_schemas import ChatRequest, ChatResponse


@app.post("/ai/chat", response_model=ChatResponse)
async def langchain_chat(request: ChatRequest):
    response = await chat(request.message, history=request.history)
    return ChatResponse(response=response)


@app.post("/ai/chat/stream")
async def langchain_chat_stream(request: ChatRequest):
    return StreamingResponse(
        chat_stream(request.message, history=request.history),
        media_type="text/event-stream",
    )
```

Wenn der Agent wächst, führen Sie explizite Toolfunktionen ein, anstatt Routen Eingabeaufforderungszeichenfolgen zusammenstellen zu lassen.

```python
from langchain_core.tools import tool


@tool
def lookup_customer(customer_id: str) -> str:
    """Return customer context for support workflows."""
    return customer_repository.summary(customer_id)
```

## Migrationen und Tests

Der generierte SQLModel-Pfad enthält die Alembic-Konfiguration. Verwenden Sie es, um Datenbankänderungen wiederholbar zu machen, insbesondere wenn Sie Konversationen, Toolaufrufe, Auswertungsspuren oder Dokumentblöcke speichern.

```bash
uv run ruff check .
uv run pytest
```

```bash
uv run alembic revision --autogenerate -m "add conversation tables"
uv run alembic upgrade head
```

Fügen Sie Tests auf zwei Ebenen hinzu: Routentests für die Anforderungsvalidierung und das Streaming-Verhalten sowie Clienttests rund um die Prompt-/Tool-Orchestrierung mit verspottetem Modellanbieter. Vermeiden Sie es, kostenpflichtige Modelle APIs in der normalen Unit-Test-Spur aufzurufen.

## Kompatibilitätshinweise

Python-Optionen AI werden über `--python-ai` ausgewählt. Dies unterscheidet sich vom Flag TypeScript `--ai`, das von den SDK-Auswahlmöglichkeiten der Web-App AI verwendet wird.

- `--python-ai langchain` ist eine Python-Ökosystemoption im Array-Stil. Zusätzliche Python-Bibliotheken AI können in Stapeln ausgewählt werden, die sie unterstützen.
- `--python-orm sqlmodel` eignet sich gut für kompakte API/Datenbankbeispiele, da Antwortschemata und Tabellenmodelle Basisklassen gemeinsam nutzen können.
- Der generierte `DATABASE_URL`-Fallback ist lokal SQLite. Verwenden Sie PostgreSQL, indem Sie `DATABASE_URL` festlegen.
- Für FastAPI-Streaming-Antworten muss das Bereitstellungsziel lang laufende HTTP-Antworten unterstützen.

## Bereitstellungshinweise

Stellen Sie dies wie einen FastAPI-Dienst bereit, aber planen Sie ein Budget für AI-spezifische Anliegen ein: Anbieteranmeldeinformationen, Anforderungszeitüberschreitungen, Wiederholungsversuche, Ratenbegrenzungen und Antwort-Streaming. Führen Sie Migrationen vor der Veröffentlichung durch und behalten Sie die Modell-/Anbietereinstellungen außerhalb des Images bei.

```bash
uv run alembic upgrade head
uv run uvicorn app.main:app --host 0.0.0.0 --port "${PORT:-8000}"
```

Fügen Sie für Produktionsagenten-Workflows Anforderungs-IDs und strukturierte Protokolle zu jedem Modellaufruf hinzu. Wenn Sie den Abruf einführen, halten Sie Einbettungen, Vektorindizes und Quelldokumente separat von der Anwendungsbereitstellung versioniert.

## Fehlerbehebung

| Symptom | Überprüfen |
| --- | --- |
| Die Route AI gibt Anbieterfehler zurück | Überprüfen Sie `OPENAI_API_KEY` oder die anbieterspezifischen Umgebungsvariablen. |
| Streaming funktioniert lokal, puffert aber in der Produktion | Bestätigen Sie, dass der Proxy/die Plattform Streaming-Antworten unterstützt. |
| Die Tests sind langsam oder unregelmäßig | Verspotten Sie LangChain/provider-Clients in Komponententests. |
| SQLModel-Tabellen fehlen | Führen Sie Alembic-Migrationen durch oder initialisieren Sie die Datenbank, bevor Sie Routen erreichen. |
