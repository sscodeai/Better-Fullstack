Utilisez cette pile lorsque vous souhaitez un Python API avec un framework Web asynchrone familier et une base de données relationnelle.

```npm
npm create better-fullstack@latest my-fastapi-api -- \
  --ecosystem python \
  --python-web-framework fastapi \
  --database postgres \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff \
```

## Ce que cela crée

- Un projet Python utilisant FastAPI.
- PostgreSQL comme option de base de données.
- SQLAlchemy pour l'accès à la base de données.
- Pydantic pour validation.
- Ruff pour la qualité du code Python.
- Un fichier de configuration Better Fullstack pour un échafaudage reproductible.

## Forme générée

L'échafaudage est un service API compact plutôt qu'un monorepo complet. FastAPI possède la limite HTTP, SQLAlchemy possède la persistance et Pydantic possède les contrats de demande et de réponse. L'application générée comprend des vérifications de l'état, un exemple de code utilisateur et post-CRUD, le câblage de la session de base de données, la configuration de l'Alembic et des tests.

```txt
my-fastapi-api/
├── pyproject.toml
├── alembic.ini
├── migrations/
│   ├── env.py
│   └── versions/
├── src/app/
│   ├── main.py
│   ├── database.py
│   ├── models.py
│   ├── schemas.py
│   └── crud.py
└── tests/
    ├── test_main.py
    └── test_database.py
```

L'URL de la base de données par défaut revient au SQLite local afin que le projet puisse démarrer immédiatement. Étant donné que ce guide sélectionne `--database postgres`, les dépendances générées incluent `psycopg[binary]` ; définissez `DATABASE_URL` sur une chaîne de connexion PostgreSQL lorsque vous souhaitez exercer la base de données cible réelle.

```bash
DATABASE_URL="postgresql+psycopg://postgres:postgres@localhost:5432/my_fastapi_api"
```

## Quand le choisir

Choisissez FastAPI pour les API, les services adjacents à AI, les plates-formes internes et les services backend pour lesquels l'accès à l'écosystème Python est important.

FastAPI est un meilleur ajustement que Django lorsque vous souhaitez des fonctions de routage explicites, une modélisation OpenAPI-première requête/réponse et une petite surface de projet. Django convient mieux lorsque vous souhaitez ses conventions d'administration, sa structure d'application et ses paramètres par défaut de framework Web intégré mature.

| Point de décision | FastAPI + SQLAlchemy | Django REST API |
| --- | --- | --- |
| Forme du projet | Petit service avec des modules explicites | Projet Django axé sur les conventions |
| modélisation API | Schémas pydantiques proches des routes | Sérialiseurs/vues DRF sur les conventions Django |
| Administrateur/back-office | Ajouter séparément | L'écosystème d'administration Django est la voie naturelle |
| Travail adjacent asynchrone | Ajustement commun aux limites de service API et AI | Possible, mais moins minime |
| Préférence ORM | SQLAlchemy-premier | Django ORM est conventionnel, bien que cette pile enregistre la sélection SQLAlchemy |

## Extraits représentatifs

La couche de modèle générée utilise des colonnes mappées de style SQLAlchemy 2. Étendez les exemples de modèles `User` et `Post` au lieu d'ajouter une logique de base de données directement dans les gestionnaires de routes.

```python
from datetime import datetime

from sqlalchemy import DateTime, Integer, String, func
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    pass


class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, index=True)
    email: Mapped[str] = mapped_column(String(255), unique=True, index=True, nullable=False)
    name: Mapped[str | None] = mapped_column(String(255), nullable=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        nullable=False,
    )
```

Les routes dépendent d'une session de base de données par requête et délèguent la persistance à `crud.py`.

```python
from fastapi import Depends, HTTPException
from sqlalchemy.orm import Session

from app import crud
from app.database import get_db
from app.schemas import UserCreate, UserResponse


@app.post("/users", response_model=UserResponse)
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    existing = crud.get_user_by_email(db, user.email)
    if existing is not None:
        raise HTTPException(status_code=400, detail="Email already registered")

    return crud.create_user(db, user)
```

## Migrations et tests

L'application générée peut créer des tables au démarrage pour une boucle locale rapide, mais les équipes de production doivent utiliser les migrations Alambic comme source de l'historique des modifications de la base de données.

```bash
uv run ruff check .
uv run pytest
```

Lorsque vous ajoutez ou modifiez des modèles, créez une migration et examinez-la avant de l'appliquer :

```bash
uv run alembic revision --autogenerate -m "add account status"
uv run alembic upgrade head
```

Gardez les tests répartis entre le comportement de la route et le comportement de la base de données. Les tests de route doivent utiliser le client de test de FastAPI et affirmer les codes d'état HTTP, tandis que les tests de base de données doivent créer un moteur de test isolé ou une limite de transaction.

## Notes de déploiement

Exécutez l'application derrière un serveur ASGI tel qu'Uvicorn. En production, définissez `DATABASE_URL`, désactivez les origines CORS larges et exécutez des migrations avant de démarrer de nouvelles instances d'application.

```bash
uv run alembic upgrade head
uv run uvicorn app.main:app --host 0.0.0.0 --port "${PORT:-8000}"
```

Si votre plate-forme de déploiement est construite à partir de la racine du référentiel, assurez-vous qu'elle installe les dépendances du projet Python à partir de `pyproject.toml` et pointe `PYTHONPATH` ou le répertoire de travail sur la disposition du package généré.

## Dépannage

| Symptôme | Vérifier |
| --- | --- |
| `psycopg` erreurs de connexion | Confirmez que `DATABASE_URL` utilise une URL PostgreSQL accessible depuis le runtime de l'application. |
| Les tables existent localement mais pas en production | Exécutez les migrations Alambic sur la base de données de production avant le démarrage. |
| Erreurs d'importation pour `app.main` | Commencez à partir de la racine du projet ou configurez le répertoire source de manière cohérente. |
| CORS fonctionne localement mais échoue lors du déploiement d'un navigateur | Remplacez le CORS générique par l’origine du frontend déployé. |

## Compromis

FastAPI est une valeur par défaut forte de API. Si vous avez besoin d'un framework Web comprenant davantage de batteries et de conventions de style administrateur, comparez-le avec Django.
