Verwenden Sie diesen Stack, wenn Sie ein Next.js-Projekt für AI-unterstützte Entwicklungsworkflows, lokale MCP-Tools und generierte Agentendokumentation vorbereiten möchten.

```npm
npm create better-fullstack@latest my-agent-workbench -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --ai ai-cli \
  --addons mcp skills \
  --ai-docs claude-md agents-md cursorrules \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## Was dadurch entsteht

- Eine Next.js-App mit einem lokalen Fullstack-Setup.
- SQLite und Drizzle.
- Better Auth.
- AI CLI Konfiguration.
- MCP und Skills-Add-ons.
- Agentenbezogene Dokumentationsdateien wie `CLAUDE.md`, `AGENTS.md` oder `.cursorrules`, abhängig von den ausgewählten Flags.

## Erzeugte Form

Diese Voreinstellung ist sowohl ein Anwendungsgerüst als auch ein Agent-Workflow-Gerüst. Die Web-App ist ein normales Next.js-Fullstack-Projekt, während die Add-ons einen Repository-Kontext für Coding-Agents und lokale AI-Tools erstellen.

Repräsentative Form:

```txt
my-agent-workbench/
  bts.jsonc
  AGENTS.md
  CLAUDE.md
  .cursorrules
  app/
  components/
  lib/
    auth/
    db/
```

Die genauen Dokumentationsdateien hängen von `--ai-docs claude-md agents-md cursorrules` ab. Der angezeigte Befehl fragt nach allen drei, damit verschiedene Agententools Projektanleitungen in ihrem bevorzugten Format finden können.

## Wie die AI-Teile passen

`--ai ai-cli` wählt die Unterstützung von AI CLI aus. `--addons mcp skills` fügt lokale Agentenfunktionen rund um MCP und Fertigkeiten hinzu. `--ai-docs` steuert, welche Repository-Anweisungsdateien generiert werden.

Verwenden Sie diese Teile für verschiedene Aufgaben:

| Stück | Zweck | Typische Verwendung |
| --- | --- | --- |
| AI CLI | Lokaler Befehlszeilen-Workflow AI | Ausführen von agentenunterstützten Projektaufgaben |
| MCP | Werkzeug-/Kontextbrücke | ExpoProjekt- oder Servicefunktionen an Agenten weitergeben |
| Fähigkeiten | Anweisungen für wiederverwendbare Agenten | Standardisierung von Arbeitsabläufen wie Debugging oder Feature-Arbeit |
| Agentendokumente | Repository-Anleitung | Den Agenten mitteilen, wie dieses Projekt strukturiert ist |

## Beispiel für eine Agentenanweisung

Dokumente für den Agenten sollten konkret sein. Bevorzugen Sie Projektregeln, die befolgt und überprüft werden können:

```md
## Verification

After editing authentication, run the smallest auth-related test or typecheck.
Do not run broad workspace commands unless the change crosses package boundaries.

## Project boundaries

Database access belongs in server-only modules.
Client components should call typed API helpers instead of importing Drizzle code.
```

Halten Sie diese Dokumente auf dem neuesten Stand, wenn sich die Architektur ändert. Veraltete Agentenanweisungen sind schlimmer als keine Anweisungen, da sie zukünftige Arbeiten sicher an die falschen Dateien weiterleiten.

## Beispielanwendungsmodell

Die App-Seite bleibt ein normaler Better Fullstack TypeScript-Stack. Eine kleine SQLite-Tabelle könnte so aussehen:

```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const agentRun = sqliteTable("agent_run", {
  id: text("id").primaryKey(),
  goal: text("goal").notNull(),
  status: text("status").notNull(),
  createdAt: integer("created_at", { mode: "timestamp" }).notNull(),
});
```

Verwenden Sie Anwendungstabellen für Produktdaten, nicht zum Speichern von Geheimnissen. MCP-Anmeldeinformationen, Anbieter-Tokens und AI-Dienstschlüssel sollten sich in Umgebungsvariablen oder einem anbieterspezifischen sicheren Speicher befinden.

## Wann man es wählt

Wählen Sie diese Option, wenn das Repository selbst für Codierungsagenten leicht zu verstehen und zu ändern sein soll. Es ist nützlich für Teams, die Agenten-Workflows, MCP-Tools und generierten Projektkontext standardisieren.

## Kompatibilitätshinweise

- Diese Voreinstellung verwendet eine TypeScript Next.js-App mit `--backend self`, sodass Next.js die Servergrenze besitzt.
- `--addons mcp skills` sind Ergänzungen zum Entwicklungs-Workflow. Sie ersetzen nicht die App-Authentifizierung, Datenbank oder API-Auswahlmöglichkeiten.
- `--ai-docs claude-md agents-md cursorrules` generiert absichtlich mehrere Anweisungsformate für verschiedene Werkzeuge.
- SQLite ist praktisch für lokale Projekte im Workbench-Stil. Verwenden Sie PostgreSQL, wenn die Workbench zu einem Mehrbenutzer-Produktionsprodukt mit gemeinsamem Status wird.

## Bereitstellungshinweise

Stellen Sie die Next.js-App wie eine normale Fullstack-Anwendung bereit: Konfigurieren Sie Better Auth-Geheimnisse, App-URL und Datenbankeinstellungen. Behandeln Sie AI-Anbieterschlüssel, MCP-Anmeldeinformationen und alle Tool-Tokens als serverseitige Geheimnisse.

Agentendokumente gehören normalerweise zur Quellcodeverwaltung. Lokale Anmeldeinformationen, generierte Protokolle und maschinenspezifische MCP-Konfigurationen funktionieren normalerweise nicht. Überprüfen Sie die generierten Ignorierungsdateien, bevor Sie sie festschreiben.

## Fehlerbehebung

- Wenn ein AI-Tool die Repository-Anleitung ignoriert, bestätigen Sie, dass es den generierten Dateinamen liest, den Sie mit `--ai-docs` ausgewählt haben.
- Wenn die Tools MCP nicht verfügbar sind, überprüfen Sie das lokale MCP-Setup und ob der konsumierende Agent darauf hingewiesen wurde.
- Wenn während der Skriptausführung Eingabeaufforderungen für das Scaffolded-Add-on-Setup angezeigt werden, verwenden Sie explizite Flags wie in diesem Handbuch, damit die Standardeinstellungen deterministisch sind.
- Wenn die App lokal funktioniert, aber die Authentifizierung nach der Bereitstellung fehlschlägt, überprüfen Sie dieselben Better Auth-URL- und Cookie-Einstellungen, die Sie in jedem Next.js-Stack überprüfen würden.

## Kompromisse

Dieser Stack ist eigensinniger als ein minimaler Next.js-Starter. Wenn Sie nur eine Produkt-App ohne Agent-Tooling benötigen, verwenden Sie [Next.js mit Drizzle und Better Auth](/guides/typescript/nextjs-drizzle-better-auth/).

Die zusätzlichen Dateien sind wertvoll, wenn Menschen und Agenten wiederholt im Repo arbeiten. Für einen Wegwerf-Prototyp sind sie ein Overhead. Wenn das Team die Anweisungen nicht einhält, beginnen Sie mit der einfacheren Anleitung Next.js und fügen Sie später die Dokumentation AI hinzu.
