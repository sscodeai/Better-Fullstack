AI 지원 개발 워크플로우, 로컬 MCP 도구 및 생성된 에이전트 문서를 위해 준비된 Next.js 프로젝트를 원할 때 이 스택을 사용하십시오.

```npm
npm create better-fullstack@latest my-agent-workbench -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --ai ai-cli \
  --addons mcp skills \
  --ai-docs claude-md agents-md cursorrules \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## 이것이 만드는 것

- 로컬 풀스택이 설정된 Next.js 앱.
- SQLite 및 Drizzle.
- Better Auth.
- AI CLI 구성.
- MCP 및 스킬 추가 기능.
- 선택한 플래그에 따라 `CLAUDE.md`, `AGENTS.md` 또는 `.cursorrules`와 같은 에이전트 관련 문서 파일입니다.

## 생성된 모양

이 사전 설정은 애플리케이션 스캐폴드이자 에이전트 워크플로 스캐폴드입니다. 웹 앱은 일반적인 Next.js 풀스택 프로젝트인 반면, 추가 기능은 코딩 에이전트 및 로컬 AI 도구를 위한 저장소 컨텍스트를 생성합니다.

대표적인 형태:

```txt
my-agent-workbench/
  bts.jsonc
  AGENTS.md
  CLAUDE.md
  .cursorrules
  app/
  components/
  lib/
    auth/
    db/
```

정확한 문서 파일은 `--ai-docs claude-md agents-md cursorrules`에 따라 다릅니다. 표시된 명령은 서로 다른 에이전트 도구가 선호하는 형식으로 프로젝트 지침을 찾을 수 있도록 세 가지 모두를 요구합니다.

## AI 조각이 맞는 방법

`--ai ai-cli`은 AI CLI 지원을 선택합니다. `--addons mcp skills`은 MCP 및 기술에 대한 로컬 에이전트 기능을 추가합니다. `--ai-docs`는 생성되는 저장소 지침 파일을 제어합니다.

다양한 작업에 해당 조각을 사용하십시오.

| 조각 | 목적 | 일반적인 사용 |
| --- | --- | --- |
| AI CLI | 로컬 명령줄 AI 작업 흐름 | 에이전트 지원 프로젝트 작업 실행 |
| MCP | 도구/컨텍스트 브리지 | Expo에이전트에게 프로젝트 또는 서비스 기능을 전달합니다. |
| 기술 | 재사용 가능한 에이전트 지침 | 디버깅이나 기능 작업 등의 워크플로 표준화 |
| 상담원 문서 | 저장소 안내 | 이 프로젝트가 어떻게 구성되어 있는지 상담원에게 알리기 |

## 에이전트 지침 예시

상담원이 접하는 문서는 구체적이어야 합니다. 따르고 검증할 수 있는 프로젝트 규칙을 선호합니다.

```md
## Verification

After editing authentication, run the smallest auth-related test or typecheck.
Do not run broad workspace commands unless the change crosses package boundaries.

## Project boundaries

Database access belongs in server-only modules.
Client components should call typed API helpers instead of importing Drizzle code.
```

아키텍처가 변경되면 이 문서를 최신 상태로 유지하세요. 오래된 에이전트 지침은 향후 작업을 잘못된 파일에 확실하게 라우팅하기 때문에 지침이 없는 것보다 나쁩니다.

## 예시 애플리케이션 모델

앱 측은 일반 Better Fullstack TypeScript 스택으로 유지됩니다. 작은 SQLite 테이블은 다음과 같습니다:

```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const agentRun = sqliteTable("agent_run", {
  id: text("id").primaryKey(),
  goal: text("goal").notNull(),
  status: text("status").notNull(),
  createdAt: integer("created_at", { mode: "timestamp" }).notNull(),
});
```

비밀을 저장하는 용도가 아닌 제품 데이터용으로 애플리케이션 테이블을 사용하세요. MCP 자격 증명, 공급자 토큰 및 AI 서비스 키는 환경 변수 또는 공급자별 보안 저장소에 있어야 합니다.

## 그것을 선택하는 경우

코딩 에이전트가 리포지토리 자체를 쉽게 이해하고 수정할 수 있어야 하는 경우 이 옵션을 선택하세요. 에이전트 워크플로, MCP 도구 및 생성된 프로젝트 컨텍스트를 표준화하는 팀에 유용합니다.

## 호환성 참고사항

- 이 사전 설정은 `--backend self`과 함께 TypeScript Next.js 앱을 사용하므로 Next.js이 서버 경계를 소유합니다.
- `--addons mcp skills`은 개발 워크플로 추가 사항입니다. 앱 인증, 데이터베이스 또는 API 선택 사항을 대체하지 않습니다.
- `--ai-docs claude-md agents-md cursorrules`은 다양한 도구에 대해 의도적으로 여러 명령 형식을 생성합니다.
- SQLite은 로컬 워크벤치 스타일 프로젝트에 편리합니다. 워크벤치가 공유 상태를 갖는 다중 사용자 프로덕션 제품이 되는 경우 PostgreSQL을 사용하십시오.

## 배포 참고사항

일반 풀스택 애플리케이션처럼 Next.js 앱을 배포합니다. Better Auth 비밀, 앱 URL 및 데이터베이스 설정을 구성합니다. AI 공급자 키, MCP 자격 증명 및 모든 도구 토큰을 서버 측 비밀로 처리합니다.

에이전트 문서는 일반적으로 소스 제어에 속합니다. 로컬 자격 증명, 생성된 로그 및 컴퓨터별 MCP 구성은 일반적으로 그렇지 않습니다. 커밋하기 전에 생성된 무시 파일을 검토하세요.

## 문제 해결

- AI 도구가 저장소 지침을 무시하는 경우 `--ai-docs`로 선택한 생성된 파일 이름을 읽는지 확인하세요.
- MCP 도구를 사용할 수 없는 경우 로컬 MCP 설정과 소비 에이전트가 이를 가리키고 있는지 확인하세요.
- 스크립트 실행 중에 스캐폴드 애드온 설정 메시지가 표시되면 이 가이드와 같이 명시적인 플래그를 사용하여 기본값이 결정적이 되도록 하세요.
- 앱이 로컬에서 작동하지만 배포 후 인증이 실패하는 경우 Next.js 스택에서 검사할 것과 동일한 Better Auth URL 및 쿠키 설정을 확인하세요.

## 트레이드오프

이 스택은 최소 Next.js 스타터보다 더 독선적입니다. 에이전트 도구 없이 제품 앱만 필요한 경우 [Next.js with Drizzle 및 Better Auth](/guides/typescript/nextjs-drizzle-better-auth/)를 사용하세요.

추가 파일은 사람과 에이전트가 저장소에서 반복적으로 작업할 때 유용합니다. 일회용 프로토타입의 오버헤드입니다. 팀이 지침을 유지하지 않을 경우 더 간단한 Next.js 가이드로 시작하고 나중에 AI 문서를 추가하세요.
