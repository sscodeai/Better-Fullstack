Usa el pack Rust Backend cuando quieras un servicio API compacto con tipos fuertes, rendimiento predecible y estructura reforzada por el compilador.

<StarterPackCta id="rust-backend" />

## Generar el starter de backend Rust

Abre el [pack Rust Backend en el Stack Builder](/new?preset=rust-api&view=command), o ejecuta:

```bash
bun create better-fullstack@latest my-rust-backend \
  --ecosystem rust \
  --rust-web-framework axum \
  --rust-frontend none \
  --rust-orm sea-orm \
  --rust-logging tracing \
  --rust-error-handling anyhow-thiserror
```

## Qué incluye el pack

- Axum para rutas HTTP.
- SeaORM para acceso tipado a base de datos.
- Una forma de backend orientada a PostgreSQL.
- Tracing para logs estructurados de aplicación.
- Patrones anyhow y thiserror para manejo práctico de errores.

## Primeras cosas para personalizar

1. Define tus módulos de rutas alrededor de recursos o workflows.
2. Añade entidades de base de datos y migraciones para el dominio real.
3. Mantén explícitos los errores de handlers para que las fallas de API sean fáciles de depurar.
4. Ejecuta `cargo check` y `cargo clippy -- -D warnings` a medida que el proyecto crezca.

## Cuándo este pack encaja

Elige esto para servicios backend de alta confianza, APIs con expectativas estrictas de rendimiento, herramientas de sistemas con superficies HTTP y equipos que prefieren las garantías compile-time de Rust.

Para un walkthrough de stack, lee [Axum con PostgreSQL y SeaORM](/guides/rust/axum-postgres-seaorm/).
