SaaS 앱을 만드는 방법을 찾고 있다면 SaaS 팩부터 시작하세요. 첫 번째 고객이 가입하기 전에 대부분의 구독 제품에 필요한 계정, 데이터베이스 테이블, 청구서, 이메일, UI 및 유형화된 앱 구조를 제공합니다.

<StarterPackCta id="saas-app" />

## SaaS 스타터 생성

[Stack Builder의 SaaS 팩](/new?preset=nextjs-saas&view=command)을 열거나 아래 명령을 실행하세요.

```bash
bun create better-fullstack@latest my-saas-app \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --payments stripe \
  --email react-email \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui \
  --package-manager bun
```

## 팩에 포함된 내용

- Next.js을 웹 및 서버 경계로 사용합니다.
- 관계형 제품 데이터의 경우 PostgreSQL 및 Drizzle
- 로그인, 세션 및 계정 흐름을 위한 Better Auth
- 구독 청구를 위한 스트라이프입니다.
- 거래 이메일 템플릿을 위한 React Email.
- 제품 인터페이스의 경우 Tailwind CSS 및 shadcn/ui입니다.
- React UI와 서버 코드 간의 입력된 호출을 위한 tRPC.

## 가장 먼저 사용자 정의해야 할 사항

1. 생성된 제품 이름, 메타데이터, 탐색을 바꿉니다.
2. Drizzle 스키마 옆에 핵심 제품 테이블을 추가합니다.
3. Better Auth 비밀 및 신뢰할 수 있는 원본을 구성합니다.
4. Stripe 제품을 생성하고 이를 앱 계획에 매핑하세요.
5. 가입, 청구, 팀 초대를 위한 거래 이메일을 추가하세요.

## 이것이 올바른 팩일 때

구독 제품, B2B 대시보드, 유료 개발자 도구, 팀 작업 공간 및 인증과 청구가 첫 번째 작업 버전의 일부여야 하는 제품에 이 팩을 사용하세요.

스택 수준 연습을 보려면 [Next.js with Drizzle 및 Better Auth](/guides/typescript/nextjs-drizzle-better-auth/)을 읽어보세요.
