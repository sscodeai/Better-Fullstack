當你希望應用程式和代理程式工作流程一起起步時，請使用 AI Agent App pack。它會準備一個 Next.js 項目，並加入 AI CLI 的文件和本機代理工具。

<StarterPackCta id="ai-agent-app" />

## 產生 AI 代理程式 starter

開啟 [Stack Builder 中的 AI Agent App pack](/new?preset=ai-cli-agent-workbench&view=command)，或執行：

```bash
bun create better-fullstack@latest my-ai-agent-app \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --ai ai-cli \
  --addons biome ruler turborepo mcp skills \
  --ai-docs claude-md agents-md cursorrules
```

## Pack 包含什麼

- Next.js 用於產品介面。
- SQLite 和 Drizzle 用於本地優先的持久化。
- Better Auth 用於帳戶和會話。
- AI CLI 設定、MCP 支援、skills，以及產生的代理文件。
- Biome 和 Ruler，讓代理指令和程式碼品質規則與倉庫放在一起。

## 首先要自訂的內容

1. 定義面向使用者的代理任務：研究、編碼、支援、自動化或分析。
2. 加入代理程式允許讀取和寫入的應用資料模型。
3. 在連接外部服務之前，請收緊 MCP/tool 指令。
4. 當協作或生產託管需要時，從 SQLite 遷移到託管資料庫。

## 什麼時候適合這個 pack

選擇它來建立 AI 輔助開發工具、內部自動化、代理儀表板、工作流程助手，或那些應用結構和代理指令需要一起演進的實驗。

如需技術堆疊層面的 walkthrough，請閱讀 [Next.js AI CLI 代理程式工作台](/guides/ai/nextjs-ai-cli-agent-workbench/)。
