## Cuándo usar un pack

Usa un pack cuando quieres un buen valor predeterminado y un proyecto funcional rápidamente. Después de generarlo, el Stack Builder todavía te permite ajustar framework, base de datos, auth, API, despliegue y tooling.

Si ya conoces el stack exacto, usa el [Stack Builder](/new?view=presets) o una guía específica de stack.

## Notas de selección

- Elige el pack SaaS cuando billing, auth y email formen parte de la primera versión del producto.
- Elige el pack AI Agent App cuando las instrucciones de agentes, MCP y el código de app deban evolucionar juntos.
- Elige los packs REST API, Java API o Rust Backend cuando el producto empiece como un límite de servicio.
- Elige el pack Mobile App cuando la experiencia en dispositivo importe antes que la arquitectura backend.
- Elige el pack Internal Tool cuando workflows CRUD, dashboards y operaciones autenticadas sean el producto.
