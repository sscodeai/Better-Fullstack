Les packs de démarrage sont des chemins avisés à travers Better Fullstack. Utilisez-les lorsque vous connaissez le résultat du produit souhaité avant de connaître chaque choix de bibliothèque.

Chaque pack explique ce qu'il faut créer, quelle pile il utilise, comment la générer et ce qu'il faut changer en premier après l'échafaudage.

## Quand utiliser un pack

Utilisez un pack lorsque vous souhaitez une valeur par défaut forte et un projet fonctionnel rapidement. Après la génération, le Stack Builder vous permet toujours d'ajuster les choix de framework, de base de données, d'authentification, API, de déploiement et d'outils.

Si vous connaissez déjà la pile exacte, utilisez plutôt le [Stack Builder](/new?view=presets) ou un guide spécifique à la pile.

## Notes de sélection des packs

- Choisissez le pack SaaS lorsque la facturation, l'authentification et la messagerie électronique font partie de la première version du produit.
- Choisissez le pack d'applications d'agent AI lorsque les instructions de l'agent, MCP et le code de l'application doivent évoluer ensemble.
- Choisissez les packs REST API, Java API ou Rust Backend lorsque le produit démarre en tant que limite de service.
- Choisissez le pack d'applications mobiles lorsque l'expérience de l'appareil compte avant l'architecture backend.
- Choisissez le pack d'outils internes lorsque les workflows CRUD, les tableaux de bord et les opérations authentifiées sont le produit.
