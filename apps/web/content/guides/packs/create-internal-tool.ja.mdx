## 内部ツールスターターを生成する

[Stack Builder の内部ツール パック](/new?preset=tanstack-hono&view=command) を開くか、次のコマンドを実行します。

```bash
bun create better-fullstack@latest my-internal-tool \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime bun \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui
```

## パックの内容

- Web アプリの場合は TanStack Router。
- バックエンドの Bun 上の Hono。
- 型付き永続性の PostgreSQL および Drizzle。
- Better Auth は保護された内部アクセス用です。
- tRPC は、型指定されたフロントエンドからバックエンドへの呼び出しの場合です。
- ダッシュボード UI の Tailwind CSS および shadcn/ui。

## 最初にカスタマイズすること

1. チームが毎日編集する運用オブジェクトをモデル化します。
2. 破壊的なアクションを公開する前にロール チェックを追加します。
3. 同じ型の API に基づいて、リスト、詳細、作成、編集、および監査ビューを構築します。
4. チームメイトを招待する前に、展開シークレットとデータベース移行を追加します。

## これが適切なパックの場合

管理パネル、内部ダッシュボード、サポート ツール、バックオフィス ワークフロー、速度は重要だが型の安全性も重要な CRUD アプリにはこれを選択してください。

スタックレベルのチュートリアルについては、[Hono と tRPC および Drizzle](/guides/typescript/hono-trpc-drizzle/) を参照してください。
