當第一個介面應該是 native 而不是 web 時，請使用 Mobile App pack。它從 Expo 和 Uniwind 開始，讓你可以先建立移動介面，再新增 backend services。

<StarterPackCta id="mobile-app" />

## 生成 mobile starter

開啟 [Stack Builder 中的 Mobile App pack](/new?preset=uniwind&view=command)，或執行：

```bash
bun create better-fullstack@latest my-mobile-app \
  --ecosystem typescript \
  --frontend native-uniwind \
  --backend none \
  --database none \
  --api none \
  --css-framework none \
  --ui-library none
```

## Pack 包含什麼

- Expo 用於 native app runtime。
- React Native 作為 UI foundation。
- Uniwind 用於 utility-first native styling。
- 一個 frontend-only 的專案結構，在需要 backend choices 前保持輕量。

## 首先要自訂的內容

1. 用核心移動流程取代範例 screens。
2. 決定 auth 應該是 native-only、API-backed，還是與 web app 共用。
3. 只有在 app 的 data boundaries 清晰後，再加入 backend services。
4. 儘早在小螢幕和大螢幕裝置尺寸上驗證 layouts。

## 什麼時候適合這個 pack

選擇它來建構 mobile-first 產品、companion apps、需要 native feel 的 prototypes，以及想先從設備體驗開始、再選擇 server infrastructure 的團隊專案。

如需 stack 層面的 walkthrough，請閱讀 [Expo Uniwind 原生應用程式](/guides/react-native/expo-uniwind-native/)。
