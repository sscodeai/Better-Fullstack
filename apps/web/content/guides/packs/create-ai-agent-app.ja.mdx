アプリケーションとエージェント ワークフローを同時に開始する場合は、AI エージェント アプリ パックを使用します。これは、Next.js プロジェクトに加えて、AI CLI 指向のドキュメントとローカル エージェント ツールを準備します。

<StarterPackCta id="ai-agent-app" />

## AI エージェント スターターを生成する

[Stack Builder の AI エージェント アプリ パック](/new?preset=ai-cli-agent-workbench&view=command) を開くか、次のコマンドを実行します。

```bash
bun create better-fullstack@latest my-ai-agent-app \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --ai ai-cli \
  --addons biome ruler turborepo mcp skills \
  --ai-docs claude-md agents-md cursorrules
```

## パックの内容

- 製品表面のNext.js。
- ローカルファーストの永続性のための SQLite および Drizzle。
- アカウントとセッションの場合は Better Auth。
- AI CLI セットアップ、MCP サポート、スキル、および生成されたエージェント ドキュメント。
- Biome と Ruler により、エージェントの指示とコードの品質がリポジトリとともに維持されます。

## 最初にカスタマイズすること

1. ユーザー対応エージェントのジョブを定義します: 調査、コーディング、サポート、自動化、または分析。
2. エージェントが読み取りと書き込みを許可されるアプリ データ モデルを追加します。
3. 外部サービスに接続する前に、MCP/tool の指示を厳密に行ってください。
4. コラボレーションまたは運用ホスティングが必要な場合は、SQLite から管理されたデータベースに移行します。

## これが適切なパックの場合

AI 支援の開発者ツール、内部自動化、エージェント ダッシュボード、ワークフロー コパイロット、またはアプリの構造とエージェントの指示を一緒に進化させる必要がある実験には、これを選択してください。

スタックレベルのチュートリアルについては、[Next.js AI CLI Agent Workbench](/guides/ai/nextjs-ai-cli-agent-workbench/) を参照してください。
