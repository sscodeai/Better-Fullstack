Starter packs 是 Better Fullstack 中带有明确取向的路径。当你先知道想要的产品结果，但还没有决定每个 library 选择时，可以使用它们。

每个 pack 都会说明要创建什么、它使用哪个 stack、如何生成，以及 scaffold 之后最先应该改什么。

## 什么时候使用 pack

当你想要一个强默认值，并快速得到可运行项目时，请使用 pack。生成之后，Stack Builder 仍然可以调整 framework、数据库、auth、API、部署和 tooling 选择。

如果你已经知道确切 stack，请使用 [Stack Builder](/new?view=presets) 或某篇具体 stack 指南。

## Pack 选择说明

- 当 billing、auth 和 email 是产品第一版的一部分时，选择 SaaS pack。
- 当代理指令、MCP 和应用代码需要一起演进时，选择 AI Agent App pack。
- 当产品从服务边界开始时，选择 REST API、Java API 或 Rust Backend pack。
- 当设备体验比 backend 架构更早成为重点时，选择 Mobile App pack。
- 当 CRUD workflows、dashboards 和认证操作就是产品时，选择 Internal Tool pack。
