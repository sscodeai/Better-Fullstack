如果你正在搜尋如何建立 SaaS 應用程式，請從 SaaS pack 開始。它提供大多數訂閱產品在第一個客戶註冊前就需要的部分：帳戶、資料庫表、billing、email、UI，以及類型化的應用程式結構。

## 生成 SaaS starter

開啟 [Stack Builder 中的 SaaS pack](/new?preset=nextjs-saas&view=command)，或執行下面的指令。

```bash
bun create better-fullstack@latest my-saas-app \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --payments stripe \
  --email react-email \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui \
  --package-manager bun
```

## Pack 包含什麼

- Next.js 作為 web 和 server 邊界。
- PostgreSQL 搭配 Drizzle，用於關係型產品資料。
- Better Auth 處理 sign-in、sessions 和帳戶流程。
- Stripe 用於訂閱 billing。
- React Email 用於交易郵件範本。
- Tailwind CSS 和 shadcn/ui 用於產品介面。
- tRPC 用於 React UI 與 server 程式碼之間的類型化呼叫。

## 首先要自訂的內容

1. 替換生成的產品名稱、metadata 和導航。
2. 在 Drizzle schema 旁邊加入你的核心產品表。
3. 配置 Better Auth secrets 和 trusted origins。
4. 創建 Stripe products，並把它們映射到你的 app plans。
5. 為 signup、billing 和團隊邀請加入事務郵件。

## 什麼時候適合這個 pack

將這個 pack 用於訂閱產品、B2B dashboards、付費 developer tools、團隊 workspaces，以及 auth 加 billing 必須出現在第一版可運行產品中的場景。

如需 stack 層面的 walkthrough，請閱讀 [Next.js 與 Drizzle 和 Better Auth](/guides/typescript/nextjs-drizzle-better-auth/)。
