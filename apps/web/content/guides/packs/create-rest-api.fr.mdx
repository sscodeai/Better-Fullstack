Utilisez le pack REST API lorsque vous avez besoin d'un service backend ciblé : routes, validation, persistance et linting sans frontend.

<StarterPackCta id="rest-api" />

## Générer le démarreur REST API

Ouvrez le pack [REST API dans le Stack Builder](/new?preset=python-fastapi&view=command) ou exécutez :

```bash
bun create better-fullstack@latest my-rest-api \
  --ecosystem python \
  --python-web-framework fastapi \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff
```

## Ce que comprend le pack

- FastAPI pour les points de terminaison REST et les documents OpenAPI.
- SQLAlchemy pour les modèles de base de données et les requêtes.
- Pydantic pour la validation des demandes et des réponses.
- Ruff pour le peluchage et le formatage Python.
- Une petite présentation de projet facile à déployer en tant que service API.

## Premières choses à personnaliser

1. Remplacez les exemples de ressources par vos modèles de domaine.
2. Ajoutez une authentification si le API n'est pas public.
3. Décidez quelles routes sont en lecture seule et lesquelles modifient les données.
4. Ajoutez des tests d'intégration pour les itinéraires qui définissent votre contrat.

## Quand c'est le bon pack

Choisissez cette option pour les API publics, les backends de service, les plates-formes internes API, les récepteurs webhook et les produits pour lesquels une interface sera construite séparément.

Pour une présentation pas à pas au niveau de la pile, lisez [FastAPI avec PostgreSQL et SQLAlchemy](/guides/python/fastapi-postgres-sqlalchemy/).
