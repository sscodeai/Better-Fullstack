Starterpakete sind eigensinnige Wege durch Better Fullstack. Verwenden Sie sie, wenn Sie das gewünschte Produktergebnis kennen, bevor Sie jede Bibliotheksauswahl kennen.

In jedem Paket wird erläutert, was erstellt werden muss, welcher Stapel verwendet wird, wie dieser generiert wird und was nach dem Gerüstbau zuerst geändert werden muss.

## Wann sollte man eine Packung verwenden?

Verwenden Sie ein Paket, wenn Sie schnell einen starken Standard und ein funktionierendes Projekt wünschen. Nach der Generierung können Sie mit Stack Builder weiterhin Framework-, Datenbank-, Authentifizierungs-, API-, Bereitstellungs- und Tooloptionen anpassen.

Wenn Sie den genauen Stack bereits kennen, verwenden Sie stattdessen [Stack Builder](/new?view=presets) oder eine Stack-spezifische Anleitung.

## Hinweise zur Paketauswahl

- Wählen Sie das SaaS-Paket, wenn Abrechnung, Authentifizierung und E-Mail Teil der ersten Produktversion sind.
- Wählen Sie das Agent-App-Paket AI, wenn Agentenanweisungen, MCP und App-Code gemeinsam weiterentwickelt werden müssen.
- Wählen Sie die Pakete REST API, Java API oder Rust Backend aus, wenn das Produkt als Servicegrenze startet.
- Wählen Sie das Mobile-App-Paket, wenn das Geräteerlebnis wichtiger ist als die Backend-Architektur.
- Wählen Sie das interne Toolpaket, wenn CRUD-Workflows, Dashboards und authentifizierte Vorgänge das Produkt sind.
