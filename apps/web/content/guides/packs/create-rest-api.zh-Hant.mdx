當你需要一個聚焦的 backend 服務時，請使用 REST API pack：routes、validation、persistence 和 linting，不帶 frontend。

<StarterPackCta id="rest-api" />

## 產生 REST API starter

開啟 [Stack Builder 中的 REST API pack](/new?preset=python-fastapi&view=command)，或執行：

```bash
bun create better-fullstack@latest my-rest-api \
  --ecosystem python \
  --python-web-framework fastapi \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff
```

## Pack 包含什麼

- FastAPI 用於 REST endpoints 和 OpenAPI docs。
- SQLAlchemy 用於資料庫 models 和 queries。
- Pydantic 用於 request 和 response validation。
- Ruff 用於 Python linting 和 formatting。
- 一個小型專案 layout，易於作為 API 服務部署。

## 首先要自訂的內容

1. 用你的 domain models 取代範例 resources。
2. 如果 API 不是公開的，請加入 authentication。
3. 決定哪些 routes 是 read-only，哪些會修改資料。
4. 為定義契約的 routes 加入 integration tests。

## 什麼時候適合這個 pack

選擇它來建構 public APIs、service backends、內部平台 APIs、webhook receivers，以及 frontend 會單獨建置的產品。

如需 stack 層面的 walkthrough，請閱讀 [FastAPI 與 PostgreSQL 和 SQLAlchemy](/guides/python/fastapi-postgres-sqlalchemy/)。
