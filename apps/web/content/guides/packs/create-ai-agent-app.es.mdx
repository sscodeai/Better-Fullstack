Usa el pack AI Agent App cuando quieras que la aplicación y el workflow de agentes empiecen juntos. Prepara un proyecto Next.js más documentación orientada a AI CLI y tooling local para agentes.

<StarterPackCta id="ai-agent-app" />

## Generar el starter de agente de IA

Abre el [pack AI Agent App en el Stack Builder](/new?preset=ai-cli-agent-workbench&view=command), o ejecuta:

```bash
bun create better-fullstack@latest my-ai-agent-app \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --ai ai-cli \
  --addons biome ruler turborepo mcp skills \
  --ai-docs claude-md agents-md cursorrules
```

## Qué incluye el pack

- Next.js para la superficie de producto.
- SQLite y Drizzle para persistencia local-first.
- Better Auth para cuentas y sessions.
- Setup de AI CLI, soporte MCP, skills y docs generadas para agentes.
- Biome y Ruler para que las instrucciones de agentes y la calidad de código vivan con el repo.

## Primeras cosas para personalizar

1. Define el trabajo visible para el usuario del agente: investigación, coding, soporte, automatización o análisis.
2. Añade el modelo de datos de app que el agente puede leer y escribir.
3. Endurece las instrucciones MCP/tool antes de conectar servicios externos.
4. Pasa de SQLite a una base de datos gestionada cuando colaboración o hosting de producción lo pidan.

## Cuándo este pack encaja

Elige esto para developer tools asistidas por IA, automatización interna, dashboards de agentes, workflow copilots o experimentos donde la estructura de app y las instrucciones de agentes deban evolucionar juntas.

Para el walkthrough de stack, lee [Workbench de agentes CLI de IA con Next.js](/guides/ai/nextjs-ai-cli-agent-workbench/).
