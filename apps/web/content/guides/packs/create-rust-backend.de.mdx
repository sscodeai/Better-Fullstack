Verwenden Sie das Rust-Backend-Paket, wenn Sie einen kompakten API-Dienst mit starken Typen, vorhersehbarer Leistung und einer vom Compiler erzwungenen Struktur wünschen.

<StarterPackCta id="rust-backend" />

## Generieren Sie den Rust-Backend-Starter

Öffnen Sie das [Rust-Backend-Paket im Stack Builder](/new?preset=rust-api&view=command) oder führen Sie Folgendes aus:

```bash
bun create better-fullstack@latest my-rust-backend \
  --ecosystem rust \
  --rust-web-framework axum \
  --rust-frontend none \
  --rust-orm sea-orm \
  --rust-logging tracing \
  --rust-error-handling anyhow-thiserror
```

## Was im Paket enthalten ist

- Axum für HTTP-Routen.
- SeaORM für typisierten Datenbankzugriff.
- PostgreSQL-orientierte Backend-Form.
- Nachverfolgung für strukturierte Anwendungsprotokolle.
- Wie auch immer und diese Fehlermuster für die praktische Fehlerbehandlung.

## Erste Dinge, die angepasst werden müssen

1. Definieren Sie Ihre Routenmodule rund um Ressourcen oder Arbeitsabläufe.
2. Fügen Sie Datenbankentitäten und Migrationen für die reale Domäne hinzu.
3. Halten Sie Handler-Fehler explizit, damit API-Fehler leicht zu debuggen sind.
4. Führen Sie `cargo check` und `cargo clippy -- -D warnings` aus, wenn das Projekt wächst.

## Wenn das das richtige Paket ist

Wählen Sie dies für hochzuverlässige Backend-Dienste, APIs mit strengen Leistungserwartungen, Systemtools mit HTTP-Oberflächen und Teams, die die Kompilierzeitgarantien von Rust bevorzugen.

Für eine exemplarische Vorgehensweise auf Stack-Ebene lesen Sie [Axum mit PostgreSQL und SeaORM](/guides/rust/axum-postgres-seaorm/).
