Usa el pack REST API cuando necesites un servicio backend enfocado: rutas, validación, persistencia y linting sin frontend.

<StarterPackCta id="rest-api" />

## Generar el starter REST API

Abre el [pack REST API en el Stack Builder](/new?preset=python-fastapi&view=command), o ejecuta:

```bash
bun create better-fullstack@latest my-rest-api \
  --ecosystem python \
  --python-web-framework fastapi \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff
```

## Qué incluye el pack

- FastAPI para endpoints REST y docs OpenAPI.
- SQLAlchemy para modelos de base de datos y queries.
- Pydantic para validación de requests y responses.
- Ruff para linting y formatting de Python.
- Un layout pequeño de proyecto que es fácil de desplegar como servicio API.

## Primeras cosas para personalizar

1. Reemplaza los recursos de ejemplo por tus modelos de dominio.
2. Añade autenticación si la API no es pública.
3. Decide qué rutas son read-only y cuáles mutan datos.
4. Añade tests de integración para las rutas que definen tu contrato.

## Cuándo este pack encaja

Elige esto para APIs públicas, service backends, APIs internas de plataforma, webhook receivers y productos donde el frontend se construirá por separado.

Para un walkthrough de stack, lee [FastAPI con PostgreSQL y SQLAlchemy](/guides/python/fastapi-postgres-sqlalchemy/).
