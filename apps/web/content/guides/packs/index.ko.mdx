스타터 팩은 Better Fullstack을 통한 독선적인 경로입니다. 모든 라이브러리 선택을 알기 전에 원하는 제품 결과를 알 때 이를 사용하십시오.

각 팩은 무엇을 생성할지, 어떤 스택을 사용할지, 생성 방법, 스캐폴딩 후 먼저 무엇을 변경할지 설명합니다.

## 팩을 선택하세요

## 팩을 사용하는 경우

강력한 기본값과 작업 프로젝트를 빠르게 원할 때 팩을 사용하세요. 생성 후에도 Stack Builder을 사용하면 프레임워크, 데이터베이스, 인증, API, 배포 및 도구 선택을 조정할 수 있습니다.

이미 정확한 스택을 알고 있다면 [Stack Builder](/new?view=presets) 또는 스택별 가이드를 대신 사용하세요.

## 팩 선택 참고사항

- 청구, 인증 및 이메일이 첫 번째 제품 버전의 일부인 경우 SaaS 팩을 선택하십시오.
- 에이전트 지침, MCP 및 앱 코드가 함께 발전해야 하는 경우 AI 에이전트 앱 팩을 선택하세요.
- 제품이 서비스 경계로 시작되면 REST API, Java API 또는 Rust 백엔드 팩을 선택하세요.
- 백엔드 아키텍처보다 장치 경험이 중요한 경우 모바일 앱 팩을 선택하세요.
- CRUD 워크플로, 대시보드 및 인증된 작업이 제품인 경우 내부 도구 팩을 선택하세요.
