強力な型、予測可能なパフォーマンス、コンパイラ強制構造を備えたコンパクトな API サービスが必要な場合は、Rust バックエンド パックを使用します。

<StarterPackCta id="rust-backend" />

## Rustバックエンドスターターを生成する

[Stack Builder の Rust バックエンド パック](/new?preset=rust-api&view=command) を開くか、次のコマンドを実行します。

```bash
bun create better-fullstack@latest my-rust-backend \
  --ecosystem rust \
  --rust-web-framework axum \
  --rust-frontend none \
  --rust-orm sea-orm \
  --rust-logging tracing \
  --rust-error-handling anyhow-thiserror
```

## パックの内容

- HTTP ルートの場合は Axum。
- SeaORM は型付きデータベース アクセス用です。
- PostgreSQL 指向のバックエンド形状。
- 構造化されたアプリケーション ログのトレース。
- とにかく、実際のエラー処理のためのこのエラー パターン。

## 最初にカスタマイズすること

1. リソースまたはワークフローに基づいてルート モジュールを定義します。
2. データベース エンティティと実際のドメインの移行を追加します。
3. API エラーをデバッグしやすいように、ハンドラー エラーを明示的にしておきます。
4. プロジェクトの成長に合わせて `cargo check` および `cargo clippy -- -D warnings` を実行します。

## これが適切なパックの場合

信頼性の高いバックエンド サービス、厳密なパフォーマンスが期待される API、HTTP サーフェスを備えたシステム ツール、および Rust のコンパイル時間保証を好むチームには、これを選択してください。

スタックレベルのチュートリアルについては、[Axum と PostgreSQL および SeaORM](/guides/rust/axum-postgres-seaorm/) を参照してください。
