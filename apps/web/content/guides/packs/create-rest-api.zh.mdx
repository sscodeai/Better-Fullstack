当你需要一个聚焦的 backend 服务时，请使用 REST API pack：routes、validation、persistence 和 linting，不带 frontend。

<StarterPackCta id="rest-api" />

## 生成 REST API starter

打开 [Stack Builder 中的 REST API pack](/new?preset=python-fastapi&view=command)，或运行：

```bash
bun create better-fullstack@latest my-rest-api \
  --ecosystem python \
  --python-web-framework fastapi \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff
```

## Pack 包含什么

- FastAPI 用于 REST endpoints 和 OpenAPI docs。
- SQLAlchemy 用于数据库 models 和 queries。
- Pydantic 用于 request 和 response validation。
- Ruff 用于 Python linting 和 formatting。
- 一个小型项目 layout，易于作为 API 服务部署。

## 首先要自定义的内容

1. 用你的 domain models 替换示例 resources。
2. 如果 API 不是公开的，请加入 authentication。
3. 决定哪些 routes 是 read-only，哪些会修改数据。
4. 为定义契约的 routes 加入 integration tests。

## 什么时候适合这个 pack

选择它来构建 public APIs、service backends、内部平台 APIs、webhook receivers，以及 frontend 会单独构建的产品。

如需 stack 层面的 walkthrough，请阅读 [FastAPI 与 PostgreSQL 和 SQLAlchemy](/guides/python/fastapi-postgres-sqlalchemy/)。
