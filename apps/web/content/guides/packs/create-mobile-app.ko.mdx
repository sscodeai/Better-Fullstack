첫 번째 화면이 웹이 아닌 기본 화면이어야 하는 경우 모바일 앱 팩을 사용하세요. Expo 및 Uniwind로 시작하므로 백엔드 서비스를 추가하기 전에 모바일 인터페이스를 구축할 수 있습니다.

<StarterPackCta id="mobile-app" />

## 모바일 스타터 생성

[Stack Builder의 모바일 앱 팩](/new?preset=uniwind&view=command)을 열거나 다음을 실행합니다.

```bash
bun create better-fullstack@latest my-mobile-app \
  --ecosystem typescript \
  --frontend native-uniwind \
  --backend none \
  --database none \
  --api none \
  --css-framework none \
  --ui-library none
```

## 팩에 포함된 내용

- 네이티브 앱 런타임의 경우 Expo입니다.
- React Native을 UI 기반으로 사용합니다.
- 유틸리티 우선 네이티브 스타일의 경우 Uniwind입니다.
- 백엔드 선택이 필요할 때까지 가볍게 유지되는 프런트엔드 전용 프로젝트 형태입니다.

## 가장 먼저 사용자 정의해야 할 사항

1. 샘플 화면을 핵심 모바일 흐름으로 교체합니다.
2. 인증이 기본 전용인지, API 지원인지, 아니면 웹 앱과 공유되는지 결정하세요.
3. 앱의 데이터 경계가 명확한 후에만 백엔드 서비스를 추가하세요.
4. 소형 및 대형 기기 크기에 대한 레이아웃을 조기에 검증하세요.

## 이것이 올바른 팩일 때

모바일 우선 제품, 컴패니언 앱, 네이티브 느낌이 필요한 프로토타입, 서버 인프라를 선택하기 전에 장치 경험부터 시작하려는 팀에 대해 이 옵션을 선택하세요.

스택 수준 연습을 보려면 [Expo Uniwind 네이티브 앱](/guides/react-native/expo-uniwind-native/)을 읽어보세요.
