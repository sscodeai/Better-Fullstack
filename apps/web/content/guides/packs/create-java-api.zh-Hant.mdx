當你想要一個從一開始就包含安全性、persistence、migrations 和 tests 的 Spring 服務時，請使用 Java API pack。

<StarterPackCta id="java-api" />

## 產生 Java API starter

開啟 [Stack Builder 中的 Java API pack](/new?preset=java-secure&view=command)，或執行：

```bash
bun create better-fullstack@latest my-java-api \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## Pack 包含什麼

- Spring Boot 用於 API application。
- Spring Security 用於受保護 routes。
- Spring Data JPA 用於 persistence。
- Flyway 用於資料庫 migrations。
- Gradle 用於 builds。
- JUnit 5 和 Testcontainers 用於 tests。

## 首先要自訂的內容

1. 用你的 aggregate roots 和 repositories 替換範例 entities。
2. 圍繞真實 authentication model 配置 Spring Security。
3. 在依賴產生的 schema 行為之前，加入 Flyway migrations。
4. 讓 Testcontainers 靠近 API contract tests。

## 什麼時候適合這個 pack

選擇它來建構 enterprise APIs、團隊維護的 backend services、安全 admin systems，以及 Java/Spring 生態是預期生產基礎的專案。

如需 stack 層面的 walkthrough，請閱讀 [安全的 Spring Boot API](/guides/java/spring-security-api/)。
