Starter packs 是 Better Fullstack 中帶有明確取向的路徑。當你先知道想要的產品結果，但還沒有決定每個 library 選擇時，可以使用它們。

每個 pack 都會說明要創建什麼、它使用哪個 stack、如何生成，以及 scaffold 之後最先應該改什麼。

## 什麼時候使用 pack

當你想要一個強預設值，並且快速得到可運行專案時，請使用 pack。生成之後，Stack Builder 仍然可以調整 framework、資料庫、auth、API、部署和 tooling 選擇。

如果你已經知道確切 stack，請使用 [Stack Builder](/new?view=presets) 或某篇具體 stack 指南。

## Pack 選擇說明

- 當 billing、auth 和 email 是產品第一版的一部分時，選擇 SaaS pack。
- 當代理指令、MCP 和應用程式碼需要一起演進時，選擇 AI Agent App pack。
- 當產品從服務邊界開始時，選擇 REST API、Java API 或 Rust Backend pack。
- 當裝置體驗比 backend 架構更早成為重點時，選擇 Mobile App pack。
- 當 CRUD workflows、dashboards 和認證操作就是產品時，選擇 Internal Tool pack。
