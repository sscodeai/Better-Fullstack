当你需要一个面向 CRUD 的可运行应用，用于 operations、admin workflows、support teams 或 product dashboards 时，请使用 Internal Tool pack。

<StarterPackCta id="internal-tool" />

## 生成内部工具 starter

打开 [Stack Builder 中的 Internal Tool pack](/new?preset=tanstack-hono&view=command)，或运行：

```bash
bun create better-fullstack@latest my-internal-tool \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime bun \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui
```

## Pack 包含什么

- TanStack Router 用于 web app。
- Hono on Bun 用于 backend。
- PostgreSQL 和 Drizzle 用于类型化 persistence。
- Better Auth 用于受保护的内部访问。
- tRPC 用于类型化 frontend-to-backend calls。
- Tailwind CSS 和 shadcn/ui 用于 dashboard UI。

## 首先要自定义的内容

1. 建模团队每天编辑的 operational objects。
2. 在暴露 destructive actions 之前加入 role checks。
3. 围绕同一个 typed API 构建 list、detail、create、edit 和 audit views。
4. 在邀请 teammates 前加入 deployment secrets 和 database migrations。

## 什么时候适合这个 pack

选择它来构建 admin panels、internal dashboards、support tools、back-office workflows，以及速度很重要但 type safety 也同样重要的 CRUD apps。

如需 stack 层面的 walkthrough，请阅读 [Hono 与 tRPC 和 Drizzle](/guides/typescript/hono-trpc-drizzle/)。
