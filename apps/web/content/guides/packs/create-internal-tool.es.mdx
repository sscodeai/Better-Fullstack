Usa el pack Internal Tool cuando necesites una app funcional orientada a CRUD para operaciones, workflows admin, equipos de soporte o dashboards de producto.

<StarterPackCta id="internal-tool" />

## Generar el starter de herramienta interna

Abre el [pack Internal Tool en el Stack Builder](/new?preset=tanstack-hono&view=command), o ejecuta:

```bash
bun create better-fullstack@latest my-internal-tool \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime bun \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui
```

## Qué incluye el pack

- TanStack Router para la web app.
- Hono en Bun para el backend.
- PostgreSQL y Drizzle para persistencia tipada.
- Better Auth para acceso interno protegido.
- tRPC para llamadas frontend-to-backend tipadas.
- Tailwind CSS y shadcn/ui para UI de dashboard.

## Primeras cosas para personalizar

1. Modela los objetos operativos que tu equipo edita cada día.
2. Añade controles de rol antes de exponer acciones destructivas.
3. Construye vistas de lista, detalle, creación, edición y auditoría alrededor de la misma API tipada.
4. Añade secretos de despliegue y migraciones de base de datos antes de invitar teammates.

## Cuándo este pack encaja

Elige esto para admin panels, dashboards internos, herramientas de soporte, back-office workflows y apps CRUD donde la velocidad importa pero la type safety también.

Para un walkthrough de stack, lee [Hono con tRPC y Drizzle](/guides/typescript/hono-trpc-drizzle/).
