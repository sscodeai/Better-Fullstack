Usa el pack Java API cuando quieras un servicio Spring con seguridad, persistencia, migraciones y tests incluidos desde el inicio.

<StarterPackCta id="java-api" />

## Generar el starter de API Java

Abre el [pack Java API en el Stack Builder](/new?preset=java-secure&view=command), o ejecuta:

```bash
bun create better-fullstack@latest my-java-api \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## Qué incluye el pack

- Spring Boot para la aplicación API.
- Spring Security para rutas protegidas.
- Spring Data JPA para persistencia.
- Flyway para migraciones de base de datos.
- Gradle para builds.
- JUnit 5 y Testcontainers para tests.

## Primeras cosas para personalizar

1. Reemplaza las entidades de ejemplo por tus aggregate roots y repositories.
2. Configura Spring Security alrededor de tu modelo real de autenticación.
3. Añade migraciones Flyway antes de depender del comportamiento de schema generado.
4. Mantén Testcontainers cerca de los tests de contrato de la API.

## Cuándo este pack encaja

Elige esto para APIs enterprise, servicios backend mantenidos por equipos, sistemas admin seguros y proyectos donde el ecosistema Java/Spring sea la base esperada de producción.

Para el walkthrough de stack, lee [API Spring Boot segura](/guides/java/spring-security-api/).
