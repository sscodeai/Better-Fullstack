강력한 유형, 예측 가능한 성능 및 컴파일러 적용 구조를 갖춘 컴팩트한 API 서비스를 원할 때 Rust 백엔드 팩을 사용하세요.

<StarterPackCta id="rust-backend" />

## Rust 백엔드 스타터 생성

[Stack Builder의 Rust 백엔드 팩](/new?preset=rust-api&view=command)을 열거나 다음을 실행합니다.

```bash
bun create better-fullstack@latest my-rust-backend \
  --ecosystem rust \
  --rust-web-framework axum \
  --rust-frontend none \
  --rust-orm sea-orm \
  --rust-logging tracing \
  --rust-error-handling anyhow-thiserror
```

## 팩에 포함된 내용

- HTTP 경로의 경우 Axum입니다.
- 형식화된 데이터베이스 액세스의 경우 SeaORM입니다.
- PostgreSQL 지향 백엔드 형태.
- 구조화된 애플리케이션 로그를 추적합니다.
- 어쨌든 실제 오류 처리를 위한 오류 패턴입니다.

## 가장 먼저 사용자 정의해야 할 사항

1. 리소스 또는 워크플로를 중심으로 경로 모듈을 정의하세요.
2. 실제 도메인에 대한 데이터베이스 엔터티 및 마이그레이션을 추가합니다.
3. API 실패를 쉽게 디버깅할 수 있도록 핸들러 오류를 명시적으로 유지하세요.
4. 프로젝트가 성장함에 따라 `cargo check` 및 `cargo clippy -- -D warnings`을 실행하세요.

## 이것이 올바른 팩일 때

신뢰도가 높은 백엔드 서비스, 성능 기대치가 엄격한 API, HTTP 표면이 있는 시스템 도구, Rust의 컴파일 시간 보장을 선호하는 팀의 경우 이 옵션을 선택하세요.

스택 수준 연습을 보려면 [Axum with PostgreSQL 및 SeaORM](/guides/rust/axum-postgres-seaorm/)를 읽어보세요.
