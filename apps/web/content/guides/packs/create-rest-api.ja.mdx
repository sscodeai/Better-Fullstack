REST API パックは、フロントエンドを使用しないルート、検証、永続性、lint などの重点的なバックエンド サービスが必要な場合に使用します。

<StarterPackCta id="rest-api" />

## REST API スターターを生成する

[Stack Builder](/new?preset=python-fastapi&view=command) の REST API パックを開くか、次のコマンドを実行します。

```bash
bun create better-fullstack@latest my-rest-api \
  --ecosystem python \
  --python-web-framework fastapi \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-quality ruff
```

## パックの内容

- REST エンドポイントの FastAPI と OpenAPI ドキュメント。
- データベース モデルとクエリの場合は SQLAlchemy。
- リクエストとレスポンスの検証用の Pydantic。
- Python のリンティングとフォーマット用の Ruff。
- API サービスとして簡単にデプロイできる小規模なプロジェクト レイアウト。

## 最初にカスタマイズすること

1. サンプル リソースをドメイン モデルに置き換えます。
2. API がパブリックでない場合は、認証を追加します。
3. どのルートが読み取り専用で、どのルートがデータを変更するかを決定します。
4. コントラクトを定義するルートの統合テストを追加します。

## これが適切なパックの場合

これは、パブリック API、サービス バックエンド、内部プラットフォーム API、Webhook レシーバー、およびフロントエンドが個別に構築される製品の場合に選択します。

スタックレベルのチュートリアルについては、[FastAPI と PostgreSQL および SQLAlchemy](/guides/python/fastapi-postgres-sqlalchemy/) を参照してください。
