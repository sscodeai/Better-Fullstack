Utilisez le pack Java API lorsque vous souhaitez un service Spring avec sécurité, persistance, migrations et tests inclus dès le départ.

<StarterPackCta id="java-api" />

## Générer le démarreur Java API

Ouvrez le pack [Java API dans Stack Builder](/new?preset=java-secure&view=command) ou exécutez :

```bash
bun create better-fullstack@latest my-java-api \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## Ce que comprend le pack

- Spring Boot pour l'application API.
- Spring Security pour les itinéraires protégés.
- Spring Data JPA pour la persistance.
- Flyway pour les migrations de bases de données.
- Gradle pour les builds.
- JUnit 5 et Testcontainers pour les tests.

## Premières choses à personnaliser

1. Remplacez les exemples d’entités par vos racines et référentiels agrégés.
2. Configurez Spring Security autour de votre véritable modèle d'authentification.
3. Ajoutez les migrations Flyway avant de vous fier au comportement du schéma généré.
4. Gardez les conteneurs de tests à proximité des tests contractuels API.

## Quand c'est le bon pack

Choisissez cette option pour les API d'entreprise, les services backend appartenant à l'équipe, les systèmes d'administration sécurisés et les projets dans lesquels l'écosystème Java/Spring constitue la base de production attendue.

Pour la procédure pas à pas au niveau de la pile, lisez [Secure Spring Boot API](/guides/java/spring-security-api/).
