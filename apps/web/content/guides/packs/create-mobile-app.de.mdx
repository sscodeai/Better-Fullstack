Verwenden Sie das Mobile-App-Paket, wenn der erste Bildschirm nativ und nicht webbasiert sein soll. Es beginnt mit Expo und Uniwind, sodass Sie eine mobile Schnittstelle erstellen können, bevor Sie Backend-Dienste hinzufügen.

<StarterPackCta id="mobile-app" />

## Erzeugen Sie den mobilen Starter

Öffnen Sie das [Mobile App-Paket im Stack Builder](/new?preset=uniwind&view=command) oder führen Sie Folgendes aus:

```bash
bun create better-fullstack@latest my-mobile-app \
  --ecosystem typescript \
  --frontend native-uniwind \
  --backend none \
  --database none \
  --api none \
  --css-framework none \
  --ui-library none
```

## Was im Paket enthalten ist

- Expo für die native App-Laufzeit.
- React Native als UI-Grundlage.
- Uniwind für natives Utility-First-Styling.
- Eine reine Frontend-Projektform, die einfach bleibt, bis Backend-Auswahlmöglichkeiten erforderlich sind.

## Erste Dinge, die angepasst werden müssen

1. Ersetzen Sie Beispielbildschirme durch den mobilen Kernfluss.
2. Entscheiden Sie, ob die Authentifizierung nur nativ, API-unterstützt oder mit einer Web-App geteilt werden soll.
3. Fügen Sie Back-End-Dienste erst hinzu, wenn die Datengrenzen der App klar sind.
4. Validieren Sie Layouts auf kleinen und großen Gerätegrößen frühzeitig.

## Wenn das das richtige Paket ist

Wählen Sie dies für Mobile-First-Produkte, Begleit-Apps, Prototypen, die ein natives Gefühl erfordern, und Teams, die mit dem Geräteerlebnis beginnen möchten, bevor sie sich für eine Serverinfrastruktur entscheiden.

Für die exemplarische Vorgehensweise auf Stapelebene lesen Sie [Expo Uniwind Native App](/guides/react-native/expo-uniwind-native/).
