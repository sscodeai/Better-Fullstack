Verwenden Sie das Java-Paket API, wenn Sie einen Spring-Dienst wünschen, der Sicherheit, Persistenz, Migrationen und Tests von Anfang an umfasst.

<StarterPackCta id="java-api" />

## Generieren Sie den Java-Starter API

Öffnen Sie das [Java API-Paket im Stack Builder](/new?preset=java-secure&view=command) oder führen Sie Folgendes aus:

```bash
bun create better-fullstack@latest my-java-api \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## Was im Paket enthalten ist

- Spring Boot für die Anwendung API.
- Spring Security für geschützte Routen.
- Spring Data JPA für Persistenz.
- Flyway für Datenbankmigrationen.
- Gradle für Builds.
- JUnit 5 und Testcontainer für Tests.

## Erste Dinge, die angepasst werden müssen

1. Ersetzen Sie Beispielentitäten durch Ihre aggregierten Roots und Repositorys.
2. Konfigurieren Sie Spring Security um Ihr echtes Authentifizierungsmodell herum.
3. Fügen Sie Flyway-Migrationen hinzu, bevor Sie sich auf das generierte Schemaverhalten verlassen.
4. Halten Sie Testcontainer in der Nähe der Vertragstests API.

## Wenn das das richtige Paket ist

Wählen Sie dies für Unternehmens-APIs, teameigene Backend-Dienste, sichere Verwaltungssysteme und Projekte, bei denen das Java/Spring-Ökosystem die erwartete Produktionsgrundlage ist.

Für die exemplarische Vorgehensweise auf Stack-Ebene lesen Sie [Secure Spring Boot API](/guides/java/spring-security-api/).
