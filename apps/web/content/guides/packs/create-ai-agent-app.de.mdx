Verwenden Sie das Agent-App-Paket AI, wenn die Anwendung und der Agent-Workflow gemeinsam gestartet werden sollen. Es bereitet ein Next.js-Projekt plus AI CLI-orientierte Dokumentation und lokale Agententools vor.

<StarterPackCta id="ai-agent-app" />

## Generieren Sie den Agentenstarter AI

Öffnen Sie das [AI Agent App Pack im Stack Builder](/new?preset=ai-cli-agent-workbench&view=command) oder führen Sie Folgendes aus:

```bash
bun create better-fullstack@latest my-ai-agent-app \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --ai ai-cli \
  --addons biome ruler turborepo mcp skills \
  --ai-docs claude-md agents-md cursorrules
```

## Was im Paket enthalten ist

- Next.js für die Produktoberfläche.
- SQLite und Drizzle für Local-First-Persistenz.
- Better Auth für Konten und Sitzungen.
- AI CLI Setup, MCP Support, Fähigkeiten und generierte Agentendokumente.
- Biome und Ruler, sodass Agentenanweisungen und Codequalität mit dem Repo leben.

## Erste Dinge, die angepasst werden müssen

1. Definieren Sie die Aufgabe des benutzerorientierten Agenten: Recherche, Codierung, Support, Automatisierung oder Analyse.
2. Fügen Sie das App-Datenmodell hinzu, das der Agent lesen und schreiben darf.
3. Ziehen Sie die MCP/tool-Anweisungen fest, bevor Sie externe Dienste anschließen.
4. Wechseln Sie von SQLite zu einer verwalteten Datenbank, wenn die Zusammenarbeit oder das Produktionshosting dies erfordert.

## Wenn das das richtige Paket ist

Wählen Sie dies für AI-unterstützte Entwicklertools, interne Automatisierung, Agenten-Dashboards, Workflow-Copiloten oder Experimente, bei denen App-Struktur und Agentenanweisungen gemeinsam weiterentwickelt werden sollen.

Die exemplarische Vorgehensweise auf Stapelebene finden Sie unter [Next.js AI CLI Agent Workbench](/guides/ai/nextjs-ai-cli-agent-workbench/).
