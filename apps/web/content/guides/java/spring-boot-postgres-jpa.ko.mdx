Spring 생태계 및 관계형 지속성을 갖춘 Java API을 원할 때 이 스택을 사용하십시오.

```npm
npm create better-fullstack@latest my-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool maven \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-testing-libraries junit5
```

## 이것이 만드는 것

- Spring Boot을 사용하는 Java 프로젝트.
- Maven을 빌드 도구로 사용합니다.
- PostgreSQL을 데이터베이스 옵션으로 사용합니다.
- 지속성을 위한 JPA.
- 테스트용 JUnit 5.

## 생성된 모양

스캐폴드는 래퍼 스크립트, 소스 세트, 상태 컨트롤러 및 JPA 지원 샘플 사용자 엔드포인트가 포함된 일반적인 Spring Boot 서비스입니다. 패키지 이름은 `com.example` 아래의 프로젝트 이름에서 파생됩니다.

```txt
my-spring-api/
├── mvnw
├── .mvn/
├── pom.xml
├── src/main/java/com/example/myspringapi/
│   ├── Application.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── UserController.java
│   ├── domain/
│   │   └── AppUser.java
│   ├── repository/
│   │   └── AppUserRepository.java
│   └── service/
│       └── AppUserService.java
├── src/main/resources/
│   └── application.yml
└── src/test/java/com/example/myspringapi/
    └── ApplicationTests.java
```

생성된 로컬 JPA 데이터 소스는 PostgreSQL 호환 모드에서 H2 파일 데이터베이스를 사용합니다. `--database postgres`을 대상 데이터베이스 선택으로 처리하고 실제 PostgreSQL 환경에 대한 데이터 소스 URL, 드라이버, 자격 증명 및 JDBC 종속성을 전환합니다.

## 그것을 선택하는 경우

대규모 에코시스템, 일반적인 엔터프라이즈 통합 및 예측 가능한 애플리케이션 구조를 갖춘 기존 Java 백엔드를 원할 경우 Spring Boot을 선택하세요.

처음부터 보안 및 마이그레이션 도구가 아닌 지속성을 갖춘 최소한의 Spring Boot API을 원할 때 이 가이드를 선택하세요. 모든 엔드포인트가 Spring Security 뒤에서 시작되어야 하고 Testcontainers와의 통합 테스트가 기준의 일부인 경우 보안 Spring 가이드를 선택하세요.

| 결정 포인트 | 최소 스프링 JPA | 보안 스프링 API |
| --- | --- | --- |
| 빌드 도구 | Maven | Gradle |
| 보안 | 선택되지 않음 | Spring Security 선택됨 |
| 데이터베이스 변경 | 최대 절전 모드 `ddl-auto` 로컬 루프 | Flyway 마이그레이션 경로 |
| 테스트 | JUnit 5 | JUnit 5 및 테스트 컨테이너 |
| 다음에 가장 적합 | 학습, 프로토타입, 내부 CRUD | 프로덕션 지향 API 기준선 |

## 대표 조각

생성된 도메인 모델은 표준 JPA 엔터티입니다. 지속형 집계에 속하는 경우 여기에 필드를 추가하세요.

```java
import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import java.time.Instant;

@Entity
public class AppUser {
  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;

  private String email;
  private String displayName;
  private Instant createdAt = Instant.now();

  protected AppUser() {
  }

  public AppUser(String email, String displayName) {
    this.email = email;
    this.displayName = displayName;
  }
}
```

Spring Data JPA는 메서드 이름에서 공통 쿼리를 파생하므로 리포지토리는 작게 유지됩니다.

```java
import java.util.Optional;
import org.springframework.data.jpa.repository.JpaRepository;

public interface AppUserRepository extends JpaRepository<AppUser, Long> {
  Optional<AppUser> findByEmail(String email);
}
```

컨트롤러는 서비스 계층을 통해 `GET /users` 및 `POST /users`을 노출합니다.

```java
@RestController
@RequestMapping("/users")
public class UserController {
  private final AppUserService userService;

  public UserController(AppUserService userService) {
    this.userService = userService;
  }

  @GetMapping
  public List<UserResponse> listUsers() {
    return userService.findAll().stream().map(UserResponse::from).toList();
  }

  @PostMapping
  @ResponseStatus(HttpStatus.CREATED)
  public UserResponse createUser(@RequestBody CreateUserRequest request) {
    AppUser createdUser = userService.create(request.email(), request.displayName());
    return UserResponse.from(createdUser);
  }
}
```

## 마이그레이션 및 테스트

이 최소 스택은 Flyway 또는 Liquibase를 선택하지 않습니다. Hibernate는 초기 개발 중에 로컬 H2 스키마 이동을 유지할 수 있지만 여러 환경이 데이터베이스 형태에 의존하기 전에 마이그레이션 라이브러리를 추가할 수 있습니다.

```bash
./mvnw test
./mvnw spring-boot:run
./mvnw package
```

프로덕션 PostgreSQL을 추가할 때 로컬 기본값에서 환경 기반 데이터 소스 설정으로 이동합니다.

```yaml
spring:
  datasource:
    url: ${DATABASE_URL}
    username: ${DATABASE_USERNAME}
    password: ${DATABASE_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: validate
```

## 호환성 참고사항

- `--java-orm spring-data-jpa`에는 Spring Boot 및 Java 빌드 도구가 필요합니다.
- Flyway 및 Liquibase는 Better Fullstack의 Spring 데이터 JPA 경로에 연결되어 있습니다.
- 이 가이드에서는 `--java-build-tool maven`을 사용합니다. Gradle을 사용하여 의도적으로 재생성하지 않는 한 Maven 래퍼 명령을 사용하세요.
- 이 스택이 PostgreSQL을 대상으로 하더라도 로컬 생성 데이터 소스 및 종속성은 H2입니다. 관리되는 Postgres 데이터베이스에 배포하기 전에 PostgreSQL JDBC 드라이버를 추가하고 `spring.datasource.*`을 재정의하세요.

## 배포 참고사항

Maven을 사용하여 jar를 빌드하고 Java 21 이상을 제공하고 환경별 데이터 소스 설정으로 애플리케이션을 실행합니다.

```bash
./mvnw package
java -jar target/*.jar
```

프로덕션의 경우 마이그레이션을 추가하고, PostgreSQL JDBC 드라이버를 추가하고, Spring Boot 데이터 소스 속성을 통해 연결 풀링을 구성하고, `ddl-auto: update`에 의존하지 마세요. 플랫폼 상태 확인을 통해 Expose `/health`.

## 문제 해결

| 징후 | 확인하다 |
| --- | --- |
| 앱이 Postgres 대신 H2로 시작됩니다. | 환경별 구성에서 `spring.datasource.*`을 재정의합니다. |
| `POST /users`은 불완전한 입력을 허용합니다. | 향후 스캐폴드에 `--java-libraries spring-validation`을 추가하거나 수동으로 Bean 유효성 검사를 추가하세요. |
| JPA 엔터티가 표시되지 않습니다. | 구성 요소 검색을 통해 엔터티를 찾을 수 있도록 생성된 애플리케이션 패키지 아래에 엔터티를 유지합니다. |
| Maven 명령이 실패했습니다. | 다른 버전으로 전역적으로 설치된 Maven이 아닌 `./mvnw`을 사용하세요. |

## 트레이드오프

Spring Boot은 최소 API 프레임워크보다 무겁지만 확립된 규칙, 라이브러리 깊이 및 장기적인 백엔드 유지 관리 가능성이 필요할 때 효과가 있습니다.
