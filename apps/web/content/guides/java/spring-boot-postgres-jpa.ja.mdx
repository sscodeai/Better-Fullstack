Spring エコシステムとリレーショナル永続性を備えた Java API が必要な場合は、このスタックを使用します。

```npm
npm create better-fullstack@latest my-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool maven \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-testing-libraries junit5
```

## これが生み出すもの

- Spring Boot を使用する Java プロジェクト。
- Maven をビルド ツールとして使用します。
- PostgreSQL をデータベース オプションとして使用します。
- JPA 永続化用。
- テスト用の JUnit 5。

## 生成された形状

スキャフォールドは、ラッパー スクリプト、ソース セット、ヘルス コントローラー、および JPA をサポートするサンプル ユーザー エンドポイントを備えた従来の Spring Boot サービスです。パッケージ名は、`com.example` のプロジェクト名から派生します。

```txt
my-spring-api/
├── mvnw
├── .mvn/
├── pom.xml
├── src/main/java/com/example/myspringapi/
│   ├── Application.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── UserController.java
│   ├── domain/
│   │   └── AppUser.java
│   ├── repository/
│   │   └── AppUserRepository.java
│   └── service/
│       └── AppUserService.java
├── src/main/resources/
│   └── application.yml
└── src/test/java/com/example/myspringapi/
    └── ApplicationTests.java
```

生成されたローカル JPA データソースは、PostgreSQL 互換モードで H2 ファイル データベースを使用します。 `--database postgres` をターゲット データベースの選択として扱い、データソース URL、ドライバー、資格情報、および JDBC 依存関係を実際の PostgreSQL 環境に切り替えます。

## いつ選択するか

大規模なエコシステム、共通のエンタープライズ統合、予測可能なアプリケーション構造を備えた従来の Java バックエンドが必要な場合は、Spring Boot を選択してください。

永続性を備えた最小限の Spring Boot API が必要だが、初日からセキュリティと移行ツールは必要ない場合は、このガイドを選択してください。すべてのエンドポイントが Spring Security の背後で開始され、Testcontainers との統合テストがベースラインの一部である場合は、安全な Spring ガイドを選択してください。

| 決定点 | 最小スプリング JPA | セキュアスプリング API |
| --- | --- | --- |
| ビルドツール | Maven | Gradle |
| 安全 | 未選択 | Spring Security が選択されました |
| データベースの変更 | Hibernate `ddl-auto` ローカル ループ | Flyway 移行パス |
| テスト | JUnit 5 | JUnit 5 とテストコンテナ |
| こんな方に最適 | 学習、プロトタイプ、内部 CRUD | 実稼働指向の API ベースライン |

## 代表的なスニペット

生成されたドメイン モデルは、標準の JPA エンティティです。フィールドが永続化された集計に属している場合は、ここにフィールドを追加します。

```java
import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import java.time.Instant;

@Entity
public class AppUser {
  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;

  private String email;
  private String displayName;
  private Instant createdAt = Instant.now();

  protected AppUser() {
  }

  public AppUser(String email, String displayName) {
    this.email = email;
    this.displayName = displayName;
  }
}
```

Spring Data JPA はメソッド名から一般的なクエリを派生するため、リポジトリは小さいままです。

```java
import java.util.Optional;
import org.springframework.data.jpa.repository.JpaRepository;

public interface AppUserRepository extends JpaRepository<AppUser, Long> {
  Optional<AppUser> findByEmail(String email);
}
```

コントローラーは、サービス層を通じて `GET /users` と `POST /users` を公開します。

```java
@RestController
@RequestMapping("/users")
public class UserController {
  private final AppUserService userService;

  public UserController(AppUserService userService) {
    this.userService = userService;
  }

  @GetMapping
  public List<UserResponse> listUsers() {
    return userService.findAll().stream().map(UserResponse::from).toList();
  }

  @PostMapping
  @ResponseStatus(HttpStatus.CREATED)
  public UserResponse createUser(@RequestBody CreateUserRequest request) {
    AppUser createdUser = userService.create(request.email(), request.displayName());
    return UserResponse.from(createdUser);
  }
}
```

## 移行とテスト

この最小スタックでは、Flyway または Liquibase は選択されません。 Hibernate は、開発の初期段階でローカル H2 スキーマを移動し続けることができますが、複数の環境がデータベースの形状に依存する前に移行ライブラリを追加します。

```bash
./mvnw test
./mvnw spring-boot:run
./mvnw package
```

本番環境 PostgreSQL を追加するときは、ローカルのデフォルトから環境主導のデータソース設定に移行します。

```yaml
spring:
  datasource:
    url: ${DATABASE_URL}
    username: ${DATABASE_USERNAME}
    password: ${DATABASE_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: validate
```

## 互換性に関する注意事項

- `--java-orm spring-data-jpa` には Spring Boot と Java ビルド ツールが必要です。
- Flyway と Liquibase は、Better Fullstack の Spring Data JPA パスに関連付けられています。
- このガイドでは `--java-build-tool maven` を使用します。 Gradle で意図的に再生成しない限り、Maven ラッパー コマンドを使用してください。
- このスタックは PostgreSQL をターゲットとしていますが、ローカルで生成されたデータソースと依存関係は H2 です。マネージド Postgres データベースに対してデプロイする前に、PostgreSQL JDBC ドライバーを追加し、`spring.datasource.*` をオーバーライドします。

## 導入メモ

Maven を使用して jar をビルドし、Java 21 以降を提供し、環境固有のデータソース設定でアプリケーションを実行します。

```bash
./mvnw package
java -jar target/*.jar
```

運用環境の場合は、移行を追加し、PostgreSQL JDBC ドライバーを追加し、Spring Boot データソース プロパティを通じて接続プーリングを構成し、`ddl-auto: update` への依存を避けます。 Expose `/health` をプラットフォームのヘルスチェックで確認してください。

## トラブルシューティング

| 症状 | チェック |
| --- | --- |
| アプリは Postgres ではなく H2 で起動します | 環境固有の構成で `spring.datasource.*` をオーバーライドします。 |
| `POST /users` は不完全な入力を受け入れます | 将来のスキャフォールドに `--java-libraries spring-validation` を追加するか、Bean Validation を手動で追加します。 |
| JPA エンティティが表示されない | コンポーネント スキャンでエンティティが検出されるように、生成されたアプリケーション パッケージの下にエンティティを保持します。 |
| Maven コマンドが失敗する | 異なるバージョンのグローバルにインストールされた Maven ではなく、`./mvnw` を使用してください。 |

## トレードオフ

Spring Boot は最小限の API フレームワークよりも重いですが、確立された規則、ライブラリの深さ、および長期にわたるバックエンドの保守性が必要な場合には十分に効果があります。
