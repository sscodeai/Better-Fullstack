當你需要一個帶有 Spring 生態和關係型持久化的 Java API 時，使用這個 stack。

```npm
npm create better-fullstack@latest my-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool maven \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-testing-libraries junit5
```

## 創建內容

- 使用 Spring Boot 的 Java 專案。
- Maven 作為 build tool。
- PostgreSQL 作為資料庫選項。
- JPA 用於持久化。
- JUnit 5 用於測試。

## 生成結構

這個 scaffold 是一個傳統 Spring Boot 服務，包含 wrapper scripts、source sets、health controller，以及由 JPA 支撐的範例使用者 endpoints。 package name 會根據專案名稱產生在 `com.example` 下。

```txt
my-spring-api/
├── mvnw
├── .mvn/
├── pom.xml
├── src/main/java/com/example/myspringapi/
│   ├── Application.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── UserController.java
│   ├── domain/
│   │   └── AppUser.java
│   ├── repository/
│   │   └── AppUserRepository.java
│   └── service/
│       └── AppUserService.java
├── src/main/resources/
│   └── application.yml
└── src/test/java/com/example/myspringapi/
    └── ApplicationTests.java
```

產生的本機 JPA datasource 使用 H2 file database，並開啟 PostgreSQL compatibility mode。把 `--database postgres` 看作目標資料庫選擇；在真實 PostgreSQL 環境中，需要切換 datasource URL、driver、credentials 和 JDBC dependency。

## 何時選擇

當你想要一個傳統 Java backend，依賴大型 Spring 生態、常見 enterprise integrations 和可預測的應用結構時，選擇 Spring Boot。

如果你想要一個帶有持久化的最小 Spring Boot API，並且第一天不需要 security 和 migration tooling，選擇這篇指南。如果每個 endpoint 一開始就應該由 Spring Security 保護，而 Testcontainers integration testing 是基線的一部分，選擇安全版 Spring 指南。

| 決策點 | Minimal Spring JPA | Secure Spring API |
| --- | --- | --- |
| Build tool | Maven | Gradle |
| Security | 未選擇 | 已選擇 Spring Security |
| 資料庫變更 | Hibernate `ddl-auto` 本地循環 | Flyway migration path |
| Tests | JUnit 5 | JUnit 5 加 Testcontainers |
| 適合 | 學習、原型、內在 CRUD | 面向生產的 API baseline |

## 代表性片段

產生的 domain model 是標準 JPA entity。當字段屬於被持久化的 aggregate 時，在這裡添加。

```java
import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import java.time.Instant;

@Entity
public class AppUser {
  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;

  private String email;
  private String displayName;
  private Instant createdAt = Instant.now();

  protected AppUser() {
  }

  public AppUser(String email, String displayName) {
    this.email = email;
    this.displayName = displayName;
  }
}
```

Repositories 會保持很小，因為 Spring Data JPA 可以從方法名稱派生常見查詢。

```java
import java.util.Optional;
import org.springframework.data.jpa.repository.JpaRepository;

public interface AppUserRepository extends JpaRepository<AppUser, Long> {
  Optional<AppUser> findByEmail(String email);
}
```

Controller 透過 service layer 暴露 `GET /users` 和 `POST /users`。

```java
@RestController
@RequestMapping("/users")
public class UserController {
  private final AppUserService userService;

  public UserController(AppUserService userService) {
    this.userService = userService;
  }

  @GetMapping
  public List<UserResponse> listUsers() {
    return userService.findAll().stream().map(UserResponse::from).toList();
  }

  @PostMapping
  @ResponseStatus(HttpStatus.CREATED)
  public UserResponse createUser(@RequestBody CreateUserRequest request) {
    AppUser createdUser = userService.create(request.email(), request.displayName());
    return UserResponse.from(createdUser);
  }
}
```

## 遷移與測試

這個最小 stack 不會選擇 Flyway 或 Liquibase。在早期開發中，Hibernate 可以讓本機 H2 schema 持續變化；但在多個環境開始依賴資料庫形狀之前，請先加入 migration library。

```bash
./mvnw test
./mvnw spring-boot:run
./mvnw package
```

新增生產 PostgreSQL 時，從本機 defaults 遷移到由環境驅動的 datasource settings：

```yaml
spring:
  datasource:
    url: ${DATABASE_URL}
    username: ${DATABASE_USERNAME}
    password: ${DATABASE_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: validate
```

## 相容性說明

- `--java-orm spring-data-jpa` 需要 Spring Boot 和 Java build tool。
- Flyway 和 Liquibase 在 Better Fullstack 中綁定到 Spring Data JPA 路徑。
- 本指南使用 `--java-build-tool maven`；除非你有意用 Gradle 重新生成，否則使用 Maven wrapper commands。
- 儘管這個 stack 的目標是 PostgreSQL，產生的本地 datasource 和 dependency 仍然是 H2。在部署到 managed Postgres database 前，新增 PostgreSQL JDBC driver 並覆寫 `spring.datasource.*`。

## 部署說明

用 Maven 建構 jar，提供 Java 21 或更新版本，並以環境特定的 datasource settings 運行應用程式。

```bash
./mvnw package
java -jar target/*.jar
```

在生產環境中，新增 migrations，新增 PostgreSQL JDBC driver，透過 Spring Boot datasource properties 配置 connection pooling，並避免依賴 `ddl-auto: update`。透過平台 health check 暴露 `/health`。

## 故障排查

| 症狀 | 檢查 |
| --- | --- |
| App 啟動時使用 H2 而不是 Postgres | 在環境特定配置中覆寫 `spring.datasource.*`。 |
| `POST /users` 接受不完整 input | 在未來 scaffold 中加入 `--java-libraries spring-validation`，或手動新增 Bean Validation。 |
| JPA entities 沒有出現 | 把 entities 保持在產生的 application package 下，讓 component scanning 能找到它們。 |
| Maven commands 失敗 | 使用 `./mvnw`，不要使用版本不同的全域 Maven。 |

## 取捨
