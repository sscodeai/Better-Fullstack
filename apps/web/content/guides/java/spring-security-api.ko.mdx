지속성, 보안, 데이터베이스 마이그레이션 및 통합 테스트 도구를 갖춘 Spring Boot 백엔드를 원할 때 이 스택을 사용하십시오.

```npm
npm create better-fullstack@latest my-secure-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## 이것이 만드는 것

- Spring Boot 자바 API.
- Gradle을 빌드 도구로 사용합니다.
- 스프링 데이터 JPA.
- Spring Security.
- 스프링 액츄에이터 및 Flyway.
- JUnit 5 및 테스트 컨테이너.

## 생성된 모양

이 스택은 프로덕션 API 기준선에 더 가깝게 시작됩니다. 여기에는 Gradle 래퍼 파일, Spring Security 구성, JPA 샘플 엔드포인트, Flyway 마이그레이션 파일, 액추에이터 엔드포인트 및 Testcontainers 테스트 경로가 포함됩니다.

```txt
my-secure-spring-api/
├── gradlew
├── gradle/
├── build.gradle.kts
├── settings.gradle.kts
├── src/main/java/com/example/mysecurespringapi/
│   ├── Application.java
│   ├── config/
│   │   └── SecurityConfig.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── UserController.java
│   ├── domain/
│   │   └── AppUser.java
│   ├── repository/
│   │   └── AppUserRepository.java
│   └── service/
│       └── AppUserService.java
├── src/main/resources/
│   ├── application.yml
│   └── db/migration/V1__init.sql
└── src/test/java/com/example/mysecurespringapi/
    ├── ApplicationTests.java
    └── ApplicationContainerTests.java
```

생성된 보안 경로는 메모리 내 사용자와 함께 HTTP 기본을 사용합니다. 이는 완전한 제품 인증 시스템이 아닌 로컬 및 내부 API 작업에 대한 실질적인 보안 기본값입니다.

## 호환성 참고사항

Spring Security, JPA 및 Spring 라이브러리에는 Java 빌드 도구와 함께 Spring Boot이 필요합니다. Flyway은 Better Fullstack의 Spring 데이터 JPA 경로와 쌍을 이룹니다.

추가 호환성 세부정보:

- `--java-auth spring-security`은(는) Maven 또는 Gradle이 있는 Spring Boot 프로젝트에 대해서만 내보내집니다.
- `--java-libraries flyway`은 `--java-orm spring-data-jpa`을 선택한 경우에만 유지됩니다.
- 더 넓은 스택에서 Flyway과 Liquibase가 모두 선택된 경우 생성기는 Flyway을 유지하고 이 경로에 대해 Liquibase를 삭제합니다.
- `--java-testing-libraries testcontainers`은 컨테이너 기반 통합 테스트 지원을 추가하므로 해당 테스트 레인에 대한 로컬 Docker 가용성이 중요합니다.

## 그것을 선택하는 경우

보안, 마이그레이션 및 통합 테스트가 초기에 제공되어야 하는 프로덕션 지향 Java API에 대해 이를 선택하십시오.

가장 작은 JPA 예제를 원하고 나중에 보안을 추가할 계획이라면 최소한의 Spring 가이드를 선택하세요. 첫 번째 커밋에서 이미 "엔드포인트는 어떻게 보호되고, 스키마는 어떻게 변경되며, 실제 데이터베이스에 대해 어떻게 테스트합니까?"라는 대답을 해야 하는 경우 이 가이드를 선택하세요.

| 결정 포인트 | 보안 스프링 API | 최소 스프링 JPA |
| --- | --- | --- |
| 보안 | Spring Security을 통한 HTTP 기본 | 선택되지 않음 |
| 마이그레이션 | Flyway 처음부터 | 나중에 추가 |
| 테스트 | JUnit 5 및 테스트 컨테이너 | JUnit 5 |
| 런타임 가시성 | 액추에이터 상태/정보/측정항목 | 기본 상태 엔드포인트 |
| 비용 | 더 많은 생성된 파일 및 구성 | 더 작은 출발점 |

## 대표 조각

생성된 보안 구성은 `/health`을 허용하고 HTTP Basic을 사용하여 나머지 애플리케이션을 보호합니다.

```java
@Configuration
public class SecurityConfig {
  @Bean
  SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
      .csrf((csrf) -> csrf.disable())
      .authorizeHttpRequests((authorize) -> authorize
        .requestMatchers("/health").permitAll()
        .anyRequest().authenticated()
      )
      .httpBasic(Customizer.withDefaults());

    return http.build();
  }
}
```

자격 증명은 로컬 기본값을 사용하여 환경 지원 속성에서 읽습니다.

```java
@Bean
UserDetailsService userDetailsService(
  @Value("${APP_BASIC_USERNAME:admin}") String username,
  @Value("${APP_BASIC_PASSWORD:change-me}") String password,
  PasswordEncoder passwordEncoder
) {
  UserDetails user = User.withUsername(username)
    .password(passwordEncoder.encode(password))
    .roles("USER")
    .build();

  return new InMemoryUserDetailsManager(user);
}
```

개발하는 동안 기본 인증으로 보호된 엔드포인트를 사용하세요.

```bash
curl -u "$APP_BASIC_USERNAME:$APP_BASIC_PASSWORD" \
  http://localhost:8080/users
```

Flyway은 선택 시 스키마 생성을 소유합니다.

```sql
CREATE TABLE IF NOT EXISTS app_users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email VARCHAR(255) NOT NULL UNIQUE,
  display_name VARCHAR(255) NOT NULL,
  created_at TIMESTAMP NOT NULL
);
```

## 마이그레이션 및 테스트

Flyway은 Hibernate 상태를 변경합니다. 생성된 구성은 Hibernate가 자유롭게 업데이트하도록 허용하는 대신 스키마를 검증합니다. 데이터베이스 변경마다 새 `V2__...sql` 파일을 추가하고 마이그레이션을 계속 검토할 수 있도록 하세요.

```bash
./gradlew test
./gradlew bootRun
./gradlew build
```

Docker를 사용할 수 있으면 Testcontainers 레인을 실행하세요. 로컬 환경에서 Docker를 사용할 수 없는 경우 빠른 경로를 계속 사용할 수 있도록 단위 테스트를 컨테이너 테스트와 별도로 유지하세요.

```java
@SpringBootTest
class ApplicationContainerTests {
  @Test
  void contextLoadsAgainstContainerDatabase() {
  }
}
```

## 배포 참고사항

배포하기 전에 로컬 자격 증명을 교체하고, 생성된 빌드에 여전히 H2만 포함된 경우 PostgreSQL JDBC 드라이버를 추가하고, 관리형 PostgreSQL 데이터 소스를 구성하고, 시작 또는 릴리스 조정의 일부로 Flyway 마이그레이션을 실행하세요.

```bash
export APP_BASIC_USERNAME="service-admin"
export APP_BASIC_PASSWORD="replace-with-secret"
export SPRING_DATASOURCE_URL="jdbc:postgresql://db.example.com:5432/my_secure_spring_api"
export SPRING_DATASOURCE_DRIVER_CLASS_NAME="org.postgresql.Driver"
export SPRING_DATASOURCE_USERNAME="app"
export SPRING_DATASOURCE_PASSWORD="replace-with-secret"
./gradlew build
java -jar build/libs/*.jar
```

액추에이터는 `--java-libraries spring-actuator`을 통해 선택되므로 플랫폼 프로브에 상태 및 메트릭 엔드포인트를 사용할 수 있습니다. 공용 인터넷에 서비스를 배치하기 전에 엔드포인트 노출을 검토하십시오.

## 문제 해결

| 징후 | 확인하다 |
| --- | --- |
| 모든 끝점은 401을 반환합니다. | `/health`을 제외한 모든 항목에 대해 기본 인증 자격 증명을 보냅니다. |
| Flyway 시작 시 실패 | 데이터베이스에 연결할 수 있고 마이그레이션 기록이 스키마와 일치하는지 확인하세요. |
| Testcontainers 테스트가 로컬에서 실패함 | Docker가 실행 중이고 현재 사용자가 컨테이너를 시작할 수 있는지 확인하세요. |
| 앱은 기본 `admin/change-me`을 사용합니다. | 런타임 환경에서 `APP_BASIC_USERNAME` 및 `APP_BASIC_PASSWORD`을 설정합니다. |
| 액추에이터 끝점이 누락되었습니다. | `spring-actuator`이(가) `--java-libraries`에 아직 존재하는지 확인하세요. |
