當你需要一個具有持久化、安全性、資料庫遷移和 integration-test tooling 的 Spring Boot backend 時，使用這個 stack。

```npm
npm create better-fullstack@latest my-secure-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## 創建內容

- Spring Boot Java API。
- Gradle 作為 build tool。
- Spring Data JPA。
- Spring Security。
- Spring Actuator 和 Flyway。
- JUnit 5 和 Testcontainers。

## 生成結構

這個 stack 的起點更接近生產導向的 API baseline。它包含 Gradle wrapper files、Spring Security 配置、JPA 範例 endpoints、Flyway migration files、Actuator endpoints，以及 Testcontainers test path。

```txt
my-secure-spring-api/
├── gradlew
├── gradle/
├── build.gradle.kts
├── settings.gradle.kts
├── src/main/java/com/example/mysecurespringapi/
│   ├── Application.java
│   ├── config/
│   │   └── SecurityConfig.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── UserController.java
│   ├── domain/
│   │   └── AppUser.java
│   ├── repository/
│   │   └── AppUserRepository.java
│   └── service/
│       └── AppUserService.java
├── src/main/resources/
│   ├── application.yml
│   └── db/migration/V1__init.sql
└── src/test/java/com/example/mysecurespringapi/
    ├── ApplicationTests.java
    └── ApplicationContainerTests.java
```

產生的 security path 使用 HTTP Basic 和一個 in-memory user。它適合作為本地和內部 API 工作的安全預設值，但不是完整的產品級認證系統。

## 相容性說明

Spring Security、JPA 和 Spring libraries 需要 Spring Boot 搭配 Java build tool。 Flyway 在 Better Fullstack 中與 Spring Data JPA 路徑配對。

額外相容性細節：

- `--java-auth spring-security` 只會為使用 Maven 或 Gradle 的 Spring Boot projects 輸出。
- `--java-libraries flyway` 只有在選擇 `--java-orm spring-data-jpa` 時保留。
- 如果在更大的 stack 中同時選擇 Flyway 和 Liquibase，generator 會在這個路徑中保留 Flyway 並丟棄 Liquibase。
- `--java-testing-libraries testcontainers` 會加入基於 container 的 integration test support，因此本地 Docker 可用性會影響這個 test lane。

## 何時選擇

適合用於生產導向的 Java APIs，其中 security、migrations 和 integration tests 應該儘早存在。

當你想要最小 JPA 範例，並計畫之後再新增 security 時，選擇最小 Spring 指南。當第一個 commit 就需要回答「endpoints 如何保護、schemas 如何變化、如何針對真實資料庫測試」時，選擇這篇指南。

| 決策點 | Secure Spring API | Minimal Spring JPA |
| --- | --- | --- |
| Security | Spring Security 提供 HTTP Basic | 未選擇 |
| Migrations | 從一開始使用 Flyway | 之後添加 |
| Tests | JUnit 5 加 Testcontainers | JUnit 5 |
| Runtime visibility | Actuator health/info/metrics | 基礎 health endpoint |
| 成本 | 更多生成檔案和配置 | 更小的起點 |

## 代表性片段

產生的 security config 允許存取 `/health`，並以 HTTP Basic 保護應用程式的其他部分。

```java
@Configuration
public class SecurityConfig {
  @Bean
  SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
      .csrf((csrf) -> csrf.disable())
      .authorizeHttpRequests((authorize) -> authorize
        .requestMatchers("/health").permitAll()
        .anyRequest().authenticated()
      )
      .httpBasic(Customizer.withDefaults());

    return http.build();
  }
}
```

Credentials 從環境支援的 properties 中讀取，並帶有本地 defaults。

```java
@Bean
UserDetailsService userDetailsService(
  @Value("${APP_BASIC_USERNAME:admin}") String username,
  @Value("${APP_BASIC_PASSWORD:change-me}") String password,
  PasswordEncoder passwordEncoder
) {
  UserDetails user = User.withUsername(username)
    .password(passwordEncoder.encode(password))
    .roles("USER")
    .build();

  return new InMemoryUserDetailsManager(user);
}
```

開發時用 Basic auth 存取受保護 endpoints：

```bash
curl -u "$APP_BASIC_USERNAME:$APP_BASIC_PASSWORD" \
  http://localhost:8080/users
```

選擇 Flyway 時，schema creation 由 Flyway 負責：

```sql
CREATE TABLE IF NOT EXISTS app_users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email VARCHAR(255) NOT NULL UNIQUE,
  display_name VARCHAR(255) NOT NULL,
  created_at TIMESTAMP NOT NULL
);
```

## 遷移與測試

Flyway 會改變 Hibernate 的角色：產生的設定會驗證 schema，而不是讓 Hibernate 自由更新它。每次資料庫變更都會新增新的 `V2__...sql` 文件，並讓 migrations 保持可 review。

```bash
./gradlew test
./gradlew bootRun
./gradlew build
```

當 Docker 可用時運行 Testcontainers lane。如果本地環境沒有 Docker，把 unit tests 與 container tests 分開，讓快速 lane 仍然可用。

```java
@SpringBootTest
class ApplicationContainerTests {
  @Test
  void contextLoadsAgainstContainerDatabase() {
  }
}
```

## 部署說明

部署前，取代本地 credentials；如果產生的 build 仍然只包含 H2，新增 PostgreSQL JDBC driver；配置 managed PostgreSQL datasource；並把 Flyway migrations 放進 startup 或 relsource orchestration 中運行。

```bash
export APP_BASIC_USERNAME="service-admin"
export APP_BASIC_PASSWORD="replace-with-secret"
export SPRING_DATASOURCE_URL="jdbc:postgresql://db.example.com:5432/my_secure_spring_api"
export SPRING_DATASOURCE_DRIVER_CLASS_NAME="org.postgresql.Driver"
export SPRING_DATASOURCE_USERNAME="app"
export SPRING_DATASOURCE_PASSWORD="replace-with-secret"
./gradlew build
java -jar build/libs/*.jar
```

Actuator 透過 `--java-libraries spring-actuator` 選擇，因此 health 和 metrics endpoints 可用於平台 probes。在把服務放到公網前，請檢視 endpoint exposure。

## 故障排查

| 症狀 | 檢查 |
| --- | --- |
| 所有 endpoints 都回傳 401 | 除 `/health` 外，都需要傳送 Basic auth credentials。 |
| Flyway 啟動失敗 | 確認資料庫可訪問，並且 migration history 與 schema 匹配。 |
| Testcontainers tests 本地失敗 | 確保 Docker 正在運行，且目前使用者可以啟動 containers。 |
| App 使用預設 `admin/change-me` | 在 runtime environment 中設定 `APP_BASIC_USERNAME` 和 `APP_BASIC_PASSWORD`。 |
| Actuator endpoint 缺失 | 確認 `spring-actuator` 仍然存在於 `--java-libraries`。 |

## 取捨

這個 stack 比最小 Spring Boot API 更重。如果想要更小的起點，請使用 [Spring Boot with PostgreSQL and JPA](/guides/java/spring-boot-postgres-jpa/)，或在 builder 中移除選用 libraries。
