永続性、セキュリティ、データベース移行、および統合テスト ツールを備えた Spring Boot バックエンドが必要な場合は、このスタックを使用します。

```npm
npm create better-fullstack@latest my-secure-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## これが生み出すもの

- Spring Boot Java API。
- Gradle をビルド ツールとして使用します。
- 春のデータ JPA。
- Spring Security。
- スプリング アクチュエーターと Flyway。
- JUnit 5 とテストコンテナ。

## 生成された形状

このスタックは、本番環境の API ベースラインに近づくように開始されます。これには、Gradle ラッパー ファイル、Spring Security 構成、JPA サンプル エンドポイント、Flyway 移行ファイル、アクチュエーター エンドポイント、および Testcontainers テスト パスが含まれます。

```txt
my-secure-spring-api/
├── gradlew
├── gradle/
├── build.gradle.kts
├── settings.gradle.kts
├── src/main/java/com/example/mysecurespringapi/
│   ├── Application.java
│   ├── config/
│   │   └── SecurityConfig.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── UserController.java
│   ├── domain/
│   │   └── AppUser.java
│   ├── repository/
│   │   └── AppUserRepository.java
│   └── service/
│       └── AppUserService.java
├── src/main/resources/
│   ├── application.yml
│   └── db/migration/V1__init.sql
└── src/test/java/com/example/mysecurespringapi/
    ├── ApplicationTests.java
    └── ApplicationContainerTests.java
```

生成されたセキュリティ パスは、メモリ内ユーザーで HTTP Basic を使用します。これは、ローカルおよび内部の API 作業のための実質的に安全なデフォルトであり、完全な製品認証システムではありません。

## 互換性に関する注意事項

Spring Security、JPA、および Spring ライブラリでは、Java ビルド ツールで Spring Boot が必要です。 Flyway は、Better Fullstack の Spring Data JPA パスとペアになっています。

追加の互換性の詳細:

- `--java-auth spring-security` は、Maven または Gradle を使用する Spring Boot プロジェクトに対してのみ出力されます。
- `--java-libraries flyway` は、`--java-orm spring-data-jpa` が選択されている場合にのみ保持されます。
- Flyway と Liquibase の両方がより広範なスタックで選択されている場合、ジェネレーターは Flyway を保持し、このパスの Liquibase を削除します。
- `--java-testing-libraries testcontainers` はコンテナベースの統合テストのサポートを追加するため、そのテストレーンではローカルの Docker の可用性が重要になります。

## いつ選択するか

セキュリティ、移行、および統合テストが早期に存在する必要がある実稼働指向の Java API の場合は、これを選択します。

最小の JPA サンプルが必要で、後でセキュリティを追加する予定がある場合は、最小の Spring ガイドを選択してください。最初のコミットで「エンドポイントはどのように保護されるのか、スキーマはどのように変更されるのか、実際のデータベースに対してどのようにテストするのか」という答えがすでに得られている場合には、このガイドを選択してください。

| 決定点 | セキュアスプリング API | 最小スプリング JPA |
| --- | --- | --- |
| 安全 | Spring Security を介した HTTP 基本 | 未選択 |
| 移行 | 最初からFlyway | 後で追加 |
| テスト | JUnit 5 とテストコンテナ | JUnit 5 |
| 実行時の可視​​性 | アクチュエータの健全性/情報/メトリクス | 基本的な健康エンドポイント |
| 料金 | さらに生成されたファイルと構成 | より小さな開始点 |

## 代表的なスニペット

生成されたセキュリティ構成は `/health` を許可し、アプリケーションの残りの部分を HTTP Basic で保護します。

```java
@Configuration
public class SecurityConfig {
  @Bean
  SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
      .csrf((csrf) -> csrf.disable())
      .authorizeHttpRequests((authorize) -> authorize
        .requestMatchers("/health").permitAll()
        .anyRequest().authenticated()
      )
      .httpBasic(Customizer.withDefaults());

    return http.build();
  }
}
```

資格情報は、ローカルのデフォルトを使用して環境に基づくプロパティから読み取られます。

```java
@Bean
UserDetailsService userDetailsService(
  @Value("${APP_BASIC_USERNAME:admin}") String username,
  @Value("${APP_BASIC_PASSWORD:change-me}") String password,
  PasswordEncoder passwordEncoder
) {
  UserDetails user = User.withUsername(username)
    .password(passwordEncoder.encode(password))
    .roles("USER")
    .build();

  return new InMemoryUserDetailsManager(user);
}
```

開発中に、基本認証で保護されたエンドポイントを使用します。

```bash
curl -u "$APP_BASIC_USERNAME:$APP_BASIC_PASSWORD" \
  http://localhost:8080/users
```

Flyway は、選択するとスキーマの作成を所有します。

```sql
CREATE TABLE IF NOT EXISTS app_users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email VARCHAR(255) NOT NULL UNIQUE,
  display_name VARCHAR(255) NOT NULL,
  created_at TIMESTAMP NOT NULL
);
```

## 移行とテスト

Flyway は Hibernate の姿勢を変更します。生成された設定は、Hibernate が自由に更新できるようにするのではなく、スキーマを検証します。データベースの変更ごとに新しい `V2__...sql` ファイルを追加し、移行を確認できるようにします。

```bash
./gradlew test
./gradlew bootRun
./gradlew build
```

Docker が使用可能な場合は、Testcontainers レーンを実行します。ローカル環境で Docker が使用できない場合は、高速レーンが引き続き使用できるように、単体テストをコンテナー テストとは別にしてください。

```java
@SpringBootTest
class ApplicationContainerTests {
  @Test
  void contextLoadsAgainstContainerDatabase() {
  }
}
```

## 導入メモ

デプロイする前に、ローカル認証情報を置き換え、生成されたビルドにまだ H2 のみが含まれている場合は PostgreSQL JDBC ドライバーを追加し、マネージド PostgreSQL データソースを構成し、起動またはリリース オーケストレーションの一部として Flyway 移行を実行します。

```bash
export APP_BASIC_USERNAME="service-admin"
export APP_BASIC_PASSWORD="replace-with-secret"
export SPRING_DATASOURCE_URL="jdbc:postgresql://db.example.com:5432/my_secure_spring_api"
export SPRING_DATASOURCE_DRIVER_CLASS_NAME="org.postgresql.Driver"
export SPRING_DATASOURCE_USERNAME="app"
export SPRING_DATASOURCE_PASSWORD="replace-with-secret"
./gradlew build
java -jar build/libs/*.jar
```

アクチュエーターは `--java-libraries spring-actuator` を通じて選択されるため、ヘルスおよびメトリクスのエンドポイントはプラットフォーム プローブで使用できます。サービスを公共のインターネットに公開する前に、エンドポイントの公開を確認してください。

## トラブルシューティング

| 症状 | チェック |
| --- | --- |
| すべてのエンドポイントは 401 を返します | `/health` を除くすべての基本認証資格情報を送信します。 |
| Flyway が起動時に失敗する | データベースにアクセス可能であり、移行履歴がスキーマと一致していることを確認します。 |
| Testcontainers テストがローカルで失敗する | Docker が実行中であり、現在のユーザーがコンテナーを開始できることを確認します。 |
| アプリはデフォルトの `admin/change-me` を使用します | 実行時環境で `APP_BASIC_USERNAME` と `APP_BASIC_PASSWORD` を設定します。 |
| アクチュエーターのエンドポイントがありません | `spring-actuator` が `--java-libraries` にまだ存在していることを確認します。 |
