当你需要带 persistence、security、database migrations 和 integration-test tooling 的 Spring Boot backend 时，使用这个 stack。

```npm
npm create better-fullstack@latest my-secure-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## 创建内容

- Spring Boot Java API。
- Gradle 作为 build tool。
- Spring Data JPA。
- Spring Security。
- Spring Actuator 和 Flyway。
- JUnit 5 和 Testcontainers。

## 生成结构

这个 stack 面向生产级 Java API：Spring Security 负责 authentication/authorization，Flyway 负责 migrations，Actuator 暴露 operational endpoints，Testcontainers 让你用真实依赖运行 tests。

代表性结构：

```txt
my-secure-spring-api/
  build.gradle
  src/
    main/
      java/
        .../
          config/
          controllers/
          security/
          services/
      resources/
        db/migration/
    test/
```

把 security rules 放在 configuration 和 filters 附近，business logic 放在 services 中，data access 放在 JPA repositories 后面。

## 兼容性

`--java-auth spring-security` 添加 security layer，不会替代 Spring Data JPA 或 Flyway。`--java-libraries spring-actuator flyway` 适合生产运营 API。`--java-testing-libraries junit5 testcontainers` 为使用真实服务的 integration tests 做好准备。

## 测试与部署

用 Gradle 运行 tests：

```bash
./gradlew test
```

生产环境中配置 Spring profiles、secrets、PostgreSQL URL 和 migrations policy。根据团队 release model，在 startup 前或 startup 期间运行 Flyway。用 Testcontainers 验证依赖真实 PostgreSQL 的行为。
